# 死锁问题排查

## 💡 大白话精华总结

**死锁是什么？**
- 两个或多个线程互相等待对方释放资源，谁也不让步，永远卡住
- 就像两辆车在窄路相遇，谁也不倒车，永远堵在那

**死锁的经典场景：**
```
线程A：持有锁1，等待锁2
线程B：持有锁2，等待锁1
    ↓
互相等待
    ↓
永远卡住
    ↓
死锁！
```

**死锁的4个必要条件（缺一不可）：**
```
1. 互斥条件：资源不能共享，一次只能一个线程用
2. 持有并等待：持有资源的同时，还在等待其他资源
3. 不可剥夺：不能强行抢走别人持有的资源
4. 循环等待：A等B，B等C，C等A，形成环

破坏任意一个条件，就能避免死锁！
```

**死锁代码示例：**
```java
// 经典死锁代码
Object lock1 = new Object();
Object lock2 = new Object();

// 线程1
new Thread(() -> {
    synchronized (lock1) {
        System.out.println("线程1获得lock1");
        Thread.sleep(100);
        synchronized (lock2) {  // 等待lock2
            System.out.println("线程1获得lock2");
        }
    }
}).start();

// 线程2
new Thread(() -> {
    synchronized (lock2) {
        System.out.println("线程2获得lock2");
        Thread.sleep(100);
        synchronized (lock1) {  // 等待lock1
            System.out.println("线程2获得lock1");
        }
    }
}).start();

// 结果：线程1持有lock1等lock2，线程2持有lock2等lock1 → 死锁！
```

**预防死锁的5种方法：**

**1. 固定加锁顺序（最常用）**
```java
// 问题代码：加锁顺序不一致
synchronized (lock1) {
    synchronized (lock2) { }
}
synchronized (lock2) {  // 顺序反了！
    synchronized (lock1) { }
}

// 解决方案：统一加锁顺序
synchronized (lock1) {  // 总是先lock1
    synchronized (lock2) {  // 再lock2
    }
}
```

**2. 使用tryLock超时**
```java
// 使用ReentrantLock的tryLock
Lock lock1 = new ReentrantLock();
Lock lock2 = new ReentrantLock();

if (lock1.tryLock(100, TimeUnit.MILLISECONDS)) {
    try {
        if (lock2.tryLock(100, TimeUnit.MILLISECONDS)) {
            try {
                // 业务逻辑
            } finally {
                lock2.unlock();
            }
        }
    } finally {
        lock1.unlock();
    }
}
// 超时后放弃，避免永久等待
```

**3. 一次性获取所有锁**
```java
// 使用synchronized同时获取
synchronized (lock1) {
    synchronized (lock2) {
        // 一次性获取所有锁
    }
}
```

**4. 使用并发工具类**
```java
// 使用ConcurrentHashMap代替HashMap + synchronized
ConcurrentHashMap<String, Object> map = new ConcurrentHashMap<>();

// 使用线程安全的集合
CopyOnWriteArrayList<Object> list = new CopyOnWriteArrayList<>();
```

**5. 死锁检测和恢复**
```java
// 定期检测死锁
ScheduledExecutorService executor = Executors.newScheduledThreadPool(1);
executor.scheduleAtFixedRate(() -> {
    ThreadMXBean threadBean = ManagementFactory.getThreadMXBean();
    long[] deadlockedThreads = threadBean.findDeadlockedThreads();
    if (deadlockedThreads != null) {
        // 发现死锁，记录日志或告警
        System.err.println("检测到死锁！");
    }
}, 0, 10, TimeUnit.SECONDS);
```

**死锁 vs 活锁 vs 饥饿：**
```
死锁（Deadlock）：
- 互相等待，永远卡住
- 线程状态：BLOCKED
- 解决：破坏死锁条件

活锁（Livelock）：
- 一直在尝试，但总是失败
- 线程状态：RUNNABLE
- 就像两个人在走廊相遇，都想让路，结果一直左右晃
- 解决：随机延迟重试

饥饿（Starvation）：
- 一直得不到资源
- 线程状态：WAITING
- 就像排队，总是被插队
- 解决：公平锁
```

**一句话记住：**
> 死锁是互相等待永远卡住，用jstack检测，固定加锁顺序预防！

## 一、什么是死锁

### 1.1 死锁的定义

```
死锁（Deadlock）：
两个或多个线程互相持有对方需要的资源
导致所有线程都无法继续执行
永久阻塞

经典示例：
线程A持有资源1，等待资源2
线程B持有资源2，等待资源1
    ↓
互相等待
    ↓
永久阻塞
    ↓
死锁
```

### 1.2 死锁示例

```java
public class DeadlockDemo {
    
    private static Object lock1 = new Object();
    private static Object lock2 = new Object();
    
    public static void main(String[] args) {
        // 线程1
        new Thread(() -> {
            synchronized (lock1) {
                System.out.println("Thread1: 持有lock1，等待lock2");
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                synchronized (lock2) {
                    System.out.println("Thread1: 获得lock2");
                }
            }
        }).start();
        
        // 线程2
        new Thread(() -> {
            synchronized (lock2) {
                System.out.println("Thread2: 持有lock2，等待lock1");
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                synchronized (lock1) {
                    System.out.println("Thread2: 获得lock1");
                }
            }
        }).start();
    }
}

// 输出：
// Thread1: 持有lock1，等待lock2
// Thread2: 持有lock2，等待lock1
// （然后永久阻塞）
```

### 1.3 死锁的四个必要条件

```
死锁的四个必要条件（缺一不可）：

1. 互斥条件（Mutual Exclusion）
   - 资源不能被多个线程同时使用
   - 一次只能被一个线程占用

2. 持有并等待（Hold and Wait）
   - 线程持有至少一个资源
   - 同时等待获取其他资源

3. 不可剥夺（No Preemption）
   - 资源不能被强制剥夺
   - 只能由持有者主动释放

4. 循环等待（Circular Wait）
   - 存在线程等待环路
   - A等B，B等C，C等A

破坏任意一个条件，就可以预防死锁
```

### 1.4 死锁的危害

```
危害1：线程永久阻塞
- 涉及的线程无法继续执行
- 占用的资源无法释放
- 影响其他线程

危害2：应用无响应
- 关键线程死锁
- 请求无法处理
- 用户无法使用

危害3：资源浪费
- 线程占用内存
- 连接占用资源
- 系统资源耗尽

危害4：难以排查
- 问题不易复现
- 可能偶发
- 排查成本高
```

## 二、死锁的类型

### 2.1 简单死锁

```java
/**
 * 最简单的死锁：两个线程，两个锁
 */
public class SimpleDeadlock {
    
    private final Object lock1 = new Object();
    private final Object lock2 = new Object();
    
    public void method1() {
        synchronized (lock1) {
            System.out.println(Thread.currentThread().getName() + ": 持有lock1");
            synchronized (lock2) {
                System.out.println(Thread.currentThread().getName() + ": 持有lock2");
            }
        }
    }
    
    public void method2() {
        synchronized (lock2) {
            System.out.println(Thread.currentThread().getName() + ": 持有lock2");
            synchronized (lock1) {
                System.out.println(Thread.currentThread().getName() + ": 持有lock1");
            }
        }
    }
    
    public static void main(String[] args) {
        SimpleDeadlock deadlock = new SimpleDeadlock();
        
        new Thread(() -> deadlock.method1(), "Thread-1").start();
        new Thread(() -> deadlock.method2(), "Thread-2").start();
    }
}

// 死锁关系：
// Thread-1: lock1 → lock2
// Thread-2: lock2 → lock1
// 形成循环等待
```

### 2.2 复杂死锁

```java
/**
 * 复杂死锁：多个线程，多个锁
 */
public class ComplexDeadlock {
    
    private final Object lockA = new Object();
    private final Object lockB = new Object();
    private final Object lockC = new Object();
    
    public void methodA() {
        synchronized (lockA) {
            System.out.println("持有lockA");
            synchronized (lockB) {
                System.out.println("持有lockB");
            }
        }
    }
    
    public void methodB() {
        synchronized (lockB) {
            System.out.println("持有lockB");
            synchronized (lockC) {
                System.out.println("持有lockC");
            }
        }
    }
    
    public void methodC() {
        synchronized (lockC) {
            System.out.println("持有lockC");
            synchronized (lockA) {
                System.out.println("持有lockA");
            }
        }
    }
    
    public static void main(String[] args) {
        ComplexDeadlock deadlock = new ComplexDeadlock();
        
        new Thread(() -> deadlock.methodA(), "Thread-A").start();
        new Thread(() -> deadlock.methodB(), "Thread-B").start();
        new Thread(() -> deadlock.methodC(), "Thread-C").start();
    }
}

// 死锁关系：
// Thread-A: lockA → lockB
// Thread-B: lockB → lockC
// Thread-C: lockC → lockA
// 形成循环等待环
```

### 2.3 动态死锁

```java
/**
 * 动态死锁：转账场景

public class TransferDeadlock {

        private int balance;
        private final int id;
        
        public Account(int id, int balance) {

            this.balance = balance;
        }

        }

            balance += amount;
        }

            transfer.transfer(account2, account1, 200);

        // 线程2持有account2，等待account1
    }
```

## 四、死锁解决方案

### 4.1 方案1：固定加锁顺序

#### 原理

```
问题：
不同线程以不同顺序获取锁
导致循环等待

解决：
所有线程以相同顺序获取锁
避免循环等待
```

#### 实现

```java
/**
 * 转账死锁的解决方案：固定加锁顺序
 */
public class TransferWithOrder {
    
    public static class Account {
        private int balance;
        private final int id;
        
        public Account(int id, int balance) {
            this.id = id;
            this.balance = balance;
        }
        
        public void debit(int amount) {
            balance -= amount;
        }
        
        public void credit(int amount) {
            balance += amount;
        }
    }
    
    // 解决方案：按账户ID顺序加锁
    public void transfer(Account from, Account to, int amount) {
        Account first, second;
        
        // 确定加锁顺序
        if (from.id < to.id) {
            first = from;
            second = to;
        } else {
            first = to;
            second = from;
        }
        
        // 按顺序加锁
        synchronized (first) {
            synchronized (second) {
                from.debit(amount);
                to.credit(amount);
            }
        }
    }
    
    public static void main(String[] args) {
        Account account1 = new Account(1, 1000);
        Account account2 = new Account(2, 1000);
        
        TransferWithOrder transfer = new TransferWithOrder();
        
        // 线程1：account1 → account2
        new Thread(() -> {
            transfer.transfer(account1, account2, 100);
        }).start();
        
        // 线程2：account2 → account1
        new Thread(() -> {
            transfer.transfer(account2, account1, 200);
        }).start();
        
        // 不会死锁：
        // 两个线程都按照 account1 → account2 的顺序加锁
    }
}
```

### 4.2 方案2：使用Lock的tryLock

#### 原理

```
问题：
synchronized无法设置超时
线程可能永久等待

解决：
使用Lock.tryLock(timeout)
超时后放弃，避免死锁
```

#### 实现

```java
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;
import java.util.concurrent.TimeUnit;

/**
 * 使用tryLock避免死锁
 */
public class TransferWithTryLock {
    
    public static class Account {
        private int balance;
        private final Lock lock = new ReentrantLock();
        
        public Account(int balance) {
            this.balance = balance;
        }
        
        public void debit(int amount) {
            balance -= amount;
        }
        
        public void credit(int amount) {
            balance += amount;
        }
        
        public Lock getLock() {
            return lock;
        }
    }
    
    // 使用tryLock，设置超时
    public boolean transfer(Account from, Account to, int amount) throws InterruptedException {
        long timeout = 1000;  // 1秒超时
        
        // 尝试获取第一个锁
        if (from.getLock().tryLock(timeout, TimeUnit.MILLISECONDS)) {
            try {
                // 尝试获取第二个锁
                if (to.getLock().tryLock(timeout, TimeUnit.MILLISECONDS)) {
                    try {
                        // 执行转账
                        from.debit(amount);
                        to.credit(amount);
                        return true;
                    } finally {
                        to.getLock().unlock();
                    }
                }
            } finally {
                from.getLock().unlock();
            }
        }
        
        // 获取锁失败
        return false;
    }
    
    public static void main(String[] args) {
        Account account1 = new Account(1000);
        Account account2 = new Account(1000);
        
        TransferWithTryLock transfer = new TransferWithTryLock();
        
        // 线程1
        new Thread(() -> {
            try {
                boolean success = transfer.transfer(account1, account2, 100);
                System.out.println("Thread1 transfer: " + success);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
        
        // 线程2
        new Thread(() -> {
            try {
                boolean success = transfer.transfer(account2, account1, 200);
                System.out.println("Thread2 transfer: " + success);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
        
        // 不会死锁：
        // 如果获取锁超时，会放弃并返回false
        // 可以重试或记录日志
    }
}
```

### 4.3 方案3：使用单一锁

#### 原理

```
问题：
多个锁可能导致死锁

解决：
使用单一锁保护所有资源
避免多锁问题
```

#### 实现

```java
/**
 * 使用单一锁避免死锁
 */
public class TransferWithSingleLock {
    
    // 全局锁
    private static final Object GLOBAL_LOCK = new Object();
    
    public static class Account {
        private int balance;
        
        public Account(int balance) {
            this.balance = balance;
        }
        
        public void debit(int amount) {
            balance -= amount;
        }
        
        public void credit(int amount) {
            balance += amount;
        }
    }
    
    // 使用全局锁
    public void transfer(Account from, Account to, int amount) {
        synchronized (GLOBAL_LOCK) {
            from.debit(amount);
            to.credit(amount);
        }
    }
    
    // 优点：
    // - 不会死锁
    // - 实现简单
    
    // 缺点：
    // - 并发性能差
    // - 所有转账串行执行
}
```

### 4.4 方案4：避免嵌套锁

#### 原理

```
问题：
嵌套锁容易导致死锁

解决：
避免在持有锁的情况下获取其他锁
```

#### 实现

```java
/**

public class TransferWithoutNesting {
```

## 五、死锁预防

### 5.1 设计原则

```
原则1：避免嵌套锁
- 尽量不要在持有锁时获取其他锁
- 如果必须嵌套，确保顺序一致

原则2：固定加锁顺序
- 为锁定义全局顺序
- 所有线程按相同顺序获取锁

原则3：使用超时机制
- 使用tryLock设置超时
- 超时后放弃并重试

原则4：减小锁的粒度
- 只锁必要的代码
- 减少持锁时间

原则5：使用并发工具类
- 使用ConcurrentHashMap
- 使用原子类
- 避免手动加锁
```

### 5.2 编码规范

```java
/**
 * 死锁预防最佳实践
 */
public class DeadlockPrevention {
    
    // 1. 避免嵌套锁
    public void badExample() {
        synchronized (lock1) {
            synchronized (lock2) {  // 嵌套锁，危险
                // 业务逻辑
            }
        }
    }
    
    public void goodExample() {
        synchronized (lock1) {
            // 业务逻辑1
        }
        synchronized (lock2) {
            // 业务逻辑2
        }
    }
    
    // 2. 使用tryLock
    private Lock lock = new ReentrantLock();
    
    public void withTryLock() {
        try {
            if (lock.tryLock(1, TimeUnit.SECONDS)) {
                try {
                    // 业务逻辑
                } finally {
                    lock.unlock();
                }
            } else {
                // 获取锁失败，记录日志或重试
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
    
    // 3. 使用并发工具类
    private ConcurrentHashMap<String, Object> map = new ConcurrentHashMap<>();
    
    public void useConcurrentUtils() {
        // 不需要手动加锁
        map.put("key", "value");
    }
    
    // 4. 减小锁的粒度
    public void fineLock() {
        // 准备数据（不需要锁）
        Object data = prepareData();
        
        // 只锁必要的部分
        synchronized (lock) {
            // 最小化的临界区
            updateSharedState(data);
        }
        
        // 后续处理（不需要锁）
        postProcess();
    }
}
```

## 六、死锁 vs 活锁 vs 饥饿

### 6.1 死锁（Deadlock）

```
定义：
线程互相等待对方持有的资源
永久阻塞

特征：
- 线程状态：BLOCKED
- 无法继续执行
- 需要外部干预

示例：
Thread-1持有A，等待B
Thread-2持有B，等待A
```

### 6.2 活锁（Livelock）

```
定义：
线程不断重试
但始终无法成功
一直在运行但没有进展

特征：
- 线程状态：RUNNABLE
- CPU占用高
- 没有实际进展

示例：
两个人在走廊相遇
都想让对方先过
不断左右移动
但都无法通过
```

```java
/**
 * 活锁示例
 */
public class LivelockDemo {
    
    static class Spoon {
        private Diner owner;
        
        public synchronized void use() {
            System.out.println(owner.name + " is eating");
        }
        
        public synchronized void setOwner(Diner diner) {
            this.owner = diner;
        }
        
        public synchronized Diner getOwner() {
            return owner;
        }
    }
    
    static class Diner {
        private String name;
        private boolean isHungry;
        
        public Diner(String name) {
            this.name = name;
            this.isHungry = true;
        }
        
        public void eatWith(Spoon spoon, Diner spouse) {
            while (isHungry) {
                // 如果勺子不是自己的
                if (spoon.getOwner() != this) {
                    try {
                        Thread.sleep(1);
                    } catch (InterruptedException e) {
                        continue;
                    }
                    continue;
                }
                
                // 如果配偶饿了，让给配偶
                if (spouse.isHungry) {
                    System.out.println(name + ": 你先吃吧");
                    spoon.setOwner(spouse);
                    continue;
                }
                
                // 吃饭
                spoon.use();
                isHungry = false;
                spoon.setOwner(spouse);
            }
        }
    }
    
    public static void main(String[] args) {
        Diner husband = new Diner("丈夫");
        Diner wife = new Diner("妻子");
        
        Spoon spoon = new Spoon();
        spoon.setOwner(husband);
        
        new Thread(() -> husband.eatWith(spoon, wife)).start();
        new Thread(() -> wife.eatWith(spoon, husband)).start();
        
        // 活锁：
        // 两人都想让对方先吃
        // 不断把勺子让给对方
        // 但都吃不到饭
    }
}
```

### 6.3 饥饿（Starvation）

```
定义：
线程长时间无法获取所需资源
一直等待

特征：
- 线程状态：WAITING/TIMED_WAITING
- 低优先级线程
- 可能最终获得资源

原因：
1. 优先级设置不当
2. 锁被长时间占用
3. 资源分配不公平

示例：
高优先级线程一直抢占CPU
低优先级线程无法执行
```

```java
/**
 * 饥饿示例
 */
public class StarvationDemo {
    
    private static final Object lock = new Object();
    
    public static void main(String[] args) {
        // 高优先级线程
        Thread highPriority = new Thread(() -> {
            while (true) {
                synchronized (lock) {
                    // 长时间持有锁
                    try {
                        Thread.sleep(100);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
            }
        });
        highPriority.setPriority(Thread.MAX_PRIORITY);
        highPriority.start();
        
        // 低优先级线程
        Thread lowPriority = new Thread(() -> {
            while (true) {
                synchronized (lock) {
                    System.out.println("低优先级线程获得锁");
                }
            }
        });
        lowPriority.setPriority(Thread.MIN_PRIORITY);
        lowPriority.start();
        
        // 饥饿：
        // 低优先级线程很难获得锁
        // 可能长时间无法执行
    }
}
```

### 6.4 对比总结

| 对比项 | 死锁 | 活锁 | 饥饿 |
|-------|------|------|------|
| **线程状态** | BLOCKED | RUNNABLE | WAITING |
| **CPU占用** | 低 | 高 | 低 |
| **是否有进展** | 无 | 无 | 可能有 |
| **能否自动恢复** | 否 | 可能 | 可能 |
| **检测难度** | 容易 | 困难 | 中等 |
