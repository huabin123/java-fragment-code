# 打破双亲委派模型

## 💡 大白话精华总结

**为什么要打破双亲委派？**
- 双亲委派虽好，但有些场景不适用，必须"不听爸爸的话"

**3个必须打破的典型场景：**

**1. JDBC驱动加载（SPI机制）**
```
问题：
- DriverManager在核心库（rt.jar），由启动类加载器加载
- MySQL驱动在你的项目里，由应用类加载器加载
- 启动类加载器看不到应用类加载器的类！

解决：
- 用"线程上下文类加载器"，让核心类反向使用应用类加载器
- 就像爷爷说："这事我干不了，让孙子来！"
```

**2. Tomcat多应用隔离**
```
问题：
- Web应用A用Spring 4.0
- Web应用B用Spring 5.0
- 双亲委派会让它们共用同一个Spring版本！

解决：
- 每个Web应用用自己的类加载器
- 各自加载各自的Spring，互不干扰
```

**3. 热部署（代码热替换）**
```
问题：
- 修改代码后，双亲委派会返回旧的类（已加载过）
- 无法实现热更新

解决：
- 扔掉旧的类加载器，创建新的类加载器
- 用新的类加载器重新加载类
```

**怎么打破双亲委派？**
1. **重写loadClass()方法**：不先问爸爸，自己先加载
2. **使用线程上下文类加载器**：让核心类反向使用应用类加载器
3. **自定义类加载器架构**：完全自己设计加载规则（如Tomcat）

**Tomcat的类加载器结构：**
```
Bootstrap ClassLoader（核心类）
    ↓
Common ClassLoader（Tomcat共享类）
    ↓
    ├─→ Catalina ClassLoader（Tomcat内部类）
    └─→ Shared ClassLoader（所有Web应用共享）
         ↓
         ├─→ WebApp1 ClassLoader（应用1独有）
         └─→ WebApp2 ClassLoader（应用2独有）
```

**一句话记住：**
> 打破双亲委派就是"不听爸爸的话"，为了实现类隔离、热部署和反向加载！

## 一、为什么要打破双亲委派模型

### 1.1 双亲委派模型的局限性

虽然双亲委派模型有很多优点，但在以下场景中会遇到问题：

#### 1. 基础类需要调用用户代码

```java
// 场景：JDBC驱动加载
// DriverManager在rt.jar中，由Bootstrap ClassLoader加载
// 但MySQL驱动在ClassPath中，由Application ClassLoader加载
// Bootstrap ClassLoader无法访问Application ClassLoader加载的类

public class DriverManager {
    static {
        // 如何加载用户提供的驱动？
        loadInitialDrivers();
    }
}
```

#### 2. 代码热替换

```java
// 场景：开发环境热部署
// 修改代码后，需要重新加载类
// 但双亲委派模型会返回已加载的类
Class<?> oldClass = loader.loadClass("com.example.Service");
// 修改代码后
Class<?> newClass = loader.loadClass("com.example.Service");
// oldClass == newClass（无法热替换）
```

#### 3. 应用隔离

```java
// 场景：Tomcat中多个Web应用
// 不同应用可能使用不同版本的同一个类库
// 双亲委派模型无法实现类的隔离
```

### 1.2 需要打破双亲委派的场景

| 场景 | 原因 | 解决方案 |
|------|------|---------|
| JDBC驱动加载 | 基础类调用用户代码 | 线程上下文类加载器 |
| Tomcat | 应用隔离 | 自定义类加载器架构 |
| OSGi | 模块化 | 网状类加载器结构 |
| 热部署 | 代码热替换 | 自定义类加载器 |
| SPI机制 | 服务提供者加载 | 线程上下文类加载器 |

## 二、打破双亲委派的三种方式

### 2.1 方式一：重写loadClass()方法

#### 原理

```java
// 双亲委派模型的实现在loadClass()方法中
// 重写此方法可以改变委派逻辑

public class BreakParentDelegationLoader extends ClassLoader {
    
    @Override
    public Class<?> loadClass(String name) throws ClassNotFoundException {
        // 不委派给父加载器，直接自己加载
        Class<?> c = findLoadedClass(name);
        if (c == null) {
            // 自己查找并加载
            c = findClass(name);
        }
        return c;
    }
    
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        // 实现类加载逻辑
        byte[] classData = loadClassData(name);
        return defineClass(name, classData, 0, classData.length);
    }
}
```

#### 示例：优先加载自己的类

```java
public class SelfFirstClassLoader extends ClassLoader {
    
    private String classPath;
    
    public SelfFirstClassLoader(String classPath) {
        this.classPath = classPath;
    }
    
    @Override
    public Class<?> loadClass(String name) throws ClassNotFoundException {
        // 1. 检查是否已加载
        Class<?> c = findLoadedClass(name);
        if (c != null) {
            return c;
        }
        
        // 2. 对于java.*等核心类，委派给父加载器
        if (name.startsWith("java.") || name.startsWith("javax.")) {
            return super.loadClass(name);
        }
        
        // 3. 优先自己加载
        try {
            c = findClass(name);
            return c;
        } catch (ClassNotFoundException e) {
            // 4. 自己加载失败，委派给父加载器
            return super.loadClass(name);
        }
    }
    
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        try {
            byte[] classData = loadClassData(name);
            return defineClass(name, classData, 0, classData.length);
        } catch (IOException e) {
            throw new ClassNotFoundException(name, e);
        }
    }
    
    private byte[] loadClassData(String name) throws IOException {
        String fileName = classPath + "/" + name.replace('.', '/') + ".class";
        try (InputStream is = new FileInputStream(fileName);
             ByteArrayOutputStream baos = new ByteArrayOutputStream()) {
            
            byte[] buffer = new byte[1024];
            int len;
            while ((len = is.read(buffer)) != -1) {
                baos.write(buffer, 0, len);
            }
            return baos.toByteArray();
        }
    }
}
```

⚠️ **注意**：
- 不要加载`java.*`等核心类，会抛出`SecurityException`
- 必须保证核心类由Bootstrap ClassLoader加载

### 2.2 方式二：线程上下文类加载器

#### 原理

线程上下文类加载器（Thread Context ClassLoader）可以让父类加载器请求子类加载器完成类加载。

```java
// 设置线程上下文类加载器
Thread.currentThread().setContextClassLoader(classLoader);

// 获取线程上下文类加载器
ClassLoader contextLoader = Thread.currentThread().getContextClassLoader();
```

#### 工作流程

```
传统双亲委派：
    子加载器 → 父加载器 → 祖父加载器
    （向上委派）

线程上下文类加载器：
    父加载器 → 线程上下文类加载器（子加载器）
    （向下委派）
```

## 三、SPI机制与线程上下文类加载器

### 3.1 什么是SPI

SPI（Service Provider Interface）是Java提供的一种服务发现机制，允许第三方为接口提供实现。

```
核心类库（接口）  ←  第三方实现
    ↓                    ↓
Bootstrap CL      Application CL
```

### 3.2 SPI的使用

#### 1. 定义接口

```java
// 在核心类库中定义接口
package java.sql;

public interface Driver {
    Connection connect(String url, Properties info) throws SQLException;
    boolean acceptsURL(String url) throws SQLException;
}
```

#### 2. 提供实现

```java
// MySQL驱动实现（在第三方jar中）
package com.mysql.jdbc;

public class Driver implements java.sql.Driver {
    static {
        try {
            DriverManager.registerDriver(new Driver());
        } catch (SQLException e) {
            throw new RuntimeException("Can't register driver!");
        }
    }
    
    @Override
    public Connection connect(String url, Properties info) {
        // 实现连接逻辑
    }
}
```

#### 3. 配置服务提供者

```
在jar包中创建文件：
META-INF/services/java.sql.Driver

文件内容：
com.mysql.jdbc.Driver
```

#### 4. 使用ServiceLoader加载

```java
public class DriverManager {
    
    static {
        loadInitialDrivers();
    }
    
    private static void loadInitialDrivers() {
        AccessController.doPrivileged(new PrivilegedAction<Void>() {
            public Void run() {
                // 使用ServiceLoader加载所有Driver实现
                ServiceLoader<Driver> loadedDrivers = ServiceLoader.load(Driver.class);
                Iterator<Driver> driversIterator = loadedDrivers.iterator();
                
                try {
                    while(driversIterator.hasNext()) {
                        driversIterator.next();
                    }
                } catch(Throwable t) {
                    // 忽略
                }
                return null;
            }
        });
    }
}
```

### 3.3 ServiceLoader的实现

```java
public final class ServiceLoader<S> implements Iterable<S> {
    
    // 服务接口
    private final Class<S> service;
    
    // 类加载器
    private final ClassLoader loader;
    
    // 已加载的服务提供者
    private LinkedHashMap<String,S> providers = new LinkedHashMap<>();
    
    // 懒加载迭代器
    private LazyIterator lookupIterator;
    
    public static <S> ServiceLoader<S> load(Class<S> service) {
        // 使用线程上下文类加载器
        ClassLoader cl = Thread.currentThread().getContextClassLoader();
        return ServiceLoader.load(service, cl);
    }
    
    public static <S> ServiceLoader<S> load(Class<S> service, ClassLoader loader) {
        return new ServiceLoader<>(service, loader);
    }
    
    private ServiceLoader(Class<S> svc, ClassLoader cl) {
        service = Objects.requireNonNull(svc, "Service interface cannot be null");
        loader = (cl == null) ? ClassLoader.getSystemClassLoader() : cl;
        reload();
    }
    
    public void reload() {
        providers.clear();
        lookupIterator = new LazyIterator(service, loader);
    }
    
    public Iterator<S> iterator() {
        return new Iterator<S>() {
            
            Iterator<Map.Entry<String,S>> knownProviders = providers.entrySet().iterator();
            
            public boolean hasNext() {
                if (knownProviders.hasNext())
                    return true;
                return lookupIterator.hasNext();
            }
            
            public S next() {
                if (knownProviders.hasNext())
                    return knownProviders.next().getValue();
                return lookupIterator.next();
            }
        };
    }
    
    // 懒加载迭代器
    private class LazyIterator implements Iterator<S> {
        
        Class<S> service;
        ClassLoader loader;
        Enumeration<URL> configs = null;
        Iterator<String> pending = null;
        String nextName = null;
        
        private LazyIterator(Class<S> service, ClassLoader loader) {
            this.service = service;
            this.loader = loader;
        }
        
        public boolean hasNext() {
            if (nextName != null) {
                return true;
            }
            if (configs == null) {
                try {
                    // 读取META-INF/services/接口全限定名文件
                    String fullName = "META-INF/services/" + service.getName();
                    if (loader == null)
                        configs = ClassLoader.getSystemResources(fullName);
                    else
                        configs = loader.getResources(fullName);
                } catch (IOException x) {
                    fail(service, "Error locating configuration files", x);
                }
            }
            while ((pending == null) || !pending.hasNext()) {
                if (!configs.hasMoreElements()) {
                    return false;
                }
                pending = parse(service, configs.nextElement());
            }
            nextName = pending.next();
            return true;
        }
        
        public S next() {
            if (!hasNext()) {
                throw new NoSuchElementException();
            }
            String cn = nextName;
            nextName = null;
            Class<?> c = null;
            try {
                // 使用指定的类加载器加载实现类
                c = Class.forName(cn, false, loader);
            } catch (ClassNotFoundException x) {
                fail(service, "Provider " + cn + " not found");
            }
            if (!service.isAssignableFrom(c)) {
                fail(service, "Provider " + cn  + " not a subtype");
            }
            try {
                S p = service.cast(c.newInstance());
                providers.put(cn, p);
                return p;
            } catch (Throwable x) {
                fail(service, "Provider " + cn + " could not be instantiated", x);
            }
            throw new Error();
        }
    }
}
```

### 3.4 SPI打破双亲委派的原理

```
1. DriverManager由Bootstrap ClassLoader加载
   ↓
2. DriverManager调用ServiceLoader.load(Driver.class)
   ↓
3. ServiceLoader使用Thread.currentThread().getContextClassLoader()
   获取Application ClassLoader
   ↓
4. 使用Application ClassLoader加载META-INF/services配置
   ↓
5. 使用Application ClassLoader加载MySQL驱动实现类
   ↓
6. 完成"父加载器请求子加载器加载类"的逆向操作
```

### 3.5 自定义SPI示例

```java
// 1. 定义服务接口
public interface PaymentService {
    void pay(double amount);
}

// 2. 实现服务（支付宝）
public class AlipayService implements PaymentService {
    @Override
    public void pay(double amount) {
        System.out.println("支付宝支付: " + amount);
    }
}

// 3. 实现服务（微信）
public class WechatPayService implements PaymentService {
    @Override
    public void pay(double amount) {
        System.out.println("微信支付: " + amount);
    }
}

// 4. 配置文件
// META-INF/services/com.example.PaymentService
// com.example.AlipayService
// com.example.WechatPayService

// 5. 使用SPI
public class SPIDemo {
    public static void main(String[] args) {
        ServiceLoader<PaymentService> loader = ServiceLoader.load(PaymentService.class);
        
        for (PaymentService service : loader) {
            service.pay(100.0);
        }
    }
}

// 输出：
// 支付宝支付: 100.0
// 微信支付: 100.0
```

## 四、Tomcat的类加载器架构

### 4.1 Tomcat类加载器层次

```
        Bootstrap ClassLoader
                ↓
        Extension ClassLoader
                ↓
        System ClassLoader
                ↓
        Common ClassLoader
            ↙       ↘
    Catalina      Shared
    ClassLoader   ClassLoader
                      ↓
                  WebApp ClassLoader
                      ↓
                  Jsp ClassLoader
```

### 4.2 各类加载器的作用

| 类加载器 | 加载路径 | 作用 | 可见性 |
|---------|---------|------|--------|
| **Common** | `$CATALINA_HOME/lib` | Tomcat和所有Web应用共享 | 对Tomcat和所有Web应用可见 |
| **Catalina** | `$CATALINA_HOME/server/lib` | 仅Tomcat使用 | 仅对Tomcat可见 |
| **Shared** | `$CATALINA_HOME/shared/lib` | 所有Web应用共享 | 对所有Web应用可见，对Tomcat不可见 |
| **WebApp** | `/WEB-INF/lib`, `/WEB-INF/classes` | 当前Web应用私有 | 仅对当前Web应用可见 |
| **Jsp** | 动态生成的JSP类 | JSP文件 | 仅对当前JSP可见 |

### 4.3 WebAppClassLoader实现

```java
public class WebAppClassLoader extends URLClassLoader {
    
    // 是否遵循双亲委派模型
    private boolean delegate = false;
    
    @Override
    public Class<?> loadClass(String name, boolean resolve) 
        throws ClassNotFoundException {
        
        synchronized (getClassLoadingLock(name)) {
            Class<?> clazz = null;
            
            // 1. 检查本地缓存
            clazz = findLoadedClass(name);
            if (clazz != null) {
                if (resolve) {
                    resolveClass(clazz);
                }
                return clazz;
            }
            
            // 2. 检查JVM缓存
            try {
                clazz = findLoadedClass0(name);
                if (clazz != null) {
                    if (resolve) {
                        resolveClass(clazz);
                    }
                    return clazz;
                }
            } catch (Exception e) {
                // ignore
            }
            
            // 3. 使用System ClassLoader加载（防止覆盖核心类）
            try {
                clazz = getSystemClassLoader().loadClass(name);
                if (clazz != null) {
                    if (resolve) {
                        resolveClass(clazz);
                    }
                    return clazz;
                }
            } catch (ClassNotFoundException e) {
                // ignore
            }
            
            // 4. 是否遵循双亲委派
            boolean delegateLoad = delegate || filter(name);
            
            if (delegateLoad) {
                // 遵循双亲委派：先委派给父加载器
                try {
                    clazz = Class.forName(name, false, getParent());
                    if (clazz != null) {
                        if (resolve) {
                            resolveClass(clazz);
                        }
                        return clazz;
                    }
                } catch (ClassNotFoundException e) {
                    // ignore
                }
            }
            
            // 5. 从当前Web应用加载
            try {
                clazz = findClass(name);
                if (clazz != null) {
                    if (resolve) {
                        resolveClass(clazz);
                    }
                    return clazz;
                }
            } catch (ClassNotFoundException e) {
                // ignore
            }
            
            // 6. 如果不遵循双亲委派，最后尝试父加载器
            if (!delegateLoad) {
                try {
                    clazz = Class.forName(name, false, getParent());
                    if (clazz != null) {
                        if (resolve) {
                            resolveClass(clazz);
                        }
                        return clazz;
                    }
                } catch (ClassNotFoundException e) {
                    // ignore
                }
            }
            
            throw new ClassNotFoundException(name);
        }
    }
    
    // 过滤器：某些类必须由父加载器加载
    private boolean filter(String name) {
        if (name == null) {
            return false;
        }
        
        // javax.*必须由父加载器加载
        if (name.startsWith("javax.")) {
            return true;
        }
        
        // org.xml.*必须由父加载器加载
        if (name.startsWith("org.xml.")) {
            return true;
        }
        
        return false;
    }
}
```

### 4.4 Tomcat类加载顺序

#### delegate=false（默认）

```
1. Bootstrap ClassLoader（JVM核心类）
   ↓ 找不到
2. WebApp ClassLoader（/WEB-INF/classes, /WEB-INF/lib）
   ↓ 找不到
3. Common ClassLoader（$CATALINA_HOME/lib）
   ↓ 找不到
4. System ClassLoader
```

#### delegate=true

```
1. Bootstrap ClassLoader
   ↓ 找不到
2. System ClassLoader
   ↓ 找不到
3. Common ClassLoader
   ↓ 找不到
4. WebApp ClassLoader
```

### 4.5 为什么Tomcat要这样设计？

#### 1. 隔离性

```java
// 应用A使用Spring 4.x
// /webapps/appA/WEB-INF/lib/spring-core-4.3.jar

// 应用B使用Spring 5.x
// /webapps/appB/WEB-INF/lib/spring-core-5.2.jar

// 两个应用使用不同的WebAppClassLoader，互不影响
```

#### 2. 灵活性

```java
// 同一个类库的不同版本可以在不同应用中共存
// 每个应用有自己的类加载器，加载自己的版本
```

#### 3. 共享性

```java
// 公共类库放在Common中，所有应用共享
// $CATALINA_HOME/lib/servlet-api.jar
// 节省内存，避免重复加载
```

#### 4. 热部署

```java
// 重新创建WebAppClassLoader即可实现热部署
public void reload(Context context) {
    // 1. 停止旧的WebAppClassLoader
    WebAppClassLoader oldLoader = context.getLoader();
    oldLoader.stop();
    
    // 2. 创建新的WebAppClassLoader
    WebAppClassLoader newLoader = new WebAppClassLoader(context.getDocBase());
    context.setLoader(newLoader);
    
    // 3. 重新加载应用
    newLoader.start();
}
```

### 4.6 配置delegate模式

```xml
<!-- 在context.xml中配置 -->
<Context>
    <Loader delegate="true"/>
</Context>
```

## 五、OSGi的类加载机制

### 5.1 OSGi简介

OSGi（Open Service Gateway Initiative）是一个动态模块化系统，实现了完全的模块化。

### 5.2 OSGi类加载器结构

```
传统双亲委派（树状）：
    Bootstrap
        ↓
    Extension
        ↓
    Application
        ↓
    Custom

OSGi（网状）：
    Bundle A ←→ Bundle B
        ↓           ↓
    Bundle C ←→ Bundle D
```

### 5.3 OSGi类加载规则

```java
// OSGi的类加载顺序：
// 1. 检查是否是java.*开头，是则委派给父加载器
// 2. 检查是否在Import-Package中，是则委派给导出该包的Bundle
// 3. 检查是否在自己的Bundle中
// 4. 检查是否在Fragment Bundle中
// 5. 检查是否在Require-Bundle中
// 6. 查找失败

public class OSGiClassLoader extends ClassLoader {
    
    private Bundle bundle;
    
    @Override
    protected Class<?> loadClass(String name, boolean resolve) 
        throws ClassNotFoundException {
        
        // 1. java.*委派给父加载器
        if (name.startsWith("java.")) {
            return getParent().loadClass(name);
        }
        
        // 2. 检查Import-Package
        String packageName = getPackageName(name);
        if (bundle.isImportedPackage(packageName)) {
            Bundle exportBundle = bundle.findExportBundle(packageName);
            return exportBundle.loadClass(name);
        }
        
        // 3. 检查自己的Bundle
        Class<?> c = findOwnClass(name);
        if (c != null) {
            return c;
        }
        
        // 4. 检查Fragment Bundle
        for (Bundle fragment : bundle.getFragments()) {
            try {
                return fragment.loadClass(name);
            } catch (ClassNotFoundException e) {
                // 继续查找
            }
        }
        
        // 5. 检查Require-Bundle
        for (Bundle requiredBundle : bundle.getRequiredBundles()) {
            try {
                return requiredBundle.loadClass(name);
            } catch (ClassNotFoundException e) {
                // 继续查找
            }
        }
        
        throw new ClassNotFoundException(name);
    }
    
    private String getPackageName(String className) {
        int lastDot = className.lastIndexOf('.');
        return (lastDot == -1) ? "" : className.substring(0, lastDot);
    }
}
```

## 六、打破双亲委派的最佳实践

### 6.1 何时应该打破双亲委派

✅ **应该打破的场景**：
1. 实现应用隔离（如Tomcat）
2. 实现热部署
3. 实现插件系统
4. SPI机制
5. 模块化系统（如OSGi）

❌ **不应该打破的场景**：
1. 普通应用程序
2. 简单的类加载需求
3. 没有特殊隔离需求

### 6.2 打破双亲委派的注意事项

#### 1. 保护核心类

```java
public class SafeClassLoader extends ClassLoader {
    
    @Override
    public Class<?> loadClass(String name) throws ClassNotFoundException {
        // 核心类必须委派给父加载器
        if (name.startsWith("java.") || 
            name.startsWith("javax.") ||
            name.startsWith("sun.")) {
            return super.loadClass(name);
        }
        
        // 其他类可以自己加载
        return findClass(name);
    }
}
```

#### 2. 避免类加载死锁

```java
public class DeadlockSafeClassLoader extends ClassLoader {
    
    @Override
    protected Class<?> loadClass(String name, boolean resolve) 
        throws ClassNotFoundException {
        
        // 使用类名作为锁，避免死锁
        synchronized (getClassLoadingLock(name)) {
            // 加载逻辑
        }
    }
}
```

#### 3. 正确处理资源

```java
public class ResourceAwareClassLoader extends ClassLoader {
    
    @Override
    public URL getResource(String name) {
        // 先查找自己的资源
        URL url = findResource(name);
        if (url != null) {
            return url;
        }
        
        // 再查找父加载器的资源
        return super.getResource(name);
    }
}
```

### 6.3 性能优化

```java
public class OptimizedClassLoader extends ClassLoader {
    
    // 缓存已加载的类
    private final Map<String, Class<?>> classCache = new ConcurrentHashMap<>();
    
    @Override
    protected Class<?> loadClass(String name, boolean resolve) 
        throws ClassNotFoundException {
        
        // 先查缓存
        Class<?> c = classCache.get(name);
        if (c != null) {
            return c;
        }
        
        // 加载类
        c = super.loadClass(name, resolve);
        
        // 放入缓存
        classCache.put(name, c);
        
        return c;
    }
}
```

## 七、实战案例

### 案例1：实现简单的插件系统

```java
public class PluginClassLoader extends ClassLoader {
    
    private String pluginPath;
    
    public PluginClassLoader(String pluginPath, ClassLoader parent) {
        super(parent);
        this.pluginPath = pluginPath;
    }
    
    @Override
    public Class<?> loadClass(String name) throws ClassNotFoundException {
        // 核心类委派给父加载器
        if (name.startsWith("java.") || name.startsWith("com.example.api.")) {
            return super.loadClass(name);
        }
        
        // 插件类自己加载
        if (name.startsWith("com.example.plugin.")) {
            return findClass(name);
        }
        
        // 其他类委派给父加载器
        return super.loadClass(name);
    }
    
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        try {
            byte[] classData = loadClassData(name);
            return defineClass(name, classData, 0, classData.length);
        } catch (IOException e) {
            throw new ClassNotFoundException(name, e);
        }
    }
    
    private byte[] loadClassData(String name) throws IOException {
        String fileName = pluginPath + "/" + name.replace('.', '/') + ".class";
        try (InputStream is = new FileInputStream(fileName);
             ByteArrayOutputStream baos = new ByteArrayOutputStream()) {
            
            byte[] buffer = new byte[1024];
            int len;
            while ((len = is.read(buffer)) != -1) {
                baos.write(buffer, 0, len);
            }
            return baos.toByteArray();
        }
    }
}

// 插件管理器
public class PluginManager {
    
    private Map<String, PluginClassLoader> plugins = new ConcurrentHashMap<>();
    
    public void loadPlugin(String pluginName, String pluginPath) throws Exception {
        // 创建插件类加载器
        PluginClassLoader loader = new PluginClassLoader(
            pluginPath,
            this.getClass().getClassLoader()
        );
        
        plugins.put(pluginName, loader);
        
        // 加载插件主类
        Class<?> pluginClass = loader.loadClass("com.example.plugin." + pluginName + ".Main");
        Object plugin = pluginClass.newInstance();
        
        // 初始化插件
        Method initMethod = pluginClass.getMethod("init");
        initMethod.invoke(plugin);
        
        System.out.println("插件 " + pluginName + " 加载成功");
    }
    
    public void unloadPlugin(String pluginName) {
        PluginClassLoader loader = plugins.remove(pluginName);
        if (loader != null) {
            // 清理资源
            try {
                Method cleanupMethod = loader.loadClass("com.example.plugin." + pluginName + ".Main")
                    .getMethod("cleanup");
                cleanupMethod.invoke(null);
            } catch (Exception e) {
                e.printStackTrace();
            }
            
            // 将类加载器置为null，等待GC回收
            loader = null;
            System.gc();
            
            System.out.println("插件 " + pluginName + " 卸载成功");
        }
    }
    
    public void reloadPlugin(String pluginName, String pluginPath) throws Exception {
        unloadPlugin(pluginName);
        Thread.sleep(100);  // 等待GC
        loadPlugin(pluginName, pluginPath);
    }
}
```

### 案例2：实现热部署

```java
public class HotDeployClassLoader extends ClassLoader {
    
    private String classPath;
    private long lastModified;
    
    public HotDeployClassLoader(String classPath) {
        this.classPath = classPath;
    }
    
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        try {
            byte[] classData = loadClassData(name);
            return defineClass(name, classData, 0, classData.length);
        } catch (IOException e) {
            throw new ClassNotFoundException(name, e);
        }
    }
    
    private byte[] loadClassData(String name) throws IOException {
        String fileName = classPath + "/" + name.replace('.', '/') + ".class";
        File file = new File(fileName);
        lastModified = file.lastModified();
        
        try (InputStream is = new FileInputStream(file);
             ByteArrayOutputStream baos = new ByteArrayOutputStream()) {
            
            byte[] buffer = new byte[1024];
            int len;
            while ((len = is.read(buffer)) != -1) {
                baos.write(buffer, 0, len);
            }
            return baos.toByteArray();
        }
    }
    
    public boolean isModified(String name) {
        String fileName = classPath + "/" + name.replace('.', '/') + ".class";
        File file = new File(fileName);
        return file.lastModified() > lastModified;
    }
}

// 热部署管理器
public class HotDeployManager {
    
    private String classPath;
    private String className;
    private HotDeployClassLoader loader;
    private Object instance;
    
    public HotDeployManager(String classPath, String className) {
        this.classPath = classPath;
        this.className = className;
        reload();
    }
    
    public void reload() {
        try {
            // 创建新的类加载器
            loader = new HotDeployClassLoader(classPath);
            
            // 加载类
            Class<?> clazz = loader.loadClass(className);
            
            // 创建实例
            instance = clazz.newInstance();
            
            System.out.println("类 " + className + " 重新加载成功");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
    
    public Object invoke(String methodName, Object... args) throws Exception {
        if (loader.isModified(className)) {
            reload();
        }
        
        Class<?> clazz = instance.getClass();
        Method method = clazz.getMethod(methodName);
        return method.invoke(instance, args);
    }
    
    public static void main(String[] args) throws Exception {
        HotDeployManager manager = new HotDeployManager(
            "/path/to/classes",
            "com.example.Service"
        );
        
        while (true) {
            // 调用方法
            Object result = manager.invoke("execute");
            System.out.println("结果: " + result);
            
            // 等待5秒
            Thread.sleep(5000);
        }
    }
}
```

## 八、常见问题

### Q1: 什么时候必须打破双亲委派？

**A**: 
1. 基础类需要调用用户代码（如JDBC）
2. 需要实现应用隔离（如Tomcat）
3. 需要实现热部署
4. 需要实现模块化（如OSGi）

### Q2: 如何安全地打破双亲委派？

**A**: 
1. 核心类（java.*、javax.*）必须委派给父加载器
2. 使用synchronized避免死锁
3. 正确处理资源加载
4. 及时清理类加载器引用

### Q3: 线程上下文类加载器的默认值是什么？

**A**: 
- 默认是Application ClassLoader
- 可以通过`Thread.setContextClassLoader()`修改

### Q4: Tomcat为什么默认不遵循双亲委派？

**A**: 
- 实现应用隔离
- 允许不同应用使用不同版本的类库
- 优先加载应用自己的类

### Q5: 如何避免类加载器内存泄漏？

**A**: 
1. 及时清理类加载器引用
2. 避免在静态字段中持有类加载器
3. 使用弱引用
4. 定期检查内存
