# 类加载器与双亲委派模型

## 💡 大白话精华总结

**类加载器是什么？**
- 就是负责把.class文件加载到JVM的"搬运工"
- Java有3个内置的"搬运工"，层层递进

**3个内置类加载器（从上到下）：**
1. **启动类加载器（Bootstrap）**：老大，加载Java核心类库（如String、Object）
2. **扩展类加载器（Extension）**：老二，加载扩展类库（lib/ext目录）
3. **应用类加载器（Application）**：老三，加载你写的代码（ClassPath）

**什么是双亲委派？**
- 简单说：儿子不自己干活，先让爸爸干，爸爸让爷爷干，爷爷干不了才轮到爸爸，爸爸干不了才轮到儿子
- 专业说：加载类时，先委托给父加载器，父加载器加载不了，才自己加载

**双亲委派的流程：**
```
你的类要加载 → 应用类加载器（老三）
                ↓ 先问老二
              扩展类加载器（老二）
                ↓ 先问老大
              启动类加载器（老大）
                ↓ 老大找不到
              扩展类加载器（老二）尝试加载
                ↓ 老二也找不到
              应用类加载器（老三）自己加载
```

**为什么要这样设计？**
- **安全**：防止核心类被篡改（你不能自己写个假的String类替换掉真的）
- **避免重复**：同一个类只会被加载一次

**举个例子：**
```java
// 假如你写了一个假的String类
package java.lang;
public class String {
    // 恶意代码
}

// 加载时：
// 1. 应用类加载器：先问扩展类加载器
// 2. 扩展类加载器：先问启动类加载器
// 3. 启动类加载器：我有String！用我的！（rt.jar里的）
// 4. 你的假String永远不会被加载 ✅ 安全！
```

**一句话记住：**
> 双亲委派就是"有事先找爸爸"，保证核心类库不被篡改！

## 一、类加载器概述

### 1.1 类加载器的作用

类加载器负责读取Java字节代码，并转换成`java.lang.Class`类的一个实例。

```java
// 类加载器的核心方法
public abstract class ClassLoader {
    // 加载类
    public Class<?> loadClass(String name) throws ClassNotFoundException {
        // ...
    }
    
    // 查找类（由子类实现）
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        // ...
    }
    
    // 定义类
    protected final Class<?> defineClass(String name, byte[] b, int off, int len) {
        // ...
    }
}
```

### 1.2 类加载器的层次结构

```
┌─────────────────────────────────────┐
│  Bootstrap ClassLoader              │  启动类加载器（C++实现）
│  加载：<JAVA_HOME>/lib              │
└─────────────────────────────────────┘
                ↑ 父加载器
┌─────────────────────────────────────┐
│  Extension ClassLoader              │  扩展类加载器
│  加载：<JAVA_HOME>/lib/ext          │
└─────────────────────────────────────┘
                ↑ 父加载器
┌─────────────────────────────────────┐
│  Application ClassLoader            │  应用程序类加载器
│  加载：ClassPath                     │
└─────────────────────────────────────┘
                ↑ 父加载器
┌─────────────────────────────────────┐
│  Custom ClassLoader                 │  自定义类加载器
│  加载：用户指定路径                  │
└─────────────────────────────────────┘
```

## 二、JVM内置的类加载器

### 2.1 启动类加载器（Bootstrap ClassLoader）

#### 特点

- 由C++实现，是虚拟机自身的一部分
- 负责加载存放在`<JAVA_HOME>/lib`目录中的核心类库
- 或者被`-Xbootclasspath`参数指定的路径中的类库
- 无法被Java程序直接引用

#### 加载的类库

```
rt.jar          - Java运行时核心类库
resources.jar   - 资源文件
charsets.jar    - 字符集
jce.jar         - 加密扩展
jsse.jar        - 安全套接字扩展
```

#### 示例

```java
public class BootstrapClassLoaderDemo {
    public static void main(String[] args) {
        // String类由启动类加载器加载
        ClassLoader classLoader = String.class.getClassLoader();
        System.out.println("String类的类加载器: " + classLoader);  // null
        
        // 启动类加载器加载的路径
        String bootClassPath = System.getProperty("sun.boot.class.path");
        System.out.println("启动类加载器加载路径:");
        for (String path : bootClassPath.split(":")) {
            System.out.println("  " + path);
        }
    }
}

// 输出：
// String类的类加载器: null
// 启动类加载器加载路径:
//   /Library/Java/JavaVirtualMachines/jdk1.8.0_301.jdk/Contents/Home/jre/lib/resources.jar
//   /Library/Java/JavaVirtualMachines/jdk1.8.0_301.jdk/Contents/Home/jre/lib/rt.jar
//   ...
```

### 2.2 扩展类加载器（Extension ClassLoader）

#### 特点

- 由Java实现，类名为`sun.misc.Launcher$ExtClassLoader`
- 负责加载`<JAVA_HOME>/lib/ext`目录中的类库
- 或者被`java.ext.dirs`系统变量指定的路径中的类库
- 开发者可以直接使用扩展类加载器

#### 加载的类库

```
dnsns.jar       - DNS命名服务
localedata.jar  - 本地化数据
sunec.jar       - 椭圆曲线加密
sunjce_provider.jar - JCE提供者
zipfs.jar       - ZIP文件系统
```

#### 示例

```java
public class ExtensionClassLoaderDemo {
    public static void main(String[] args) {
        // 获取扩展类加载器
        ClassLoader extClassLoader = ClassLoader.getSystemClassLoader().getParent();
        System.out.println("扩展类加载器: " + extClassLoader);
        
        // 扩展类加载器加载的路径
        String extDirs = System.getProperty("java.ext.dirs");
        System.out.println("扩展类加载器加载路径:");
        for (String path : extDirs.split(":")) {
            System.out.println("  " + path);
        }
    }
}

// 输出：
// 扩展类加载器: sun.misc.Launcher$ExtClassLoader@...
// 扩展类加载器加载路径:
//   /Users/xxx/Library/Java/Extensions
//   /Library/Java/JavaVirtualMachines/jdk1.8.0_301.jdk/Contents/Home/jre/lib/ext
//   /Library/Java/Extensions
//   /Network/Library/Java/Extensions
//   /System/Library/Java/Extensions
//   /usr/lib/java
```

### 2.3 应用程序类加载器（Application ClassLoader）

#### 特点

- 由Java实现，类名为`sun.misc.Launcher$AppClassLoader`
- 负责加载用户类路径（ClassPath）上的类库
- 是程序中默认的类加载器
- 可以通过`ClassLoader.getSystemClassLoader()`获取

#### 示例

```java
public class ApplicationClassLoaderDemo {
    public static void main(String[] args) {
        // 获取应用程序类加载器
        ClassLoader appClassLoader = ClassLoader.getSystemClassLoader();
        System.out.println("应用程序类加载器: " + appClassLoader);
        
        // 当前类的类加载器
        ClassLoader currentClassLoader = ApplicationClassLoaderDemo.class.getClassLoader();
        System.out.println("当前类的类加载器: " + currentClassLoader);
        
        // 类路径
        String classPath = System.getProperty("java.class.path");
        System.out.println("类路径:");
        for (String path : classPath.split(":")) {
            System.out.println("  " + path);
        }
    }
}

// 输出：
// 应用程序类加载器: sun.misc.Launcher$AppClassLoader@...
// 当前类的类加载器: sun.misc.Launcher$AppClassLoader@...
// 类路径:
//   /Users/xxx/project/target/classes
//   /Users/xxx/.m2/repository/...
```

### 2.4 类加载器对比

| 类加载器 | 实现语言 | 父加载器 | 加载路径 | 获取方式 |
|---------|---------|---------|---------|---------|
| Bootstrap | C++ | 无 | `<JAVA_HOME>/lib` | null |
| Extension | Java | Bootstrap | `<JAVA_HOME>/lib/ext` | `getParent()` |
| Application | Java | Extension | ClassPath | `getSystemClassLoader()` |
| Custom | Java | Application | 自定义 | 自己创建 |

## 三、双亲委派模型

### 3.1 什么是双亲委派模型

双亲委派模型（Parent Delegation Model）是类加载器的工作模式：

```
当一个类加载器收到类加载请求时：
1. 首先不会自己尝试加载这个类
2. 而是把请求委派给父类加载器完成
3. 每一层都是如此
4. 只有当父加载器反馈无法完成加载时
5. 子加载器才会尝试自己加载
```

### 3.2 双亲委派模型的实现

```java
// java.lang.ClassLoader
protected Class<?> loadClass(String name, boolean resolve)
    throws ClassNotFoundException
{
    synchronized (getClassLoadingLock(name)) {
        // 1. 首先检查类是否已经被加载
        Class<?> c = findLoadedClass(name);
        if (c == null) {
            long t0 = System.nanoTime();
            try {
                // 2. 如果没有被加载，委派给父加载器
                if (parent != null) {
                    c = parent.loadClass(name, false);
                } else {
                    // 3. 如果父加载器为null，使用启动类加载器
                    c = findBootstrapClassOrNull(name);
                }
            } catch (ClassNotFoundException e) {
                // 父加载器无法加载，抛出ClassNotFoundException
            }

            if (c == null) {
                // 4. 如果父加载器无法加载，自己尝试加载
                long t1 = System.nanoTime();
                c = findClass(name);

                // 记录统计信息
                sun.misc.PerfCounter.getParentDelegationTime().addTime(t1 - t0);
                sun.misc.PerfCounter.getFindClassTime().addElapsedTimeFrom(t1);
                sun.misc.PerfCounter.getFindClasses().increment();
            }
        }
        if (resolve) {
            // 5. 链接类
            resolveClass(c);
        }
        return c;
    }
}
```

### 3.3 双亲委派模型的工作流程

```
加载 com.example.User 类

Step 1: Application ClassLoader 收到请求
        ↓ 委派给父加载器
Step 2: Extension ClassLoader 收到请求
        ↓ 委派给父加载器
Step 3: Bootstrap ClassLoader 收到请求
        ↓ 尝试加载
        ✗ 在 <JAVA_HOME>/lib 中找不到
        ↓ 返回失败
Step 4: Extension ClassLoader 尝试加载
        ✗ 在 <JAVA_HOME>/lib/ext 中找不到
        ↓ 返回失败
Step 5: Application ClassLoader 尝试加载
        ✓ 在 ClassPath 中找到
        ↓ 加载成功
返回 Class 对象
```

### 3.4 双亲委派模型示例

```java
public class ParentDelegationDemo {
    public static void main(String[] args) throws Exception {
        // 自定义类加载器
        ClassLoader customLoader = new CustomClassLoader();
        
        // 加载String类（会委派到Bootstrap ClassLoader）
        Class<?> stringClass = customLoader.loadClass("java.lang.String");
        System.out.println("String类的类加载器: " + stringClass.getClassLoader());
        
        // 加载自定义类（会由CustomClassLoader加载）
        Class<?> userClass = customLoader.loadClass("com.example.User");
        System.out.println("User类的类加载器: " + userClass.getClassLoader());
    }
}

// 输出：
// String类的类加载器: null（Bootstrap ClassLoader）
// User类的类加载器: CustomClassLoader@...
```

## 四、双亲委派模型的优势

### 4.1 避免类的重复加载

```java
// 场景：多个类加载器加载同一个类
public class AvoidDuplicateLoadingDemo {
    public static void main(String[] args) throws Exception {
        ClassLoader loader1 = new CustomClassLoader();
        ClassLoader loader2 = new CustomClassLoader();
        
        // 两个类加载器加载String类
        Class<?> class1 = loader1.loadClass("java.lang.String");
        Class<?> class2 = loader2.loadClass("java.lang.String");
        
        // 由于双亲委派，都委派给Bootstrap ClassLoader
        // 所以加载的是同一个类
        System.out.println(class1 == class2);  // true
    }
}
```

### 4.2 保护核心类库

```java
// 尝试自定义java.lang.String类
package java.lang;

public class String {
    // 恶意代码
    public void hack() {
        System.out.println("I'm a fake String!");
    }
}

// 运行时会报错：
// java.lang.SecurityException: Prohibited package name: java.lang
```

**原因**：
- 双亲委派模型会先委派给Bootstrap ClassLoader
- Bootstrap ClassLoader会加载JDK自带的String类
- 自定义的String类永远不会被加载

### 4.3 保证类的唯一性

```java
// 类的唯一性由类加载器和类本身共同决定
public class ClassUniquenessDemo {
    public static void main(String[] args) throws Exception {
        ClassLoader loader1 = new CustomClassLoader();
        ClassLoader loader2 = new CustomClassLoader();
        
        // 加载核心类（双亲委派）
        Class<?> string1 = loader1.loadClass("java.lang.String");
        Class<?> string2 = loader2.loadClass("java.lang.String");
        System.out.println("String类相同: " + (string1 == string2));  // true
        
        // 加载自定义类（不同的类加载器）
        Class<?> user1 = loader1.loadClass("com.example.User");
        Class<?> user2 = loader2.loadClass("com.example.User");
        System.out.println("User类相同: " + (user1 == user2));  // false
    }
}
```

### 4.4 实现类的隔离

```java
// 不同的类加载器可以加载相同的类，实现隔离
public class ClassIsolationDemo {
    public static void main(String[] args) throws Exception {
        // 插件1的类加载器
        ClassLoader plugin1Loader = new PluginClassLoader("plugin1");
        Class<?> plugin1Class = plugin1Loader.loadClass("com.example.Plugin");
        Object plugin1 = plugin1Class.newInstance();
        
        // 插件2的类加载器
        ClassLoader plugin2Loader = new PluginClassLoader("plugin2");
        Class<?> plugin2Class = plugin2Loader.loadClass("com.example.Plugin");
        Object plugin2 = plugin2Class.newInstance();
        
        // 两个插件的类是不同的
        System.out.println(plugin1Class == plugin2Class);  // false
        System.out.println(plugin1 instanceof plugin2Class);  // 编译错误
    }
}
```

## 五、类的唯一性

### 5.1 类的唯一性判定

在JVM中，判断两个类是否"相同"，需要满足以下条件：

1. **类的全限定名相同**
2. **加载这个类的类加载器相同**

```java
public class ClassEqualityDemo {
    public static void main(String[] args) throws Exception {
        // 使用不同的类加载器加载同一个类
        ClassLoader loader1 = new CustomClassLoader();
        ClassLoader loader2 = new CustomClassLoader();
        
        Class<?> class1 = loader1.loadClass("com.example.User");
        Class<?> class2 = loader2.loadClass("com.example.User");
        
        // 类名相同，但类加载器不同
        System.out.println("类名相同: " + class1.getName().equals(class2.getName()));  // true
        System.out.println("类相同: " + (class1 == class2));  // false
        
        // 创建实例
        Object obj1 = class1.newInstance();
        Object obj2 = class2.newInstance();
        
        // instanceof判断
        System.out.println(obj1 instanceof class1);  // 编译错误
        System.out.println(class1.isInstance(obj1));  // true
        System.out.println(class1.isInstance(obj2));  // false
    }
}
```

### 5.2 类加载器的命名空间

每个类加载器都有自己的命名空间，命名空间由该加载器及所有父加载器所加载的类组成。

```
Bootstrap ClassLoader的命名空间：
    - java.lang.String
    - java.lang.Object
    - ...

Extension ClassLoader的命名空间：
    - Bootstrap的所有类
    - javax.crypto.*
    - ...

Application ClassLoader的命名空间：
    - Bootstrap的所有类
    - Extension的所有类
    - com.example.*
    - ...
```

**规则**：
- 子加载器可以看见父加载器加载的类
- 父加载器看不见子加载器加载的类
- 兄弟加载器之间互相看不见

```java
public class NamespaceDemo {
    public static void main(String[] args) throws Exception {
        ClassLoader appLoader = ClassLoader.getSystemClassLoader();
        ClassLoader extLoader = appLoader.getParent();
        
        // Application ClassLoader可以看到Extension ClassLoader加载的类
        Class<?> class1 = appLoader.loadClass("javax.crypto.Cipher");
        System.out.println("成功加载: " + class1.getName());
        
        // Extension ClassLoader看不到Application ClassLoader加载的类
        try {
            Class<?> class2 = extLoader.loadClass("com.example.User");
        } catch (ClassNotFoundException e) {
            System.out.println("Extension ClassLoader无法加载User类");
        }
    }
}
```

## 六、自定义类加载器

### 6.1 为什么需要自定义类加载器

1. **隔离加载类**：中间件、插件系统
2. **修改类加载方式**：按需加载、延迟加载
3. **扩展加载源**：从数据库、网络、加密文件加载
4. **防止源码泄漏**：加密class文件

### 6.2 自定义类加载器的实现步骤

```java
public class CustomClassLoader extends ClassLoader {
    
    private String classPath;
    
    public CustomClassLoader(String classPath) {
        this.classPath = classPath;
    }
    
    // 核心方法：查找类
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        try {
            // 1. 读取类文件的字节码
            byte[] classData = loadClassData(name);
            if (classData == null) {
                throw new ClassNotFoundException();
            }
            
            // 2. 将字节码转换为Class对象
            return defineClass(name, classData, 0, classData.length);
        } catch (IOException e) {
            throw new ClassNotFoundException(name, e);
        }
    }
    
    // 加载类文件的字节码
    private byte[] loadClassData(String name) throws IOException {
        // 将类名转换为文件路径
        String fileName = classPath + "/" + name.replace('.', '/') + ".class";
        
        // 读取文件
        try (InputStream is = new FileInputStream(fileName);
             ByteArrayOutputStream baos = new ByteArrayOutputStream()) {
            
            byte[] buffer = new byte[1024];
            int len;
            while ((len = is.read(buffer)) != -1) {
                baos.write(buffer, 0, len);
            }
            return baos.toByteArray();
        }
    }
}
```

### 6.3 使用自定义类加载器

```java
public class CustomClassLoaderDemo {
    public static void main(String[] args) throws Exception {
        // 创建自定义类加载器
        CustomClassLoader loader = new CustomClassLoader("/path/to/classes");
        
        // 加载类
        Class<?> clazz = loader.loadClass("com.example.User");
        System.out.println("类加载器: " + clazz.getClassLoader());
        
        // 创建实例
        Object obj = clazz.newInstance();
        
        // 调用方法（使用反射）
        Method method = clazz.getMethod("getName");
        String name = (String) method.invoke(obj);
        System.out.println("Name: " + name);
    }
}
```

### 6.4 加密类加载器

```java
public class EncryptedClassLoader extends ClassLoader {
    
    private String classPath;
    private String key;
    
    public EncryptedClassLoader(String classPath, String key) {
        this.classPath = classPath;
        this.key = key;
    }
    
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        try {
            // 1. 读取加密的类文件
            byte[] encryptedData = loadEncryptedClassData(name);
            
            // 2. 解密
            byte[] classData = decrypt(encryptedData, key);
            
            // 3. 定义类
            return defineClass(name, classData, 0, classData.length);
        } catch (Exception e) {
            throw new ClassNotFoundException(name, e);
        }
    }
    
    private byte[] loadEncryptedClassData(String name) throws IOException {
        String fileName = classPath + "/" + name.replace('.', '/') + ".encrypted";
        try (InputStream is = new FileInputStream(fileName);
             ByteArrayOutputStream baos = new ByteArrayOutputStream()) {
            
            byte[] buffer = new byte[1024];
            int len;
            while ((len = is.read(buffer)) != -1) {
                baos.write(buffer, 0, len);
            }
            return baos.toByteArray();
        }
    }
    
    private byte[] decrypt(byte[] data, String key) {
        // 简单的异或加密（实际应使用AES等算法）
        byte[] result = new byte[data.length];
        byte[] keyBytes = key.getBytes();
        for (int i = 0; i < data.length; i++) {
            result[i] = (byte) (data[i] ^ keyBytes[i % keyBytes.length]);
        }
        return result;
    }
}
```

### 6.5 网络类加载器

```java
public class NetworkClassLoader extends ClassLoader {
    
    private String baseUrl;
    
    public NetworkClassLoader(String baseUrl) {
        this.baseUrl = baseUrl;
    }
    
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        try {
            // 1. 构建URL
            String url = baseUrl + "/" + name.replace('.', '/') + ".class";
            
            // 2. 从网络下载类文件
            byte[] classData = downloadClassData(url);
            
            // 3. 定义类
            return defineClass(name, classData, 0, classData.length);
        } catch (Exception e) {
            throw new ClassNotFoundException(name, e);
        }
    }
    
    private byte[] downloadClassData(String urlString) throws IOException {
        URL url = new URL(urlString);
        try (InputStream is = url.openStream();
             ByteArrayOutputStream baos = new ByteArrayOutputStream()) {
            
            byte[] buffer = new byte[1024];
            int len;
            while ((len = is.read(buffer)) != -1) {
                baos.write(buffer, 0, len);
            }
            return baos.toByteArray();
        }
    }
}
```

## 八、实战案例

### 案例1：实现热部署

```java
public class HotSwapClassLoader extends ClassLoader {
    
    private String classPath;
    
    public HotSwapClassLoader(String classPath) {
        this.classPath = classPath;
    }
    
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        try {
            byte[] classData = loadClassData(name);
            return defineClass(name, classData, 0, classData.length);
        } catch (IOException e) {
            throw new ClassNotFoundException(name, e);
        }
    }
    
    private byte[] loadClassData(String name) throws IOException {
        String fileName = classPath + "/" + name.replace('.', '/') + ".class";
        try (InputStream is = new FileInputStream(fileName);
             ByteArrayOutputStream baos = new ByteArrayOutputStream()) {
            
            byte[] buffer = new byte[1024];
            int len;
            while ((len = is.read(buffer)) != -1) {
                baos.write(buffer, 0, len);
            }
            return baos.toByteArray();
        }
    }
    
    // 重新加载类
    public Class<?> reload(String name) throws ClassNotFoundException {
        return findClass(name);
    }
}

// 使用示例
public class HotSwapDemo {
    public static void main(String[] args) throws Exception {
        String classPath = "/path/to/classes";
        
        while (true) {
            // 每次都创建新的类加载器
            HotSwapClassLoader loader = new HotSwapClassLoader(classPath);
            
            // 加载类
            Class<?> clazz = loader.loadClass("com.example.Service");
            Object service = clazz.newInstance();
            
            // 调用方法
            Method method = clazz.getMethod("execute");
            method.invoke(service);
            
            // 等待5秒后重新加载
            Thread.sleep(5000);
        }
    }
}
```

### 案例2：插件系统

```java
public class PluginClassLoader extends ClassLoader {
    
    private String pluginPath;
    
    public PluginClassLoader(String pluginPath, ClassLoader parent) {
        super(parent);
        this.pluginPath = pluginPath;
    }
    
    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {
        // 只加载插件包中的类
        if (!name.startsWith("com.example.plugin")) {
            throw new ClassNotFoundException(name);
        }
        
        try {
            byte[] classData = loadClassData(name);
            return defineClass(name, classData, 0, classData.length);
        } catch (IOException e) {
            throw new ClassNotFoundException(name, e);
        }
    }
    
    private byte[] loadClassData(String name) throws IOException {
        String fileName = pluginPath + "/" + name.replace('.', '/') + ".class";
        try (InputStream is = new FileInputStream(fileName);
             ByteArrayOutputStream baos = new ByteArrayOutputStream()) {
            
            byte[] buffer = new byte[1024];
            int len;
            while ((len = is.read(buffer)) != -1) {
                baos.write(buffer, 0, len);
            }
            return baos.toByteArray();
        }
    }
}

// 插件管理器
public class PluginManager {
    
    private Map<String, PluginClassLoader> plugins = new HashMap<>();
    
    // 加载插件
    public void loadPlugin(String pluginName, String pluginPath) throws Exception {
        PluginClassLoader loader = new PluginClassLoader(
            pluginPath, 
            ClassLoader.getSystemClassLoader()
        );
        plugins.put(pluginName, loader);
        
        // 加载插件主类
        Class<?> pluginClass = loader.loadClass("com.example.plugin." + pluginName + ".Main");
        Object plugin = pluginClass.newInstance();
        
        // 初始化插件
        Method initMethod = pluginClass.getMethod("init");
        initMethod.invoke(plugin);
    }
    
    // 卸载插件
    public void unloadPlugin(String pluginName) {
        PluginClassLoader loader = plugins.remove(pluginName);
        // 将类加载器置为null，等待GC回收
        loader = null;
    }
}
```

## 九、常见问题

### Q1: 为什么叫"双亲"委派？

**A**: 
- 这是翻译问题，英文是"Parent Delegation Model"
- "Parent"是单数，应该翻译为"父类委派"
- 实际上每个类加载器只有一个父加载器

### Q2: 如何打破双亲委派模型？

**A**: 
- 重写`loadClass()`方法（不推荐）
- 使用线程上下文类加载器
- 使用OSGi等模块化框架
- 详见下一篇文档

### Q3: 类加载器的父子关系是继承关系吗？

**A**: 
- 不是继承关系
- 是组合关系（通过`parent`字段关联）

### Q4: 为什么Bootstrap ClassLoader返回null？

**A**: 
- Bootstrap ClassLoader由C++实现
- 在Java中没有对应的类
- 所以返回null表示

### Q5: 如何避免类加载器泄漏？

**A**: 
- 及时清理类加载器的引用
- 避免在静态字段中持有类加载器
- 使用弱引用
- 定期检查内存

### 1. 自定义类加载器的建议
