## 为什么需要直接内存

### 问题1：传统IO的性能瓶颈是什么？

**传统IO流程（使用堆内存）**：

```
应用程序读取文件
    ↓
┌─────────────────────────────────────────────┐
│ 1. 应用程序发起read()系统调用               │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│ 2. 操作系统将数据从磁盘读到内核缓冲区       │
│    (Kernel Buffer)                          │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│ 3. 数据从内核缓冲区复制到JVM堆内存          │  ← 额外的复制开销
│    (Heap Memory)                            │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│ 4. 应用程序处理数据                         │
└─────────────────────────────────────────────┘

问题：
❌ 多了一次内存复制（内核 → JVM堆）
❌ 增加了CPU开销
❌ 降低了IO性能
```

**使用直接内存的NIO流程**：

```
应用程序读取文件（NIO）
    ↓
┌─────────────────────────────────────────────┐
│ 1. 应用程序发起read()系统调用               │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│ 2. 操作系统将数据直接读到直接内存           │
│    (Direct Memory / Native Memory)          │
│    ↓                                        │
│    DMA (Direct Memory Access)               │  ← 零拷贝
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│ 3. 应用程序直接访问直接内存                 │
└─────────────────────────────────────────────┘

优势：
✅ 减少一次内存复制
✅ 降低CPU开销
✅ 提高IO性能
✅ 支持零拷贝（Zero-Copy）
```

### 问题2：什么是直接内存？

**定义**：

```
直接内存（Direct Memory）
    ↓
也称为堆外内存（Off-Heap Memory）
    ↓
特点：
1. 不在JVM堆内存中分配
2. 直接在操作系统的本地内存中分配
3. 通过Native函数库（如malloc）分配
4. 不受JVM堆大小限制
5. 不受GC管理（需要手动释放）
```

**内存布局对比**：

```
┌─────────────────────────────────────────────┐
│              JVM进程内存                     │
├─────────────────────────────────────────────┤
│                                              │
│  ┌────────────────────────────────────┐     │
│  │         JVM堆内存                   │     │
│  │  - 受-Xmx参数限制                   │     │
│  │  - 受GC管理                         │     │
│  │  - 分配速度快                       │     │
│  │  - 适合频繁创建的小对象             │     │
│  └────────────────────────────────────┘     │
│                                              │
│  ┌────────────────────────────────────┐     │
│  │       直接内存（堆外内存）          │     │
│  │  - 受-XX:MaxDirectMemorySize限制    │     │
│  │  - 不受GC管理                       │     │
│  │  - 分配/释放成本高                  │     │
│  │  - 适合大块、长期存活的数据         │     │
│  │  - 适合IO操作                       │     │
│  └────────────────────────────────────┘     │
│                                              │
│  ┌────────────────────────────────────┐     │
│  │         元空间（Metaspace）         │     │
│  │  - 存储类元数据                     │     │
│  │  - 使用本地内存                     │     │
│  └────────────────────────────────────┘     │
│                                              │
│  ┌────────────────────────────────────┐     │
│  │         其他本地内存                │     │
│  │  - 线程栈                           │     │
│  │  - JNI代码                          │     │
```

## 直接内存的原理

### 问题3：DirectByteBuffer是如何分配内存的？

**分配流程**：

```java
// 应用层代码
ByteBuffer buffer = ByteBuffer.allocateDirect(1024);

// JDK内部实现
public static ByteBuffer allocateDirect(int capacity) {
    return new DirectByteBuffer(capacity);
}

// DirectByteBuffer构造函数
DirectByteBuffer(int cap) {
    super(-1, 0, cap, cap);
    
    // 1. 检查直接内存是否超限
    boolean pa = VM.isDirectMemoryPageAligned();
    int ps = Bits.pageSize();
    long size = Math.max(1L, (long)cap + (pa ? ps : 0));
    
    // 2. 预留直接内存空间（记账）
    Bits.reserveMemory(size, cap);
    
    long base = 0;
    try {
        // 3. 调用Unsafe分配本地内存
        base = unsafe.allocateMemory(size);
    } catch (OutOfMemoryError x) {
        Bits.unreserveMemory(size, cap);
        throw x;
    }
    
    // 4. 初始化内存为0
    unsafe.setMemory(base, size, (byte) 0);
    
    // 5. 设置内存地址
    if (pa && (base % ps != 0)) {
        address = base + ps - (base & (ps - 1));
    } else {
        address = base;
    }
    
    // 6. 注册Cleaner（用于自动回收）
    cleaner = Cleaner.create(this, new Deallocator(base, size, cap));
    
    att = null;
}
```

**底层实现（Unsafe.allocateMemory）**：

```c
// HotSpot源码：unsafe.cpp

UNSAFE_ENTRY(jlong, Unsafe_AllocateMemory(JNIEnv *env, jobject unsafe, jlong size))
  UnsafeWrapper("Unsafe_AllocateMemory");
  
  // 调用操作系统的malloc分配内存
  void* p = os::malloc(size, mtInternal);
  
  if (p == NULL) {
    // 分配失败，抛出OOM
    throw_new(env, "java/lang/OutOfMemoryError");
    return 0;
  }
  
  // 返回内存地址
  return (jlong)(uintptr_t)p;
UNSAFE_END
```

**内存分配对比**：

```
堆内存分配（new byte[1024]）
    ↓
1. 在JVM堆中分配
2. 通过指针碰撞或空闲列表
3. 分配速度快（纳秒级）
4. 受GC管理
5. 有对象头开销

直接内存分配（ByteBuffer.allocateDirect(1024)）
    ↓
1. 调用Unsafe.allocateMemory
2. 底层调用malloc
3. 分配速度慢（微秒级）
4. 不受GC管理
5. 无对象头开销
```

### 问题4：直接内存如何释放？

**自动释放机制（Cleaner）**：

```java
// DirectByteBuffer中的Cleaner
private static class Deallocator implements Runnable {
    private long address;
    private long size;
    private int capacity;

    private Deallocator(long address, long size, int capacity) {
        this.address = address;
        this.size = size;
        this.capacity = capacity;
    }

    public void run() {
        if (address == 0) {
            return;
        }
        
        // 释放本地内存
        unsafe.freeMemory(address);
        address = 0;
        
        // 释放直接内存记账
        Bits.unreserveMemory(size, capacity);
    }
}

// 注册Cleaner
cleaner = Cleaner.create(this, new Deallocator(base, size, cap));
```

**释放流程**：

```
DirectByteBuffer对象不再被引用
    ↓
GC时，DirectByteBuffer对象被回收
    ↓
触发Cleaner.clean()
    ↓
执行Deallocator.run()
    ↓
调用unsafe.freeMemory(address)
    ↓
底层调用free()释放本地内存
    ↓
直接内存被释放
```

**手动释放**：

```java
// 方式1：通过反射调用cleaner.clean()
public static void clean(ByteBuffer buffer) {
    if (buffer.isDirect()) {
        try {
            Method cleanerMethod = buffer.getClass().getMethod("cleaner");
            cleanerMethod.setAccessible(true);
            Object cleaner = cleanerMethod.invoke(buffer);
            
            Method cleanMethod = cleaner.getClass().getMethod("clean");
            cleanMethod.invoke(cleaner);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

// 方式2：JDK 9+ 使用Unsafe
sun.misc.Unsafe unsafe = getUnsafe();
unsafe.invokeCleaner(buffer);
```

### 问题6：DirectByteBuffer的读写操作如何实现？

**读操作**：

```java
// 读取单个字节
public byte get() {
    // 1. 检查位置
    if (position >= limit) {
        throw new BufferUnderflowException();
    }
    
    // 2. 通过Unsafe直接读取内存
    return unsafe.getByte(address + position++);
}

// 读取字节数组
public ByteBuffer get(byte[] dst, int offset, int length) {
    // 1. 检查参数
    checkBounds(offset, length, dst.length);
    
    // 2. 检查剩余空间
    if (length > remaining()) {
        throw new BufferUnderflowException();
    }
    
    // 3. 批量复制（使用Unsafe.copyMemory）
    long pos = ix(position);
    unsafe.copyMemory(null, pos, dst, 
                      Unsafe.ARRAY_BYTE_BASE_OFFSET + offset, 
                      length);
    position += length;
    return this;
}
```

**写操作**：

```java
// 写入单个字节
public ByteBuffer put(byte x) {
    // 1. 检查位置
    if (position >= limit) {
        throw new BufferOverflowException();
    }
    
    // 2. 通过Unsafe直接写入内存
    unsafe.putByte(address + position++, x);
    return this;
}

// 写入字节数组
public ByteBuffer put(byte[] src, int offset, int length) {
    // 1. 检查参数
    checkBounds(offset, length, src.length);
    
    // 2. 检查剩余空间
    if (length > remaining()) {
        throw new BufferOverflowException();
    }
    
    // 3. 批量复制
    long pos = ix(position);
    unsafe.copyMemory(src, 
                      Unsafe.ARRAY_BYTE_BASE_OFFSET + offset,
                      null, pos, length);
    position += length;
    return this;
}
```

**性能优势**：


## 堆外内存的管理

### 问题7：如何限制直接内存的大小？

**JVM参数**：

```bash
# 设置最大直接内存大小（默认与-Xmx相同）
-XX:MaxDirectMemorySize=512m

# 示例：
java -Xmx2g -XX:MaxDirectMemorySize=512m MyApp
```

**内存记账机制**：

```java
// Bits类：直接内存记账
class Bits {
    // 已分配的直接内存总量
    private static volatile long reservedMemory;
    
    // 最大直接内存限制
    private static volatile long maxMemory;
    
    // 预留内存
    static void reserveMemory(long size, int cap) {
        long totalCap;
        while (cap <= maxMemory - (totalCap = totalCapacity.get())) {
            if (totalCapacity.compareAndSet(totalCap, totalCap + cap)) {
                reservedMemory += size;
                count.incrementAndGet();
                return;
            }
        }
        
        // 超过限制，尝试GC
        System.gc();
        
        try {
            Thread.sleep(100);
        } catch (InterruptedException x) {
            Thread.currentThread().interrupt();
        }
        
        // 再次尝试
        synchronized (Bits.class) {
            if (reservedMemory + size > maxMemory) {
                throw new OutOfMemoryError("Direct buffer memory");
            }
            reservedMemory += size;
        }
    }
    
    // 释放内存
    static void unreserveMemory(long size, int cap) {
        synchronized (Bits.class) {
            reservedMemory -= size;
            totalCapacity.addAndGet(-cap);
            count.decrementAndGet();
        }
    }
}
```

**超限处理流程**：

```
分配直接内存
    ↓
检查：reservedMemory + size > maxMemory ?
    ↓
    ┌──────┴──────┐
    ↓             ↓
   否            是
    ↓             ↓
分配成功      触发System.gc()
              ↓
          等待100ms
              ↓
          再次检查
              ↓
          ┌──────┴──────┐
          ↓             ↓
        成功          失败
          ↓             ↓
      分配成功    抛出OOM异常
```

```java
public class DirectMemoryStats {
    public static void printDirectMemoryStats() throws Exception {
        Class<?> bitsClass = Class.forName("java.nio.Bits");
        
        // 获取maxMemory字段
        Field maxMemoryField = bitsClass.getDeclaredField("maxMemory");
        maxMemoryField.setAccessible(true);

    }
}
```

## 性能对比与优化

### 问题9：堆内存 vs 直接内存，性能差异有多大？

**性能测试代码**：

```java
import java.nio.ByteBuffer;

public class BufferPerformanceTest {
    private static final int ITERATIONS = 1000000;
    private static final int BUFFER_SIZE = 1024;
    
    public static void main(String[] args) {
        testHeapBuffer();
        testDirectBuffer();
    }
    
    // 测试堆内存
    private static void testHeapBuffer() {
        long start = System.nanoTime();
        
        for (int i = 0; i < ITERATIONS; i++) {
            ByteBuffer buffer = ByteBuffer.allocate(BUFFER_SIZE);
            for (int j = 0; j < BUFFER_SIZE; j++) {
                buffer.put((byte) j);
            }
            buffer.flip();
            for (int j = 0; j < BUFFER_SIZE; j++) {
                buffer.get();
            }
        }
        
        long end = System.nanoTime();
        System.out.println("Heap Buffer: " + (end - start) / 1000000 + " ms");
    }
    
    // 测试直接内存
    private static void testDirectBuffer() {
        long start = System.nanoTime();
        
        for (int i = 0; i < ITERATIONS; i++) {
            ByteBuffer buffer = ByteBuffer.allocateDirect(BUFFER_SIZE);
            for (int j = 0; j < BUFFER_SIZE; j++) {
                buffer.put((byte) j);
            }
            buffer.flip();
            for (int j = 0; j < BUFFER_SIZE; j++) {
                buffer.get();
            }
        }
        
        long end = System.nanoTime();
        System.out.println("Direct Buffer: " + (end - start) / 1000000 + " ms");
    }
}

// 输出示例（仅内存操作）：
// Heap Buffer: 1200 ms
// Direct Buffer: 1500 ms  ← 分配/释放成本高

// 输出示例（包含IO操作）：
// Heap Buffer: 5000 ms  ← 需要复制
// Direct Buffer: 2000 ms  ← 零拷贝，性能更好
```

**性能对比总结**：

| 场景 | 堆内存 | 直接内存 | 推荐 |
|------|--------|---------|------|
| **分配/释放** | 快（纳秒级） | 慢（微秒级） | 堆内存 |
| **读写操作** | 快（有JIT优化） | 快（直接访问） | 相近 |
| **IO操作** | 慢（需要复制） | 快（零拷贝） | **直接内存** |
| **GC影响** | 受GC影响 | 不受GC影响 | 直接内存 |
| **内存占用** | 有对象头 | 无对象头 | 直接内存 |
| **适用场景** | 频繁创建的小对象 | 大块、长期存活的数据 | - |

### 问题10：如何优化直接内存的使用？

**优化策略1：池化复用**

```java
import java.nio.ByteBuffer;
import java.util.concurrent.ConcurrentLinkedQueue;

public class DirectBufferPool {
    private final int bufferSize;
    private final int maxPoolSize;
    private final ConcurrentLinkedQueue<ByteBuffer> pool;
    
    public DirectBufferPool(int bufferSize, int maxPoolSize) {
        this.bufferSize = bufferSize;
        this.maxPoolSize = maxPoolSize;
        this.pool = new ConcurrentLinkedQueue<>();
    }
    
    // 获取Buffer
    public ByteBuffer acquire() {
        ByteBuffer buffer = pool.poll();
        if (buffer == null) {
            // 池中没有，创建新的
            buffer = ByteBuffer.allocateDirect(bufferSize);
        } else {
            // 重置Buffer
            buffer.clear();
        }
        return buffer;
    }
    
    // 归还Buffer
    public void release(ByteBuffer buffer) {
        if (buffer == null || !buffer.isDirect()) {
            return;
        }
        
        if (pool.size() < maxPoolSize) {
            buffer.clear();
            pool.offer(buffer);
        } else {
            // 池已满，让GC回收
            // DirectByteBuffer会通过Cleaner自动释放
        }
    }
    
    // 清空池
    public void clear() {
        pool.clear();
    }
}

// 使用示例
DirectBufferPool pool = new DirectBufferPool(1024, 100);

// 获取Buffer
ByteBuffer buffer = pool.acquire();
try {
    // 使用buffer...
} finally {
    // 归还Buffer
    pool.release(buffer);
}
```

**优化策略2：延迟分配**

```java
public class LazyDirectBuffer {
    private ByteBuffer buffer;
    private final int capacity;
    
    public LazyDirectBuffer(int capacity) {
        this.capacity = capacity;
    }
    
    // 延迟分配
    public ByteBuffer getBuffer() {
        if (buffer == null) {
            synchronized (this) {
                if (buffer == null) {
                    buffer = ByteBuffer.allocateDirect(capacity);
                }
            }
        }
        return buffer;
    }
}
```

**优化策略3：批量操作**

```java
// ❌ 不好：频繁的小IO操作

    ByteBuffer buffer = ByteBuffer.allocateDirect(1024);
    channel.write(buffer);
}

// ✅ 好：批量操作
```

## 内存泄漏与排查

### 问题11：直接内存会发生泄漏吗？

**会！常见泄漏场景**：

**场景1：忘记释放**

```java
// ❌ 泄漏代码
public void processData() {
    ByteBuffer buffer = ByteBuffer.allocateDirect(1024 * 1024);  // 1MB
    // 使用buffer...
    // 忘记释放，且buffer对象一直被引用
}

// ✅ 正确做法
public void processData() {
    ByteBuffer buffer = ByteBuffer.allocateDirect(1024 * 1024);
    try {
        // 使用buffer...
    } finally {
        // 手动释放
        cleanDirectBuffer(buffer);
    }
}

private void cleanDirectBuffer(ByteBuffer buffer) {
    if (buffer.isDirect()) {
        ((sun.nio.ch.DirectBuffer) buffer).cleaner().clean();
    }
}
```

**场景2：循环引用**

```java
// ❌ 泄漏代码
public class BufferHolder {
    private static List<ByteBuffer> buffers = new ArrayList<>();
    
    public void addBuffer() {
        ByteBuffer buffer = ByteBuffer.allocateDirect(1024 * 1024);
        buffers.add(buffer);  // 一直持有引用，无法被GC
    }
}

// ✅ 正确做法
public class BufferHolder {
    private static List<WeakReference<ByteBuffer>> buffers = new ArrayList<>();
    
    public void addBuffer() {
        ByteBuffer buffer = ByteBuffer.allocateDirect(1024 * 1024);
        buffers.add(new WeakReference<>(buffer));  // 使用弱引用
    }
}
```

**场景3：异常导致未释放**

```java
// ❌ 泄漏代码
public void processFile(String path) throws IOException {
    FileChannel channel = new FileInputStream(path).getChannel();
    ByteBuffer buffer = ByteBuffer.allocateDirect(1024 * 1024);
    
    channel.read(buffer);  // 可能抛异常
    // 异常时buffer未释放
    
    cleanDirectBuffer(buffer);
}

// ✅ 正确做法
public void processFile(String path) throws IOException {
    FileChannel channel = null;
    ByteBuffer buffer = null;
    
    try {
        channel = new FileInputStream(path).getChannel();
        buffer = ByteBuffer.allocateDirect(1024 * 1024);
        channel.read(buffer);
    } finally {
        if (buffer != null) {
            cleanDirectBuffer(buffer);
        }
        if (channel != null) {
            channel.close();
        }
    }
}
```

```java
import java.nio.ByteBuffer;
import java.util.concurrent.ConcurrentHashMap;

public class OffHeapCache {
    private final ConcurrentHashMap<String, ByteBuffer> cache;
    private final int valueSize;
    
    public OffHeapCache(int valueSize) {
        this.cache = new ConcurrentHashMap<>();
        this.valueSize = valueSize;
    }
    
    // 存储数据
    public void put(String key, byte[] value) {
        if (value.length > valueSize) {
            throw new IllegalArgumentException("Value too large");
        }
        
        // 使用直接内存存储，不占用堆空间
        ByteBuffer buffer = ByteBuffer.allocateDirect(valueSize);
        buffer.put(value);
        buffer.flip();
        
        ByteBuffer old = cache.put(key, buffer);
        if (old != null) {
            // 释放旧的直接内存
            cleanDirectBuffer(old);
        }
    }
    
    // 获取数据
    public byte[] get(String key) {
        ByteBuffer buffer = cache.get(key);
        if (buffer == null) {
            return null;
        }
        
        byte[] value = new byte[buffer.remaining()];
        buffer.get(value);
        buffer.rewind();  // 重置position
        return value;
    }
    
    // 删除数据
    public void remove(String key) {
        ByteBuffer buffer = cache.remove(key);
        if (buffer != null) {
            cleanDirectBuffer(buffer);
        }
    }
    
    // 清空缓存
    public void clear() {
        cache.values().forEach(this::cleanDirectBuffer);
        cache.clear();
    }
    
    private void cleanDirectBuffer(ByteBuffer buffer) {
        if (buffer.isDirect()) {
            ((sun.nio.ch.DirectBuffer) buffer).cleaner().clean();
        }
    }
}
```

## 总结

### 核心要点

```
┌─────────────────────────────────────────────────┐
│           直接内存 vs 堆内存                     │
├─────────────────────────────────────────────────┤
│  直接内存（Direct Memory）：                     │
│  ✅ 适合大块、长期存活的数据                     │
│  ✅ 适合IO密集型操作（零拷贝）                   │
│  ✅ 不受GC影响                                   │
│  ✅ 不占用堆空间                                 │
│  ❌ 分配/释放成本高                              │
│  ❌ 需要手动管理（容易泄漏）                     │
├─────────────────────────────────────────────────┤
│  堆内存（Heap Memory）：                         │
│  ✅ 适合频繁创建的小对象                         │
│  ✅ 分配/释放速度快                              │
│  ✅ 自动GC管理                                   │
│  ❌ IO操作需要复制                               │
│  ❌ 受GC影响                                     │
│  ❌ 占用堆空间                                   │
└─────────────────────────────────────────────────┘
```

### 最佳实践

1. **选择合适的内存类型**
   - IO密集型：使用直接内存
   - 计算密集型：使用堆内存
   - 混合场景：根据数据大小和生命周期选择

2. **池化复用**
   - 避免频繁分配/释放直接内存
   - 使用对象池管理DirectByteBuffer

3. **及时释放**
   - 使用try-finally确保释放
   - 考虑使用WeakReference

4. **监控告警**
   - 监控直接内存使用率
   - 设置合理的MaxDirectMemorySize
   - 建立告警机制

5. **避免泄漏**
   - 注意异常处理
   - 避免循环引用
   - 定期检查内存使用
