# 如何检测当前使用的垃圾收集器

## 📚 概述

在生产环境中，我们经常需要确认Java应用使用的是哪种垃圾收集器。本文介绍多种检测方法，从简单到高级，从命令行到编程方式。

## 🎯 核心问题

- ❓ 如何快速查看当前使用的GC？
- ❓ 如何判断GC是默认的还是手动指定的？
- ❓ 如何在代码中检测GC类型？
- ❓ 如何通过GC日志判断GC类型？

---

## 一、命令行工具检测

### 1.1 使用 jinfo 命令（推荐）

**jinfo** 是最直接的检测工具，可以查看JVM参数。

#### 基本用法

```bash
# 1. 查找Java进程ID
jps -l

# 2. 检查特定GC是否启用
jinfo -flag UseG1GC <pid>
jinfo -flag UseParallelGC <pid>
jinfo -flag UseConcMarkSweepGC <pid>
jinfo -flag UseSerialGC <pid>
jinfo -flag UseZGC <pid>

# 3. 查看所有JVM参数
jinfo -flags <pid>
```

#### 实战示例

```bash
# 查找Java进程
$ jps -l
12345 /data/apps/seckill-service.jar
23456 org.apache.catalina.startup.Bootstrap

# 检查是否使用G1 GC
$ jinfo -flag UseG1GC 12345
-XX:+UseG1GC

# 检查是否使用Parallel GC
$ jinfo -flag UseParallelGC 12345
-XX:-UseParallelGC

# 查看所有GC相关参数
$ jinfo -flags 12345 | grep -i gc
-XX:+UseG1GC
-XX:MaxGCPauseMillis=100
-XX:G1HeapRegionSize=16777216
-XX:InitiatingHeapOccupancyPercent=40
```

#### 参数说明

| 参数 | 含义 |
|------|------|
| `-XX:+UseG1GC` | 启用G1 GC（+表示启用） |
| `-XX:-UseParallelGC` | 禁用Parallel GC（-表示禁用） |

### 1.2 使用 jcmd 命令

**jcmd** 是JDK 7+提供的多功能诊断工具。

```bash
# 查看所有JVM参数

# 查看命令行参数
jcmd <pid> VM.command_line

# 查看系统属性
jcmd <pid> VM.system_properties

# 只看GC相关参数
jcmd <pid> VM.flags | grep -i gc
```

### 1.3 使用 jstat 命令

**jstat** 主要用于查看GC统计信息，也可以推断GC类型。

```bash
# 查看GC统计
jstat -gc <pid> 1000

# 查看GC原因
jstat -gccause <pid>

# 查看GC容量
jstat -gccapacity <pid>
```

#### 实战示例

```bash
$ jstat -gc 12345
 S0C    S1C    S0U    S1U      EC       EU        OC         OU       MC     MU    CCSC   CCSU   YGC     YGCT    FGC    FGCT     GCT   
 0.0   512.0   0.0   512.0  524288.0 102400.0 11534336.0  5767168.0  51200.0 48640.0 6400.0 5760.0    156    2.340     0    0.000    2.340

# 从输出可以看出：
# - S0C/S1C: Survivor区容量
# - EC: Eden区容量
# - OC: Old区容量
# - YGC: Young GC次数
# - FGC: Full GC次数
```

### 1.4 使用 ps 命令

**ps** 命令可以查看进程的完整命令行。

```bash
# 查看Java进程的完整命令行
ps -ef | grep java

# 或者
ps aux | grep java

# 只看GC参数
ps -ef | grep java | grep -o '\-XX:[^ ]*GC[^ ]*'
```

#### 实战示例

```bash
$ ps -ef | grep java
root     12345     1  0 10:00 ?        00:05:23 java -Xms12g -Xmx12g -XX:+UseG1GC -XX:MaxGCPauseMillis=100 -jar /data/apps/seckill-service.jar

$ ps -ef | grep java | grep -o '\-XX:[^ ]*GC[^ ]*'
-XX:+UseG1GC
-XX:MaxGCPauseMillis=100
```

## 二、通过GC日志检测

### 2.1 GC日志关键字

不同的垃圾收集器在GC日志中有不同的标识。

| GC类型 | 日志关键字 | 示例 |
|--------|-----------|------|
| **Serial GC** | `DefNew`, `Tenured` | `[GC (Allocation Failure) [DefNew: 8192K->1024K(9216K), 0.0012345 secs]` |
| **Parallel GC** | `PSYoungGen`, `ParOldGen` | `[GC (Allocation Failure) [PSYoungGen: 8192K->1024K(9216K)]` |
| **ParNew + CMS** | `ParNew`, `CMS` | `[GC (Allocation Failure) [ParNew: 8192K->1024K(9216K)]` |
| **CMS** | `CMS Initial Mark`, `concurrent mark` | `[GC (CMS Initial Mark) [1 CMS-initial-mark: 10240K(20480K)]` |
| **G1 GC** | `G1 Evacuation Pause`, `G1 Young` | `[GC pause (G1 Evacuation Pause) (young) 256M->45M(1024M)]` |
| **ZGC** | `ZGC`, `Pause Mark Start` | `[gc,start] GC(0) Garbage Collection (Warmup)` |
| **Shenandoah** | `Shenandoah`, `Pause Init Mark` | `[gc] Pause Init Mark 0.123ms` |

### 2.2 查看GC日志

```bash
# 查看GC日志文件
tail -f gc.log

# 搜索GC类型关键字
grep -i "gc pause" gc.log | head -5

# 统计GC类型
grep -o "PSYoungGen\|ParNew\|DefNew\|G1 Evacuation" gc.log | sort | uniq -c
```

## 四、编程方式检测

### 4.1 使用 ManagementFactory

```java
import java.lang.management.ManagementFactory;
import java.lang.management.RuntimeMXBean;
import java.util.List;

public class GCDetector {
    
    public static void main(String[] args) {
        String gcType = detectGarbageCollector();
        System.out.println("当前使用的垃圾收集器: " + gcType);
        
        // 打印所有JVM参数
        printJVMArguments();
    }
    
    /**
     * 检测当前使用的垃圾收集器
     */
    public static String detectGarbageCollector() {
        RuntimeMXBean runtimeMxBean = ManagementFactory.getRuntimeMXBean();
        List<String> arguments = runtimeMxBean.getInputArguments();
        
        // 检查显式指定的GC
        for (String arg : arguments) {
            if (arg.contains("-XX:+UseG1GC")) {
                return "G1 GC";
            } else if (arg.contains("-XX:+UseParallelGC")) {
                return "Parallel GC";
            } else if (arg.contains("-XX:+UseParallelOldGC")) {
                return "Parallel Old GC";
            } else if (arg.contains("-XX:+UseConcMarkSweepGC")) {
                return "CMS GC";
            } else if (arg.contains("-XX:+UseSerialGC")) {
                return "Serial GC";
            } else if (arg.contains("-XX:+UseZGC")) {
                return "ZGC";
            } else if (arg.contains("-XX:+UseShenandoahGC")) {
                return "Shenandoah GC";
            }
        }
        
        // 如果没有显式指定，返回默认GC
        return getDefaultGC();
    }
    
    /**
     * 获取默认的垃圾收集器（根据JDK版本）
     */
    private static String getDefaultGC() {
        String javaVersion = System.getProperty("java.version");
        
        if (javaVersion.startsWith("1.8") || javaVersion.startsWith("8")) {
            return "Parallel GC (JDK 8 默认)";
        } else if (javaVersion.startsWith("9") || javaVersion.startsWith("10") || 
                   javaVersion.startsWith("11") || javaVersion.startsWith("12") ||
                   javaVersion.startsWith("13") || javaVersion.startsWith("14")) {
            return "G1 GC (JDK 9+ 默认)";
        } else {
            return "G1 GC (默认)";
        }
    }
    
    /**
     * 打印所有JVM参数
     */
    public static void printJVMArguments() {
        RuntimeMXBean runtimeMxBean = ManagementFactory.getRuntimeMXBean();
        List<String> arguments = runtimeMxBean.getInputArguments();
        
        System.out.println("\n所有JVM参数:");
        System.out.println("=".repeat(50));
        for (String arg : arguments) {
            System.out.println(arg);
        }
    }
}
```

### 4.2 使用 GarbageCollectorMXBean

```java
import java.lang.management.GarbageCollectorMXBean;
import java.lang.management.ManagementFactory;
import java.util.List;

public class GCInfoDetector {
    
    public static void main(String[] args) {
        detectGCByMXBean();
    }
    
    /**
     * 通过 GarbageCollectorMXBean 检测GC
     */
    public static void detectGCByMXBean() {
        List<GarbageCollectorMXBean> gcBeans = ManagementFactory.getGarbageCollectorMXBeans();
        
        System.out.println("垃圾收集器信息:");
        System.out.println("=".repeat(50));
        
        for (GarbageCollectorMXBean gcBean : gcBeans) {
            String name = gcBean.getName();
            long count = gcBean.getCollectionCount();
            long time = gcBean.getCollectionTime();
            
            System.out.println("名称: " + name);
            System.out.println("GC次数: " + count);
            System.out.println("GC总耗时: " + time + "ms");
            System.out.println("内存池: " + String.join(", ", gcBean.getMemoryPoolNames()));
            System.out.println("-".repeat(50));
        }
        
        // 根据GC名称判断GC类型
        String gcType = identifyGCType(gcBeans);
        System.out.println("\n当前使用的垃圾收集器: " + gcType);
    }
    
    /**
     * 根据GC名称识别GC类型
     */
    private static String identifyGCType(List<GarbageCollectorMXBean> gcBeans) {
        for (GarbageCollectorMXBean gcBean : gcBeans) {
            String name = gcBean.getName();
            
            if (name.contains("G1")) {
                return "G1 GC";
            } else if (name.contains("PS") || name.contains("Parallel")) {
                return "Parallel GC";
            } else if (name.contains("ConcurrentMarkSweep") || name.contains("CMS")) {
                return "CMS GC";
            } else if (name.contains("Copy") || name.contains("MarkSweepCompact")) {
                return "Serial GC";
            } else if (name.contains("ZGC")) {
                return "ZGC";
            } else if (name.contains("Shenandoah")) {
                return "Shenandoah GC";
            }
        }
        return "Unknown GC";
    }
}
```

### 4.3 GC名称对照表

| GC类型 | Young GC名称 | Old GC名称 |
|--------|-------------|-----------|
| **Serial GC** | Copy | MarkSweepCompact |
| **Parallel GC** | PS Scavenge | PS MarkSweep |
| **ParNew + CMS** | ParNew | ConcurrentMarkSweep |
| **G1 GC** | G1 Young Generation | G1 Old Generation |
| **ZGC** | ZGC | ZGC |
| **Shenandoah** | Shenandoah Cycles | Shenandoah Cycles |

### 4.4 完整的检测工具类

```java
import java.lang.management.*;
import java.util.List;

/**
 * 垃圾收集器检测工具类
 */
public class GarbageCollectorDetector {
    
    /**
     * 获取当前使用的垃圾收集器信息
     */
    public static GCInfo getGCInfo() {
        GCInfo gcInfo = new GCInfo();
        
        // 1. 从JVM参数检测
        RuntimeMXBean runtimeMxBean = ManagementFactory.getRuntimeMXBean();
        List<String> arguments = runtimeMxBean.getInputArguments();
        gcInfo.setJvmArguments(arguments);
        
        // 2. 从GarbageCollectorMXBean检测
        List<GarbageCollectorMXBean> gcBeans = ManagementFactory.getGarbageCollectorMXBeans();
        gcInfo.setGcBeans(gcBeans);
        
        // 3. 识别GC类型
        String gcType = identifyGCType(arguments, gcBeans);
        gcInfo.setGcType(gcType);
        
        // 4. 获取GC统计信息
        gcInfo.setGcStatistics(getGCStatistics(gcBeans));
        
        return gcInfo;
    }
    
    /**
     * 识别GC类型
     */
    private static String identifyGCType(List<String> arguments, List<GarbageCollectorMXBean> gcBeans) {
        // 先从JVM参数识别
        for (String arg : arguments) {
            if (arg.contains("-XX:+UseG1GC")) return "G1 GC";
            if (arg.contains("-XX:+UseParallelGC")) return "Parallel GC";
            if (arg.contains("-XX:+UseConcMarkSweepGC")) return "CMS GC";
            if (arg.contains("-XX:+UseSerialGC")) return "Serial GC";
            if (arg.contains("-XX:+UseZGC")) return "ZGC";
            if (arg.contains("-XX:+UseShenandoahGC")) return "Shenandoah GC";
        }
        
        // 再从GC名称识别
        for (GarbageCollectorMXBean gcBean : gcBeans) {
            String name = gcBean.getName();
            if (name.contains("G1")) return "G1 GC";
            if (name.contains("PS") || name.contains("Parallel")) return "Parallel GC";
            if (name.contains("ConcurrentMarkSweep")) return "CMS GC";
            if (name.contains("Copy") || name.contains("MarkSweepCompact")) return "Serial GC";
            if (name.contains("ZGC")) return "ZGC";
            if (name.contains("Shenandoah")) return "Shenandoah GC";
        }
        
        // 返回默认GC
        return getDefaultGC();
    }
    
    /**
     * 获取默认GC
     */
    private static String getDefaultGC() {
        String javaVersion = System.getProperty("java.version");
        if (javaVersion.startsWith("1.8") || javaVersion.startsWith("8")) {
            return "Parallel GC (默认)";
        } else {
            return "G1 GC (默认)";
        }
    }
    
    /**
     * 获取GC统计信息
     */
    private static GCStatistics getGCStatistics(List<GarbageCollectorMXBean> gcBeans) {
        GCStatistics stats = new GCStatistics();
        
        for (GarbageCollectorMXBean gcBean : gcBeans) {
            String name = gcBean.getName();
            long count = gcBean.getCollectionCount();
            long time = gcBean.getCollectionTime();
            
            if (isYoungGC(name)) {
                stats.setYoungGCCount(count);
                stats.setYoungGCTime(time);
            } else {
                stats.setOldGCCount(count);
                stats.setOldGCTime(time);
            }
        }
        
        return stats;
    }
    
    /**
     * 判断是否为Young GC
     */
    private static boolean isYoungGC(String name) {
        return name.contains("Young") || 
               name.contains("Scavenge") || 
               name.contains("ParNew") || 
               name.contains("Copy");
    }
    
    /**
     * GC信息类
     */
    public static class GCInfo {
        private String gcType;
        private List<String> jvmArguments;
        private List<GarbageCollectorMXBean> gcBeans;
        private GCStatistics gcStatistics;
        
        // Getters and Setters
        public String getGcType() { return gcType; }
        public void setGcType(String gcType) { this.gcType = gcType; }
        
        public List<String> getJvmArguments() { return jvmArguments; }
        public void setJvmArguments(List<String> jvmArguments) { this.jvmArguments = jvmArguments; }
        
        public List<GarbageCollectorMXBean> getGcBeans() { return gcBeans; }
        public void setGcBeans(List<GarbageCollectorMXBean> gcBeans) { this.gcBeans = gcBeans; }
        
        public GCStatistics getGcStatistics() { return gcStatistics; }
        public void setGcStatistics(GCStatistics gcStatistics) { this.gcStatistics = gcStatistics; }
        
        @Override
        public String toString() {
            StringBuilder sb = new StringBuilder();
            sb.append("垃圾收集器类型: ").append(gcType).append("\n");
            sb.append("\nGC统计信息:\n");
            sb.append(gcStatistics);
            sb.append("\nGC详细信息:\n");
            for (GarbageCollectorMXBean gcBean : gcBeans) {
                sb.append("  - ").append(gcBean.getName())
                  .append(": GC次数=").append(gcBean.getCollectionCount())
                  .append(", 总耗时=").append(gcBean.getCollectionTime()).append("ms\n");
            }
            return sb.toString();
        }
    }
    
    /**
     * GC统计信息类
     */
    public static class GCStatistics {
        private long youngGCCount;
        private long youngGCTime;
        private long oldGCCount;
        private long oldGCTime;
        
        // Getters and Setters
        public long getYoungGCCount() { return youngGCCount; }
        public void setYoungGCCount(long youngGCCount) { this.youngGCCount = youngGCCount; }
        
        public long getYoungGCTime() { return youngGCTime; }
        public void setYoungGCTime(long youngGCTime) { this.youngGCTime = youngGCTime; }
        
        public long getOldGCCount() { return oldGCCount; }
        public void setOldGCCount(long oldGCCount) { this.oldGCCount = oldGCCount; }
        
        public long getOldGCTime() { return oldGCTime; }
        public void setOldGCTime(long oldGCTime) { this.oldGCTime = oldGCTime; }
        
        public long getTotalGCCount() { return youngGCCount + oldGCCount; }
        public long getTotalGCTime() { return youngGCTime + oldGCTime; }
        
        @Override
        public String toString() {
            return String.format(
                "  Young GC: 次数=%d, 总耗时=%dms, 平均耗时=%.2fms\n" +
                "  Old GC: 次数=%d, 总耗时=%dms, 平均耗时=%.2fms\n" +
                "  Total GC: 次数=%d, 总耗时=%dms",
                youngGCCount, youngGCTime, youngGCCount > 0 ? (double)youngGCTime/youngGCCount : 0,
                oldGCCount, oldGCTime, oldGCCount > 0 ? (double)oldGCTime/oldGCCount : 0,
                getTotalGCCount(), getTotalGCTime()
            );
        }
    }
    
    /**
     * 测试方法
     */
    public static void main(String[] args) {
        GCInfo gcInfo = getGCInfo();
        System.out.println(gcInfo);
    }
}
```

## 五、Shell脚本自动检测

### 5.1 快速检测脚本

```bash
#!/bin/bash
# gc-detector.sh - 快速检测Java进程使用的GC

# 颜色定义
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

# 检查参数
if [ -z "$1" ]; then
    echo -e "${YELLOW}用法: $0 <pid>${NC}"
    echo ""
    echo "可用的Java进程:"
    jps -l
    exit 1
fi

PID=$1

# 检查进程是否存在
if ! ps -p $PID > /dev/null 2>&1; then
    echo -e "${RED}错误: 进程 $PID 不存在${NC}"
    exit 1
fi

echo "========================================="
echo "检测进程 $PID 的垃圾收集器"
echo "========================================="

# 获取进程信息
PROCESS_INFO=$(jps -l | grep "^$PID")
echo -e "${GREEN}进程信息:${NC} $PROCESS_INFO"
echo ""

# 方法1: 使用 jinfo 检测
echo -e "${GREEN}方法1: jinfo 检测${NC}"
echo "-----------------------------------------"

declare -A gc_flags=(
    ["UseG1GC"]="G1 GC"
    ["UseParallelGC"]="Parallel GC"
    ["UseParallelOldGC"]="Parallel Old GC"
    ["UseConcMarkSweepGC"]="CMS GC"
    ["UseSerialGC"]="Serial GC"
    ["UseZGC"]="ZGC"
    ["UseShenandoahGC"]="Shenandoah GC"
)

DETECTED_GC=""
for flag in "${!gc_flags[@]}"; do
    result=$(jinfo -flag $flag $PID 2>/dev/null)
    if [[ $result == *"+$flag"* ]]; then
        DETECTED_GC="${gc_flags[$flag]}"
        echo -e "${GREEN}✓${NC} $result"
    else
        echo "  $result"
    fi
done

if [ -n "$DETECTED_GC" ]; then
    echo -e "\n${GREEN}检测结果: $DETECTED_GC${NC}"
else
    echo -e "\n${YELLOW}未检测到显式指定的GC，使用默认GC${NC}"
fi

echo ""

# 方法2: 使用 jcmd 检测
echo -e "${GREEN}方法2: jcmd 检测${NC}"
echo "-----------------------------------------"
jcmd $PID VM.flags 2>/dev/null | grep -i "use.*gc"
echo ""

# 方法3: 查看GC相关参数
echo -e "${GREEN}方法3: 所有GC相关参数${NC}"
echo "-----------------------------------------"
jinfo -flags $PID 2>/dev/null | grep -i gc
echo ""

# 方法4: 使用 GarbageCollectorMXBean
echo -e "${GREEN}方法4: GC MXBean 信息${NC}"
echo "-----------------------------------------"

# 创建临时Java程序
TEMP_DIR=$(mktemp -d)
cat > $TEMP_DIR/GCCheck.java << 'EOF'
import java.lang.management.*;
import javax.management.*;
import java.util.*;

public class GCCheck {
    public static void main(String[] args) throws Exception {
        if (args.length == 0) {
            System.out.println("Usage: java GCCheck <pid>");
            return;
        }
        
        String pid = args[0];
        String url = "service:jmx:rmi:///jndi/rmi://localhost:" + pid + "/jmxrmi";
        
        try {
            List<GarbageCollectorMXBean> gcBeans = ManagementFactory.getGarbageCollectorMXBeans();
            for (GarbageCollectorMXBean gcBean : gcBeans) {
                System.out.println("GC名称: " + gcBean.getName());
                System.out.println("  GC次数: " + gcBean.getCollectionCount());
                System.out.println("  GC总耗时: " + gcBean.getCollectionTime() + "ms");
            }
        } catch (Exception e) {
            // Ignore
        }
    }
}
EOF

# 编译并运行（如果可能）
# javac $TEMP_DIR/GCCheck.java 2>/dev/null
# java -cp $TEMP_DIR GCCheck $PID 2>/dev/null

# 清理
rm -rf $TEMP_DIR

# 使用 jstat 显示GC统计
echo -e "${GREEN}GC统计信息:${NC}"
jstat -gc $PID 2>/dev/null | head -2

echo ""
echo "========================================="
echo -e "${GREEN}检测完成!${NC}"
echo "========================================="
```

### 5.2 批量检测脚本

```bash
#!/bin/bash
# gc-detector-all.sh - 批量检测所有Java进程的GC

echo "========================================="
echo "检测所有Java进程的垃圾收集器"
echo "========================================="
echo ""

# 获取所有Java进程
PIDS=$(jps -q)

if [ -z "$PIDS" ]; then
    echo "没有找到Java进程"
    exit 0
fi

# 遍历所有进程
for PID in $PIDS; do
    # 获取进程信息
    PROCESS_INFO=$(jps -l | grep "^$PID")
    echo "进程: $PROCESS_INFO"
    
    # 检测GC类型
    GC_TYPE="Unknown"
    
    # 检查各种GC标志
    if jinfo -flag UseG1GC $PID 2>/dev/null | grep -q "+UseG1GC"; then
        GC_TYPE="G1 GC"
    elif jinfo -flag UseParallelGC $PID 2>/dev/null | grep -q "+UseParallelGC"; then
        GC_TYPE="Parallel GC"
    elif jinfo -flag UseConcMarkSweepGC $PID 2>/dev/null | grep -q "+UseConcMarkSweepGC"; then
        GC_TYPE="CMS GC"
    elif jinfo -flag UseSerialGC $PID 2>/dev/null | grep -q "+UseSerialGC"; then
        GC_TYPE="Serial GC"
    elif jinfo -flag UseZGC $PID 2>/dev/null | grep -q "+UseZGC"; then
        GC_TYPE="ZGC"
    else
        GC_TYPE="Default GC"
    fi
    
    echo "  垃圾收集器: $GC_TYPE"
    
    # 显示GC统计
    echo "  GC统计:"
    jstat -gc $PID 2>/dev/null | tail -1 | awk '{printf "    YGC: %s, YGCT: %ss, FGC: %s, FGCT: %ss\n", $13, $14, $15, $16}'
    
    echo "-----------------------------------------"
done

echo ""
echo "检测完成!"
```

## 七、常见问题排查

### 7.1 jinfo命令不可用

**问题**: `jinfo: command not found`

**解决方案**:
```bash
# 确认JDK路径
which java

# 添加JDK bin目录到PATH
export PATH=$JAVA_HOME/bin:$PATH

# 或者使用完整路径
/path/to/jdk/bin/jinfo -flag UseG1GC <pid>
```

### 7.2 权限不足

**问题**: `Unable to open socket file: target process not responding or HotSpot VM not loaded`

**解决方案**:
```bash
# 使用相同用户执行
sudo -u <java-user> jinfo -flag UseG1GC <pid>

# 或者使用root用户
sudo jinfo -flag UseG1GC <pid>
```

### 7.3 进程在容器中

**问题**: Docker容器中的Java进程无法检测

**解决方案**:
```bash
# 进入容器
docker exec -it <container-id> bash

# 在容器内执行检测
jinfo -flag UseG1GC <pid>

# 或者从宿主机执行
docker exec <container-id> jinfo -flag UseG1GC <pid>
```

### 7.4 远程JVM检测

**问题**: 需要检测远程服务器的JVM

**解决方案**:
```bash
# 1. 确保远程JVM开启了JMX
# 启动参数中包含:
-Dcom.sun.management.jmxremote
-Dcom.sun.management.jmxremote.port=9999
-Dcom.sun.management.jmxremote.authenticate=false
-Dcom.sun.management.jmxremote.ssl=false

# 2. 使用 jconsole 连接
jconsole <remote-host>:9999

# 3. 使用 VisualVM 连接
jvisualvm
# 添加远程主机: <remote-host>:9999
```

## 九、总结

### 9.1 快速检测方法

**最简单**: 一行命令
```bash
jinfo -flags <pid> | grep -E "Use.*GC"
```

**最准确**: 查看所有参数
```bash
jcmd <pid> VM.flags | grep -i gc
```

**最直观**: 查看GC日志
```bash
tail -f gc.log | grep -i "gc pause"
```

### 9.2 检测方法对比

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **jinfo** | 简单快速 | 需要JDK工具 | 生产环境快速检测 |
| **jcmd** | 信息全面 | 需要JDK 7+ | 详细诊断 |
| **GC日志** | 最准确 | 需要开启日志 | 问题排查 |
| **可视化工具** | 直观易用 | 性能开销 | 开发环境 |
| **编程方式** | 自动化 | 需要编码 | 监控系统集成 |

### 9.3 推荐检测流程

```
1. 快速检测
   ↓
   jinfo -flag UseG1GC <pid>
   
2. 详细检测
   ↓
   jcmd <pid> VM.flags | grep -i gc
   
3. 验证确认
   ↓
   查看GC日志或使用可视化工具
   
4. 记录归档
   ↓
   保存检测结果到文档
```
