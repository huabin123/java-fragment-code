# 第三章：字节码与字节码增强

## 3.1 字节码是什么？

```
Java 源代码 (.java)
    ↓ javac 编译
字节码 (.class)          ← 平台无关的中间表示
    ↓ JVM 解释/JIT 编译
本地机器码               ← 平台相关，直接执行
```

字节码是一套 JVM 规范定义的**指令集**，每条指令由操作码（1字节）+ 操作数组成。

---

## 3.2 查看字节码

```bash
# 用 javap 反编译查看字节码
javap -c -verbose MyClass.class

# 示例：简单的 add 方法
public static int add(int a, int b) {
    return a + b;
}

# 对应字节码：
0: iload_0    # 将 slot[0]（参数 a）压入操作数栈
1: iload_1    # 将 slot[1]（参数 b）压入操作数栈
2: iadd       # 弹出栈顶两个 int，相加，结果压栈
3: ireturn    # 弹出栈顶 int，作为返回值

# 前缀含义：
# i = int, l = long, f = float, d = double
# a = reference（引用类型）
# b = byte, c = char, s = short
```

## 3.3 字节码增强技术

字节码增强 = 在类加载前/后**修改字节码**，动态改变类的行为：

```
三种时机：
1. 编译期：APT（Annotation Processing Tool）、Lombok
2. 类加载时：Java Agent（Instrumentation API）
3. 运行时：ASM、Javassist、CGLib 动态生成/修改字节码
```

### ASM（最底层，最高效）

```java
// OptimizationDemo.java 演示了 ASM 字节码生成

// 用 ASM 动态创建一个类
ClassWriter cw = new ClassWriter(ClassWriter.COMPUTE_FRAMES);
cw.visit(V11, ACC_PUBLIC, "com/example/DynamicClass", null, "java/lang/Object", null);

// 添加方法：public String greet() { return "Hello, ASM!"; }
MethodVisitor mv = cw.visitMethod(ACC_PUBLIC, "greet", "()Ljava/lang/String;", null, null);
mv.visitCode();
mv.visitLdcInsn("Hello, ASM!");  // 加载字符串常量
mv.visitInsn(ARETURN);           // 返回引用类型
mv.visitMaxs(1, 1);
mv.visitEnd();

byte[] bytes = cw.toByteArray();
// 用自定义 ClassLoader 加载这个字节数组
```

### Javassist（高层 API，操作简单）

```java
// 用 Javassist 修改现有类的方法（添加日志）
ClassPool pool = ClassPool.getDefault();
CtClass cc = pool.get("com.example.UserService");
CtMethod method = cc.getDeclaredMethod("findUser");

// 在方法执行前插入代码
method.insertBefore("System.out.println(\"findUser called with id=\" + $1);");
// 在方法执行后插入代码
method.insertAfter("System.out.println(\"findUser completed\");");

cc.toClass();  // 加载修改后的类
```

## 3.4 Java Agent：无侵入字节码增强

Java Agent 在 JVM 启动时加载，可以在类加载时拦截并修改字节码：

```java
// premain：JVM 启动时加载 Agent
public class MonitorAgent {
    public static void premain(String args, Instrumentation inst) {
        inst.addTransformer((loader, className, classBeingRedefined,
                             protectionDomain, classfileBuffer) -> {
            if (className.startsWith("com/example/")) {
                return addMonitoring(classfileBuffer);  // 修改字节码
            }
            return null;  // 不修改
        });
    }

    private static byte[] addMonitoring(byte[] original) {
        // 用 ASM 修改字节码，添加方法耗时统计
        ClassReader cr = new ClassReader(original);
        ClassWriter cw = new ClassWriter(cr, ClassWriter.COMPUTE_FRAMES);
        cr.accept(new TimingClassVisitor(cw), 0);
        return cw.toByteArray();
    }
}

// MANIFEST.MF 中声明：
// Premain-Class: com.example.MonitorAgent
// Can-Redefine-Classes: true

// 启动时加载：
// java -javaagent:monitor-agent.jar -jar app.jar
```

## 3.5 字节码增强的实际应用

| 框架/工具 | 使用技术 | 用途 |
|---------|---------|-----|
| Spring AOP | CGLib / JDK 动态代理 | 切面增强（事务、日志）|
| Hibernate | CGLib / Javassist | 懒加载代理 |
| Lombok | APT（编译期） | 自动生成 getter/setter/Builder |
| JaCoCo | Java Agent + ASM | 代码覆盖率统计 |
| SkyWalking | Java Agent + ASM | 分布式链路追踪 |
| Arthas | Java Agent + Instrumentation | 线上字节码热更新 |
| Mockito | ByteBuddy | Mock 对象生成 |

## 3.6 本章总结

- **字节码**：平台无关的中间表示，`javap -c` 查看；操作数栈 + 局部变量表是执行基础
- **增强三时机**：编译期（Lombok/APT）、类加载时（Java Agent）、运行时（ASM/Javassist）
- **ASM**：最底层，直接操作字节码指令，性能最好，学习曲线高
- **Javassist**：源码级 API，操作简单，性能稍差
- **Java Agent**：无侵入，在类加载时拦截，APM/链路追踪/覆盖率的标准实现方式
