# 第一章：JVM 内存区域总览

## 1.1 为什么要理解 JVM 内存区域？

当程序出现以下问题时，你必须知道发生在哪块内存：

```
java.lang.OutOfMemoryError: Java heap space         → 堆内存不足
java.lang.OutOfMemoryError: Metaspace               → 方法区（元空间）不足
java.lang.OutOfMemoryError: Direct buffer memory    → 直接内存不足
java.lang.StackOverflowError                        → 虚拟机栈溢出（无限递归）
```

不知道内存分区，就无法定位问题、无法调优 JVM 参数。

## 1.2 JVM 内存的整体划分

```
┌─────────────────────────────────────────────────────────────┐
│                        JVM 内存                              │
│                                                             │
│  ┌───────────────────────────────────┐   ┌───────────────┐ │
│  │           线程共享区域              │   │   线程私有区域  │ │
│  │                                   │   │               │ │
│  │  ┌──────────────┐  ┌──────────┐  │   │ 程序计数器(PC) │ │
│  │  │     堆        │  │  方法区  │  │   │ 虚拟机栈       │ │
│  │  │  (新生代+老年代)│  │ (元空间) │  │   │ 本地方法栈     │ │
│  │  └──────────────┘  └──────────┘  │   └───────────────┘ │
│  └───────────────────────────────────┘                     │
│                                                             │
│  ┌─────────────────────────────────────┐                   │
│  │   直接内存（堆外内存，不受 JVM 管控）  │                   │
│  └─────────────────────────────────────┘                   │
└─────────────────────────────────────────────────────────────┘
```

---

## 1.3 各区域一句话总结

| 内存区域 | 线程归属 | 核心职责 | OOM/SOE |
|---------|---------|---------|---------|
| 程序计数器 | 线程私有 | 记录当前执行的字节码行号 | 无（唯一不会OOM的区域）|
| 虚拟机栈 | 线程私有 | 存放方法的栈帧（局部变量、操作数栈）| StackOverflowError |
| 本地方法栈 | 线程私有 | 执行 native 方法 | StackOverflowError |
| 堆 | 线程共享 | 存储所有对象实例和数组 | OOM: Java heap space |
| 方法区（元空间）| 线程共享 | 存储类信息、常量、静态变量 | OOM: Metaspace |
| 直接内存 | 进程级 | NIO DirectByteBuffer，绕过堆 | OOM: Direct buffer memory |

---

## 1.4 JDK 7 vs JDK 8 的关键变化

```
JDK 7：方法区 = 永久代（PermGen），在 JVM 堆内，-XX:MaxPermSize 控制
         → 容易 OOM（PermGen space），因为大小固定

JDK 8：方法区 = 元空间（Metaspace），在本机内存（堆外），-XX:MaxMetaspaceSize 控制
         → 默认无上限（受物理内存限制），大幅减少 OOM 频率
         → 字符串常量池从方法区移到堆中（JDK 7 已移动）
         → 运行时常量池移到堆中

迁移原因：
1. PermGen 大小难以预估，频繁 OOM
2. Oracle JRockit 没有 PermGen，合并时需要统一
3. 元空间动态扩展，更适合动态生成类的场景（如 CGLib、反射）
```

---

## 1.5 本章总结

- **JVM 内存 = 线程共享（堆 + 方法区）+ 线程私有（PC + 虚拟机栈 + 本地方法栈）+ 直接内存**
- **堆**：最大的一块，所有对象在这里分配
- **方法区**：JDK 8 起是元空间（堆外内存），存类元数据
- **虚拟机栈**：每个方法一个栈帧，递归过深 → StackOverflowError
- **程序计数器**：唯一不会 OOM 的区域，线程切换后恢复执行位置的依据
