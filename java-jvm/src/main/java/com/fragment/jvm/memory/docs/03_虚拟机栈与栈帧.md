# 第三章：虚拟机栈与栈帧

## 3.1 虚拟机栈的本质

每个线程拥有独立的虚拟机栈，**每调用一个方法就压入一个栈帧，方法返回就弹出**：

```java
// MemoryStructureDemo.java → VirtualMachineStackDemo.test()

public static int methodA() {
    int a = 10;           // 局部变量，存在栈帧的局部变量表中
    return methodB(a);    // 调用 methodB，压入新栈帧
}

public static int methodB(int x) {
    int b = x * 2;        // 新栈帧的局部变量
    return b;             // 返回，弹出栈帧
}

// 调用链：main → methodA → methodB
// 栈状态：[main栈帧][methodA栈帧][methodB栈帧]  ← 栈顶
// methodB 返回后：[main栈帧][methodA栈帧]
```

## 3.2 栈帧的内部结构

```
┌──────────────────────────────────────────┐
│                 栈帧                      │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │      局部变量表（Local Variables）  │    │
│  │  slot[0]=this, slot[1]=参数1, ... │    │  存储：方法参数 + 方法内局部变量
│  └──────────────────────────────────┘    │  long/double 占两个 slot
│                                          │
│  ┌──────────────────────────────────┐    │
│  │      操作数栈（Operand Stack）     │    │  字节码指令的"工作台"
│  │  LIFO，用于执行字节码运算          │    │
│  └──────────────────────────────────┘    │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │      动态链接（Dynamic Linking）   │    │  指向方法区中该方法的符号引用
│  └──────────────────────────────────┘    │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │    方法返回地址（Return Address）   │    │  方法正常/异常返回后回到哪里继续执行
│  └──────────────────────────────────┘    │
└──────────────────────────────────────────┘
```

## 3.3 StackOverflowError：栈深度超限

```java
// 无限递归：每次调用都压栈帧，最终栈空间耗尽
public static void infinite() {
    infinite();  // 无出口的递归
}
// 抛出：java.lang.StackOverflowError

// 参数控制：
// -Xss256k   # 每个线程的栈大小（默认 512k ~ 1m，依平台而异）
// 栈越小：支持更多线程，但递归深度更浅
// 栈越大：递归更深，但线程数受限（内存 / 栈大小 = 最大线程数）
```

## 3.4 局部变量表的 slot 复用

```java
// 局部变量表中的 slot 可以被复用（节省空间）
public static void slotReuse() {
    {
        byte[] placeholder = new byte[64 * 1024 * 1024];  // 64MB
        // placeholder 在此块结束后"失效"
    }
    // 但 placeholder 仍可能占用 slot，阻止 GC 回收！
    // 直到该 slot 被新变量覆盖

    int reuse = 1;  // 复用了 placeholder 的 slot → GC 可以回收 placeholder
    System.gc();
}
```

这也是为什么在大内存块使用完后，手动赋 `null` 有时有意义的原因。

---

## 3.5 本章总结

- **虚拟机栈线程私有**：每个线程独立，方法调用 = 压栈帧，返回 = 弹栈帧
- **栈帧四要素**：局部变量表（存变量）、操作数栈（执行运算）、动态链接（符号引用解析）、返回地址
- **局部变量类型**：基本类型存值，引用类型存堆地址；long/double 占两个 slot
- **StackOverflowError**：递归过深或栈帧过大，-Xss 控制每线程栈大小
- **slot 复用**：出了作用域的变量 slot 可复用，但不自动清空，手动赋 null 可辅助 GC
