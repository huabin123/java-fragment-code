# 第四章：方法区与元空间

## 4.1 方法区存储什么？

方法区（JDK 8+ 实现为元空间）存储**类级别**的数据，每个类只存一份：

```
方法区中存储的内容：
┌────────────────────────────────────────────────────────┐
│  类元数据：类名、父类、接口列表、访问修饰符               │
│  字段信息：字段名、类型、修饰符                          │
│  方法信息：方法名、参数、返回类型、字节码                 │
│  静态变量：static 字段的值（JDK 7 起移到堆）             │
│  运行时常量池：字面量、符号引用（JDK 7 起移到堆）         │
│  JIT 编译后的代码缓存                                    │
└────────────────────────────────────────────────────────┘
```

## 4.2 元空间 vs 永久代

```java
// 演示：动态生成大量类会撑爆元空间（OOMDemo.java → demonstrateMetaspaceOOM）

// 使用 CGLib 或 ASM 动态生成类
while (true) {
    Enhancer enhancer = new Enhancer();
    enhancer.setSuperclass(OOMDemo.class);
    enhancer.setUseCache(false);  // 禁用缓存，每次生成新类
    enhancer.setCallback((MethodInterceptor) (obj, method, args, proxy) -> proxy.invokeSuper(obj, args));
    enhancer.create();
    // 不停生成新类 → 元空间占用不断增长 → OOM: Metaspace
}
```

**参数控制**：
```bash
# JDK 8+ 元空间参数
-XX:MetaspaceSize=128m        # 初始元空间大小（触发第一次 GC 的阈值）
-XX:MaxMetaspaceSize=512m     # 最大元空间大小（不设置则无限制）
-XX:MinMetaspaceFreeRatio=40  # GC 后最小空闲比例
-XX:MaxMetaspaceFreeRatio=70  # GC 后最大空闲比例（超出则缩小）

# JDK 7 永久代参数（已废弃）
-XX:PermSize=128m
-XX:MaxPermSize=256m
```

## 4.3 字符串常量池

```java
// 字符串常量池在 JDK 7 起移到堆中
String s1 = "hello";          // 直接用常量池中的对象
String s2 = "hello";          // 同一个对象
String s3 = new String("hello"); // 在堆中新建对象，内容指向常量池

System.out.println(s1 == s2);     // true（同一常量池对象）
System.out.println(s1 == s3);     // false（s3 是堆中新对象）
System.out.println(s1 == s3.intern()); // true（intern() 返回常量池中的引用）

// intern() 的行为：
// 1. 常量池中已有该字符串 → 返回常量池中的引用
// 2. 常量池中没有 → 在常量池中创建并返回

// OOMDemo.java → demonstrateStringPoolOOM()
// 大量调用 intern() 可在 JDK 6 撑爆 PermGen（JDK 7 起常量池在堆，受 -Xmx 限制）
```

## 4.4 运行时常量池

```
编译期：.java → .class 文件中的 Constant Pool（符号引用：类名/方法名/字段名的字符串形式）

运行时：类加载时，Constant Pool 加载到方法区，成为运行时常量池
        符号引用 → 直接引用（真实的内存地址）的解析过程发生在这里

对比：
  Class 文件常量池   → 磁盘，静态
  运行时常量池       → 方法区（元空间），动态（可以在运行时 intern() 新增字符串）
```

---

## 4.5 本章总结

- **方法区**（元空间）：存储类元数据、方法字节码、JIT 代码缓存
- **JDK 7 迁移**：字符串常量池、静态变量、运行时常量池 → 堆
- **JDK 8 迁移**：永久代 → 元空间（本机内存），大幅减少 OOM
- **元空间 OOM**：大量动态生成类（CGLib、反射代理）时出现，用 `-XX:MaxMetaspaceSize` 限制
- **字符串常量池**：`"字面量"` 直接进池；`new String()` 在堆，`.intern()` 入池
