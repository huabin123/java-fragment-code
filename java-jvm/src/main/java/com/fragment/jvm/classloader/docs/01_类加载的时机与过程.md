# 第一章：类加载的时机与过程

## 1.1 类加载做了什么？

Java 程序运行前，`.class` 文件只是磁盘上的二进制文件。**类加载**是把它变成 JVM 内存中可执行的 `Class` 对象的过程：

```
磁盘上的 .class 文件
         ↓ 类加载
JVM 方法区中的 Class 对象（类元数据）
         ↓ 实例化
堆中的对象实例
```

---

## 1.2 类加载的完整生命周期

```
加载（Loading）
    ↓
验证（Verification）  ──┐
    ↓                   │
准备（Preparation）     │  连接阶段（Linking）
    ↓                   │
解析（Resolution）   ──┘
    ↓
初始化（Initialization）
    ↓
使用（Using）
    ↓
卸载（Unloading）
```

---

## 1.3 五个阶段详解

### 加载（Loading）
```
1. 通过类的全限定名找到对应的二进制字节流（.class 文件、JAR、网络、动态生成）
2. 将字节流代表的静态存储结构转化为方法区中的运行时数据结构
3. 在堆中生成一个代表该类的 java.lang.Class 对象（作为方法区数据的入口）
```

### 验证（Verification）
```
确保 .class 文件的字节流符合 JVM 规范，防止恶意代码：
- 文件格式验证（魔数 0xCAFEBABE）
- 元数据验证（父类是否存在、是否继承 final 类）
- 字节码验证（指令的合法性）
- 符号引用验证（引用的类/字段/方法是否存在）
```

### 准备（Preparation）
```java
// 准备阶段：为静态变量分配内存并赋零值（不是初始值！）
// ClassLoadingDemo.java → demonstratePreparationVsInitialization()

public static int value = 123;
// 准备阶段后：value = 0（int 的零值）
// 初始化阶段后：value = 123（执行 static 赋值语句）

public static final int CONSTANT = 456;
// 特例：final static 常量，准备阶段直接赋 456（编译期常量）
```

### 解析（Resolution）
```
将常量池中的符号引用替换为直接引用（内存地址）
符号引用：类名、方法名的字符串形式（如 "java/lang/String"）
直接引用：真实的内存指针

解析可能在初始化前（静态解析）或使用时（动态解析，如虚方法调用）发生
```

### 初始化（Initialization）
```java
// 初始化 = 执行类构造器 <clinit>()
// <clinit> 由编译器自动合并：所有 static 语句块 + static 变量赋值

static int a = 1;
static { a = 2; }  // 按代码顺序执行

// 最终 a = 2

// 多线程安全：JVM 保证 <clinit> 只执行一次（加锁），这是单例模式
// "静态内部类" 实现延迟加载的基础
```

## 1.4 触发类初始化的六种时机（主动引用）

`ClassLoadingDemo.java → demonstrateClassLoadingTiming()` 演示了这六种情况：

```java
// 1. new 实例、访问静态字段、调用静态方法
new MyClass();
MyClass.staticField = 1;
MyClass.staticMethod();

// 2. 反射
Class.forName("com.example.MyClass");

// 3. 初始化子类时，先初始化父类
class Child extends Parent {}
new Child();  // 先初始化 Parent，再初始化 Child

// 4. JVM 启动时，初始化含 main() 的类

// 5. JDK 7：MethodHandle 解析结果为 REF_getStatic 等时

// 6. JDK 8：接口定义了 default 方法时，实现类初始化会触发接口初始化
```

---

## 1.5 不触发初始化的情况（被动引用）

`ClassLoadingDemo.java → demonstrateNoInitialization()` 演示：

```java
// ❌ 通过子类引用父类静态字段，只初始化父类
System.out.println(Child.PARENT_FIELD);  // 只触发 Parent 初始化

// ❌ 数组声明不触发元素类型的初始化
MyClass[] arr = new MyClass[10];  // MyClass 不会被初始化

// ❌ 访问 final static 常量，不触发定义类的初始化（编译期常量内联）
System.out.println(MyClass.FINAL_CONSTANT);  // MyClass 不被初始化
```

## 1.6 本章总结

- **类生命周期**：加载 → 验证 → 准备 → 解析 → 初始化 → 使用 → 卸载
- **准备 vs 初始化**：准备赋零值，初始化才执行 `<clinit>` 赋真实值
- **六种主动引用**触发初始化：new / 访问静态成员 / 反射 / 初始化子类 / main 类 / default 方法
- **被动引用不初始化**：子类引用父类字段、数组声明、final 常量访问
