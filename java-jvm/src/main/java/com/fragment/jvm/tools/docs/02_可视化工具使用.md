# 第二章：JVM 可视化工具使用

命令行工具灵活但不直观，可视化工具提供图表趋势、实时监控和交互分析，适合开发调试和深度排查。

```
工具对比速查：

工具        类型          性能开销    适用场景
JConsole    实时监控      小         快速诊断、JMX 管理
VisualVM    监控+分析     中         开发测试、全面排查
JMC/JFR     事件录制      极小(<1%)  生产环境、深度分析
MAT         离线分析      无         内存泄漏专项分析
```

---

## 2.1 JConsole — JDK 自带基础监控

JDK 自带，基于 JMX，无需额外安装。

```bash
# 启动
jconsole                    # 弹出进程选择界面
jconsole <PID>              # 直接连接本地进程
jconsole <host>:<port>      # 远程连接

# 远程连接：目标 JVM 需开启 JMX
java -Dcom.sun.management.jmxremote \
     -Dcom.sun.management.jmxremote.port=9999 \
     -Dcom.sun.management.jmxremote.ssl=false \
     -Dcom.sun.management.jmxremote.authenticate=false \
     -jar app.jar
```

### 核心页面

```
1. 概览（Overview）
   CPU / 内存 / 线程 / 类加载 四合一趋势图

2. 内存（Memory）
   堆/非堆各区使用趋势 + [Perform GC] 按钮
   判断：锯齿状 = 正常 | 持续上升 = 可能泄漏

3. 线程（Threads）
   线程列表 + 堆栈查看 + [检测死锁] 按钮

4. 类（Classes）
   已加载/已卸载类数量趋势

5. MBean
   查看和操作所有 MBean（JMX 管理）
```

## 2.2 VisualVM — 一体化故障排查

功能最全面的免费工具，支持插件扩展。

```bash
# JDK 8 自带
jvisualvm

# 独立版本（推荐，功能更新）
# 下载：https://visualvm.github.io/
./visualvm

# 推荐插件
Tools → Plugins → Available Plugins
  - VisualVM-MBeans
  - VisualVM-Tracer
  - BTrace Workbench
```

### 核心功能

```
1. Monitor（监控）
   CPU / 堆内存 / Metaspace / 线程数 实时趋势
   [Perform GC] [Heap Dump] 按钮

2. Threads（线程）
   线程时间线（彩色条形图直观展示状态变化）
   绿=RUNNABLE  紫=SLEEPING  黄=WAITING  红=BLOCKED

3. Sampler（采样）— 推荐生产使用
   CPU 采样：热点方法列表 + 执行时间占比
   Memory 采样：对象分配统计
   优势：性能开销小，适合线上

4. Profiler（性能分析）— 仅限开发/测试
   CPU Profiling：精确统计方法时间 + 调用树
   Memory Profiling：精确统计对象分配 + 泄漏检测
   ⚠️ 开销大（10-50%），不适合生产

5. Heap Dump 分析
   对比两个 Heap Dump → 观察对象增长 → 定位泄漏
```

### 实战：CPU 飙高分析

```
1. 连接到应用进程
2. 切换到 Sampler → 点击 CPU 开始采样
3. 运行 30 秒~1 分钟
4. 停止采样 → 查看热点方法（Self Time 排序）
5. 双击方法 → 定位代码 → 优化
```

### 实战：内存泄漏分析

```
1. 连接进程 → 观察堆内存趋势
2. [Heap Dump] 生成快照 A
3. 运行一段时间
4. [Heap Dump] 生成快照 B
5. Compare → 查看实例数增长的类
6. 右键 → Show in Instances View → 查看引用链
```

## 2.3 JMC + JFR — 企业级生产监控

Java Mission Control + Java Flight Recorder，Oracle 官方出品，性能开销极小（<1%），**唯一适合生产环境持续开启的分析工具**。

```bash
# 启动 JFR 录制（JDK 11+ 免费，JDK 8u262+ 也已开源）
java -XX:StartFlightRecording=duration=60s,filename=recording.jfr -jar app.jar

# 运行中启动录制
jcmd <PID> JFR.start duration=60s filename=/tmp/recording.jfr

# 停止录制
jcmd <PID> JFR.stop

# 用 JMC 打开分析
jmc   # 打开 GUI → File → Open → recording.jfr
```

### JFR 记录的事件

```
GC 事件       每次 GC 的时间、类型、回收量
线程事件      线程创建/销毁、锁竞争、park/unpark
类加载事件    加载/卸载的类、耗时
方法采样      热点方法统计（CPU Profiling）
异常事件      抛出的异常类型、位置
IO 事件       文件/网络 IO 耗时
编译事件      JIT 编译、去优化
```

### JMC 自动分析

```
打开 JFR 文件后，JMC 自动分析并给出报告：

  ⚠️ High GC Time (15%)            → GC 耗时占比过高
  ⚠️ Long GC Pauses (max 1.2s)     → 最大暂停时间过长
  ℹ️ Thread Contention detected     → 存在锁竞争
  ✓  No Memory Leak detected       → 未检测到内存泄漏

比手动 jstat/jstack 高效得多，生产环境强烈推荐。
```

## 2.4 MAT — 内存分析专家

Eclipse Memory Analyzer Tool，专注 Heap Dump 分析，处理 GB 级别的 dump 文件。

```
下载：https://eclipse.dev/mat/

打开 .hprof 文件后的核心视图：

1. Leak Suspects Report（泄漏嫌疑报告）
   → 自动分析，列出最可能的泄漏点
   → 80% 的情况第一步看这个就够

2. Dominator Tree（支配树）
   → 按 Retained Heap 降序排列
   → 找出内存占用大户 → 展开引用链

3. Histogram（对象直方图）
   → 按类统计实例数 + Shallow/Retained Heap
   → 哪个类实例最多？占用最大？

4. Path to GC Roots
   → 对可疑对象右键 → exclude weak/soft refs
   → 找到谁在持有它不释放
```

### OQL 查询

```sql
-- 查询所有 String
SELECT * FROM java.lang.String

-- 查询大对象（Retained Heap > 1MB）
SELECT * FROM java.lang.String s WHERE s.@retainedHeapSize > 1048576

-- 统计业务对象数量
SELECT COUNT(*) FROM com.example.User
```

## 2.5 工具选择指南

```
场景                      推荐工具
快速看一眼 JVM 状态        JConsole（最轻量）
开发阶段全面排查           VisualVM（功能全）
生产环境持续监控           JMC + JFR（开销最小）
OOM / 内存泄漏深度分析     MAT（专业）
命令行环境（无 GUI）       jstat + jstack + jcmd
线上不重启诊断             Arthas（见下一章）
```

---

## 2.6 本章总结

- **JConsole**：JDK 自带、JMX 驱动、适合快速诊断
- **VisualVM**：功能全面、插件丰富、开发测试首选
- **JMC/JFR**：开销极小、生产可用、自动分析报告
- **MAT**：Heap Dump 专家、Dominator Tree + OQL 强大
- 可视化工具**不替代**命令行工具，两者互补使用
