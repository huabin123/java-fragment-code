# 第二章：注解的定义与元注解

## 2.1 注解的语法结构

注解用 `@interface` 关键字定义，本质上是一个**特殊的接口**，编译后会实现 `java.lang.annotation.Annotation`：

```java
// AnnotationBasicDemo.java 中定义的注解示例
@Retention(RetentionPolicy.RUNTIME)  // 元注解：声明保留到运行时
@Target(ElementType.FIELD)           // 元注解：只能修饰字段
@Documented                          // 元注解：出现在 Javadoc 中
public @interface Column {
    String name();                   // 必填属性（无默认值）
    boolean nullable() default true; // 可选属性（有默认值）
    int length() default 255;        // 可选属性
}
```

**属性的本质**：注解属性实际上是接口中的抽象方法，返回类型只能是：
- 基本类型（int, boolean, ...）
- String
- Class
- 枚举
- 注解类型
- 上述类型的一维数组

### 特殊属性 `value`

当注解只有一个属性且名称为 `value` 时，使用时可以省略属性名：

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface CacheEvict {
    String value();  // 属性名为 value
}

// 使用时可以简写：
@CacheEvict("users")           // 等价于 @CacheEvict(value = "users")
@CacheEvict(value = "users")  // 完整写法
```

## 2.2 五个元注解详解

元注解是"修饰注解的注解"，定义了注解的行为规则。

### @Retention：注解的生命周期

```java
@Retention(RetentionPolicy.SOURCE)  // 仅存在于源码，编译后丢弃
@Retention(RetentionPolicy.CLASS)   // 存在于 .class 文件，JVM 加载时丢弃（默认）
@Retention(RetentionPolicy.RUNTIME) // 运行时可通过反射读取（最常用）
```

**如果忘记写 @Retention 会怎样？** 默认是 `CLASS`，运行时 `getAnnotation()` 会返回 `null`，是常见的 Bug 来源。

### @Target：注解可以修饰哪些元素

```java
@Target(ElementType.TYPE)           // 类、接口、枚举
@Target(ElementType.FIELD)          // 字段
@Target(ElementType.METHOD)         // 方法
@Target(ElementType.PARAMETER)      // 方法参数
@Target(ElementType.CONSTRUCTOR)    // 构造函数
@Target(ElementType.LOCAL_VARIABLE) // 局部变量
@Target(ElementType.ANNOTATION_TYPE)// 注解类型（元注解就用这个）
@Target(ElementType.PACKAGE)        // 包

// 允许多个目标（数组形式）：
@Target({ElementType.TYPE, ElementType.METHOD})
```

`AnnotationBasicDemo.java` 中每个注解都精确声明了 Target，如 `@Entity` 只能用于 TYPE，`@Column` 只能用于 FIELD——这让编译器在误用时能直接报错。

### @Inherited：注解是否被子类继承

```java
// AnnotationBasicDemo.java → demonstrateAnnotationInheritance()

@Inherited  // 有这个，子类才能继承
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
public @interface InheritedAnnotation { String value(); }

@InheritedAnnotation("parent")
class Parent {}

class Child extends Parent {}  // Child 未直接标注任何注解

// 运行时：
Child.class.isAnnotationPresent(InheritedAnnotation.class);  // true！（继承了）
Child.class.isAnnotationPresent(NonInheritedAnnotation.class); // false（未标 @Inherited）
```

**重要限制**：`@Inherited` 只对类有效，对接口、方法无效。父类方法上的注解不会被子类方法继承（即使父类方法被 override）。

### @Repeatable：可重复注解（JDK 8+）

```java
// AnnotationBasicDemo.java 中的可重复注解定义
@Repeatable(Schedules.class)  // 指定容器注解
public @interface Schedule {
    String day();
    String time() default "00:00";
}

// 容器注解（必须定义）
public @interface Schedules {
    Schedule[] value();
}

// 使用：同一位置多次标注
@Schedule(day = "Monday", time = "09:00")
@Schedule(day = "Wednesday", time = "14:00")
@Schedule(day = "Friday", time = "18:00")
public void backupDatabase() { }

// 读取（两种方式）：
// 方式1：通过容器
Schedules schedules = method.getAnnotation(Schedules.class);

// 方式2：直接读取（推荐，JDK 8+）
Schedule[] scheduleArray = method.getAnnotationsByType(Schedule.class);
```

### @Documented：是否出现在 Javadoc 中

加了 `@Documented` 的注解，在生成 Javadoc 时会显示在被标注元素的文档中。API 级别的注解（如 `@Deprecated`）通常都带这个，内部使用的注解可以不带。

## 2.3 注解属性的默认值陷阱

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
public @interface Table {
    String name();           // 无默认值：使用时必须填写
    String schema() default "public";  // 有默认值：可以省略
}

// 正确：
@Table(name = "t_user")           // schema 使用默认值 "public"
@Table(name = "t_user", schema = "mydb")

// 编译错误：
@Table()  // name 没有默认值，必须填写
```

**注解属性不能为 null**：注解属性没有 null 的概念，如果允许"不传"，只能通过特殊默认值约定（如空字符串 `""` 或 `-1`）来表达。

## 2.4 注解 vs 接口的核心差异

虽然 `@interface` 编译后实现了 `Annotation` 接口，但它和普通接口有根本区别：

| 维度 | 注解 | 接口 |
|------|------|------|
| 继承 | 不能继承其他注解 | 可以继承接口 |
| 属性类型 | 受限（8种）| 无限制 |
| 实例化 | 由 JVM/代理实现 | 由类实现 |
| 用途 | 元数据标记 | 行为契约 |

---

## 2.5 本章总结

- **`@interface` 定义**：属性是无参方法，类型受限，`default` 指定默认值
- **`value` 属性特权**：单属性且名为 `value` 时，使用可省略属性名
- **五个元注解**：`@Retention`（生命周期，忘写默认 CLASS）、`@Target`（适用元素）、`@Inherited`（类继承，非方法）、`@Repeatable`（重复标注）、`@Documented`（Javadoc）
- **@Inherited 陷阱**：只对类注解有效，方法注解不继承
- **属性不能为 null**：用空字符串或约定值代替
