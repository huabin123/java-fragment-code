# 第五章：序列化最佳实践

## 5.1 十条核心实践

### 实践一：始终显式声明 serialVersionUID

```java
// ✅ 任何实现 Serializable 的类都要声明
public class UserDto implements Serializable {
    private static final long serialVersionUID = 1L;
    // ...
}

// IDE 配置：在 IntelliJ IDEA 中开启 "Serializable class without 'serialVersionUID'" 检查
```

### 实践二：用 transient 保护敏感字段

```java
public class UserSession implements Serializable {
    private static final long serialVersionUID = 1L;

    private String sessionId;
    private String userId;

    private transient String password;       // 密码
    private transient String accessToken;    // Token
    private transient Connection connection; // 数据库连接
    private transient Cache localCache;      // 本地缓存
}
```

### 实践三：不可变类用枚举或 readResolve 保护单例

```java
// 枚举的序列化天然安全（JVM 保证枚举单例在反序列化后不会创建新实例）
public enum AppConfig {
    INSTANCE;
    // 枚举单例是序列化安全的最简单方式
}

// 非枚举单例：用 readResolve
public class DatabasePool implements Serializable {
    private static final long serialVersionUID = 1L;
    private static final DatabasePool INSTANCE = new DatabasePool();

    private Object readResolve() { return INSTANCE; }
}
```

### 实践四：优先考虑序列化代理模式

```java
// Effective Java 条目 90：优先考虑序列化代理
public class Period implements Serializable {
    private static final long serialVersionUID = 1L;
    private final Date start;
    private final Date end;

    // 写入时替换为代理
    private Object writeReplace() {
        return new SerializationProxy(this);
    }

    // 防止攻击者直接构造 Period 字节流绕过验证
    private void readObject(ObjectInputStream ois) throws InvalidObjectException {
        throw new InvalidObjectException("使用序列化代理");
    }

    private static class SerializationProxy implements Serializable {
        private static final long serialVersionUID = 1L;
        private final Date start;
        private final Date end;

        SerializationProxy(Period p) {
            this.start = p.start;
            this.end = p.end;
        }

        private Object readResolve() {
            return new Period(start, end);  // 经过完整验证
        }
    }
}
```

### 实践五：对集合字段做防御性拷贝

```java
// ❌ 直接返回反序列化的集合，可能被外部修改
private void readObject(ObjectInputStream ois) throws IOException, ClassNotFoundException {
    ois.defaultReadObject();
    // 此时 this.items 是反序列化出来的可变列表，可能被共享和修改
}

// ✅ 反序列化后做防御性拷贝
private void readObject(ObjectInputStream ois) throws IOException, ClassNotFoundException {
    ois.defaultReadObject();
    // 防御性拷贝，确保内部状态不被外部修改
    this.items = Collections.unmodifiableList(new ArrayList<>(this.items));
    this.tags = new HashSet<>(this.tags);
}
```

### 实践六：序列化大集合时注意内存

```java
// ❌ 一次性序列化超大集合，可能 OOM
List<BigObject> million = loadMillionObjects();
oos.writeObject(million);  // 几 GB 的字节流，OOM 风险

// ✅ 流式序列化（逐条写入）
oos.writeInt(list.size());  // 先写数量
for (BigObject obj : list) {
    oos.writeObject(obj);   // 逐条写入
}

// 对应的流式反序列化
int size = ois.readInt();
List<BigObject> result = new ArrayList<>(size);
for (int i = 0; i < size; i++) {
    result.add((BigObject) ois.readObject());
}
```

## 5.2 现代 Java 项目的序列化选型

`SerializationBestPracticesDemo.java` 对比了常见序列化方案在实际项目中的选型建议：

### 场景一：RESTful API / 微服务间通信

```java
// ✅ 首选：Jackson JSON（可读性好，调试方便）
@RestController
public class UserController {
    @PostMapping("/users")
    public User createUser(@RequestBody User user) { ... }
    // Spring Boot 自动用 Jackson 序列化/反序列化 JSON
}

// 如果对性能有极高要求（如高频 RPC）：改用 Protobuf + gRPC
```

### 场景二：Redis 缓存

```java
// ✅ 推荐：Jackson JSON（可读，便于 Redis 命令行查看）
@Bean
public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory factory) {
    RedisTemplate<String, Object> template = new RedisTemplate<>();
    template.setConnectionFactory(factory);

    // 用 Jackson 序列化 Value
    Jackson2JsonRedisSerializer<Object> serializer =
        new Jackson2JsonRedisSerializer<>(Object.class);
    template.setValueSerializer(serializer);
    return template;
}

// ❌ 避免用 JdkSerializationRedisSerializer（默认）
// 缺点：字节流大、不可读、有安全风险、跨语言困难
```

### 场景三：消息队列（Kafka/RabbitMQ）

```java
// ✅ Kafka：用 JSON 序列化消息
@Bean
public ProducerFactory<String, User> producerFactory() {
    Map<String, Object> config = new HashMap<>();
    config.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
        JsonSerializer.class);  // Spring Kafka 内置 JSON 序列化
    return new DefaultKafkaProducerFactory<>(config);
}
```

### 场景四：对象持久化到文件（数据存档）

```java
// ✅ 如果需要人类可读：JSON
objectMapper.writerWithDefaultPrettyPrinter().writeValue(new File("data.json"), data);

// ✅ 如果需要紧凑高效：Jackson SMILE（二进制 JSON）或 Protobuf
```

## 5.4 本章总结

**十条实践精华**：
1. 始终显式声明 `serialVersionUID = 1L`
2. 用 `transient` 保护密码、连接、缓存、锁等字段
3. 枚举单例天然安全，普通单例用 `readResolve`
4. 不可变类用序列化代理模式（Effective Java 条目 90）
5. `readObject` 中对可变对象做防御性拷贝
6. 大集合流式序列化，避免内存溢出
7. 现代项目优先用 JSON（Jackson）替代 Java 原生序列化
8. Redis 缓存用 `Jackson2JsonRedisSerializer`，不用 JDK 默认序列化
9. 消息队列消息用 JSON，跨语言兼容且可读
10. 高性能场景考虑 Kryo 或 Protobuf
