# 第一章：什么是序列化？

## 1.1 序列化解决的核心问题

Java 对象存在于内存中，但内存是临时的。序列化（Serialization）把内存中的对象转换为可以**持久化或传输**的字节流，反序列化（Deserialization）则把字节流还原为对象。

```
对象（内存）→ 序列化 → 字节流（磁盘/网络）→ 反序列化 → 对象（内存）
```

**四种典型需求**：

1. **网络传输**：RPC 调用、消息队列（Kafka/RabbitMQ）传递对象
2. **持久化**：把 Session 对象保存到 Redis，保存游戏存档
3. **深度拷贝**：序列化后反序列化得到完全独立的对象拷贝
4. **分布式缓存**：把对象存入 Redis/Memcached

---

## 1.2 Java 原生序列化：实现 Serializable

```java
// SerializationBasicDemo.java 的核心演示
public class User implements Serializable {
    // serialVersionUID 的作用：版本控制（详见第二章）
    private static final long serialVersionUID = 1L;

    private Long id;
    private String username;
    private String email;
    private transient String password;  // transient：不参与序列化

    // getter/setter...
}

// 序列化：对象 → 文件
try (ObjectOutputStream oos = new ObjectOutputStream(
        new FileOutputStream("user.dat"))) {
    oos.writeObject(user);
}

// 反序列化：文件 → 对象
try (ObjectInputStream ois = new ObjectInputStream(
        new FileInputStream("user.dat"))) {
    User restoredUser = (User) ois.readObject();
}
```

## 1.3 序列化的工作原理

Java 序列化不是简单地把字段值写出去，它写出的是完整的**对象图**（Object Graph），包括：

1. **类元数据**：类名、`serialVersionUID`、字段描述信息
2. **字段值**：所有非 `transient`、非 `static` 的字段值
3. **引用关系**：对象之间的引用关系（同一对象被多次引用时只序列化一次）

```java
// SerializationBasicDemo.java → demonstrateObjectGraph()
// 演示对象图序列化：共享引用
User user = new User("张三");
Order order1 = new Order(user, "商品1");
Order order2 = new Order(user, "商品2");  // order1 和 order2 共享同一个 user 对象

// 序列化后，user 只被写入一次
// 反序列化后，order1.getUser() == order2.getUser() 仍然为 true！
// （引用关系被完整保留）
```

## 1.4 transient 关键字的使用场景

```java
public class UserSession implements Serializable {
    private static final long serialVersionUID = 1L;

    private String sessionId;
    private String userId;

    // ✅ 安全敏感：密码不应该被序列化（防止意外泄露）
    private transient String password;

    // ✅ 不可序列化的对象：数据库连接无法序列化
    private transient Connection dbConnection;

    // ✅ 可以重建的缓存：反序列化后重新计算即可，不需要保存
    private transient Map<String, Object> cache;

    // ✅ 线程相关：Lock、Thread 对象不应该跨 JVM 传递
    private transient ReentrantLock lock;
}
```

**static 字段不参与序列化**：静态字段是类级别的，不属于对象状态，序列化/反序列化不会改变它的值。

## 1.5 序列化的几个常见误解

**误解一：实现 Serializable 就万事大吉**

```java
// ❌ 包含不可序列化字段，运行时爆炸
public class UserService implements Serializable {
    private UserRepository repository;  // UserRepository 没有实现 Serializable！
    // 序列化时：java.io.NotSerializableException: UserRepository
}

// ✅ 用 transient 标记不可序列化的字段
private transient UserRepository repository;
```

**误解二：序列化后的字节流是可读的/安全的**

```java
// Java 原生序列化字节流包含类型信息，可以被反序列化为任意对象
// 著名安全漏洞：Apache Commons Collections 反序列化 RCE（远程代码执行）
// 攻击者发送恶意序列化数据，服务端反序列化时执行任意代码

// ✅ 不要反序列化不可信来源的数据
// ✅ 考虑用 JSON（Jackson/Gson）替代 Java 原生序列化
```

**误解三：序列化是最好的深拷贝方式**

```java
// 序列化深拷贝：简单但慢
ByteArrayOutputStream baos = new ByteArrayOutputStream();
new ObjectOutputStream(baos).writeObject(original);
T copy = (T) new ObjectInputStream(new ByteArrayInputStream(baos.toByteArray())).readObject();

// 更快的方式：手动实现 clone()，或用 Apache BeanUtils.copyProperties()
```

## 1.6 本章总结

- **序列化本质**：把内存对象转换为字节流（含类元数据 + 字段值 + 引用关系）
- **实现方式**：实现 `Serializable` 接口（标记接口，无方法）
- **transient**：排除不需要/不能序列化的字段（密码、连接、锁、缓存）
- **static 字段**：不参与序列化，属于类级别状态
- **安全隐患**：Java 原生序列化有反序列化漏洞，不可信数据不要反序列化
