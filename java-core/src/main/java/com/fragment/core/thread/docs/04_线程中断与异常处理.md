# 第四章：线程中断与异常处理

## 4.1 线程中断机制

Java 的线程中断是一种**协作机制**，不是强制停止。`Thread.interrupt()` 只是设置线程的中断标志位，线程本身决定何时响应中断。

```java
// ThreadInterruptDemo.java → 中断基础演示
Thread worker = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {  // 检查中断标志
        doWork();
    }
    System.out.println("线程检测到中断，正常退出");
});

worker.start();
Thread.sleep(1000);
worker.interrupt();  // 设置中断标志，线程在下次检查时退出
```

## 4.2 InterruptedException 的正确处理

阻塞方法（`sleep`、`wait`、`join`）在收到中断时，会：
1. **清除中断标志位**
2. **抛出 `InterruptedException`**

```java
// ThreadInterruptDemo.java → demonstrateInterruptedSleep()
Thread worker = new Thread(() -> {
    try {
        Thread.sleep(10000);  // 阻塞中被中断
    } catch (InterruptedException e) {
        // ❌ 错误1：什么都不做（吞掉中断）
        // 中断标志已被清除，外层代码无法感知中断

        // ❌ 错误2：只打印日志
        // e.printStackTrace();

        // ✅ 正确：重新设置中断标志
        Thread.currentThread().interrupt();
        System.out.println("线程被中断，执行清理...");
    }
});
```

### 三种正确处理模式

**模式1：向上传播（最干净）**

```java
// 如果方法签名允许，直接抛出 InterruptedException
public void doBlockingWork() throws InterruptedException {
    Thread.sleep(1000);  // 不捕获，让调用方处理
}
```

**模式2：恢复中断标志（最常用）**

```java
// 必须捕获但不能传播时，恢复中断标志
public void doWork() {
    try {
        Thread.sleep(1000);
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();  // 恢复中断标志
        return;  // 提前返回，让调用方通过 isInterrupted() 感知
    }
}
```

**模式3：转换为运行时异常（在确实无法处理时）**

```java
// 极少数场景：无法传播受检异常时
public void run() {
    try {
        process();
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        throw new RuntimeException("任务被中断", e);
    }
}
```

## 4.3 如何安全停止一个线程

```java
// ThreadInterruptDemo.java → SafeStoppableTask

public class SafeWorker implements Runnable {
    private volatile boolean stopped = false;

    @Override
    public void run() {
        while (!stopped && !Thread.currentThread().isInterrupted()) {
            try {
                doWork();            // 可能耗时的工作
                Thread.sleep(100);  // 可能阻塞的操作
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();  // 恢复标志
                break;  // 退出循环
            }
        }
        cleanup();  // 确保清理资源
        System.out.println("Worker 已安全停止");
    }

    public void stop() {
        stopped = true;
        // 如果线程正在 sleep/wait，需要中断唤醒它
        // 由外部调用 thread.interrupt()
    }

    private void cleanup() { /* 释放资源 */ }
}

// 使用
Thread thread = new Thread(new SafeWorker());
thread.start();
// 停止时：
worker.stop();
thread.interrupt();  // 唤醒可能在 sleep 中的线程
thread.join(5000);  // 等待最多 5 秒让线程退出
```

## 4.4 线程的未捕获异常处理

线程中的未捕获异常不会传播到创建者线程，默认只打印堆栈到 stderr：

```java
// ThreadInterruptDemo.java → UncaughtExceptionDemo

// 为特定线程设置异常处理器
Thread thread = new Thread(() -> {
    throw new RuntimeException("线程内部异常");
});
thread.setUncaughtExceptionHandler((t, e) -> {
    System.err.println("线程 [" + t.getName() + "] 发生未捕获异常: " + e.getMessage());
    alertService.sendAlert(e);  // 发送告警
    metricsService.recordError(e);  // 记录监控指标
});
thread.start();

// 为所有线程设置默认处理器
Thread.setDefaultUncaughtExceptionHandler((t, e) -> {
    log.error("线程 {} 发生未捕获异常", t.getName(), e);
});

// 线程池中的未捕获异常处理
ThreadFactory factory = r -> {
    Thread t = new Thread(r);
    t.setUncaughtExceptionHandler((thread, ex) -> {
        log.error("线程池任务异常", ex);
    });
    return t;
};
ExecutorService pool = Executors.newFixedThreadPool(4, factory);
```

## 4.5 Future 的异常处理

线程池中的异常不同于普通线程，需要通过 `Future.get()` 获取：

```java
// ThreadInterruptDemo.java → FutureExceptionDemo

ExecutorService pool = Executors.newFixedThreadPool(2);

Future<String> future = pool.submit(() -> {
    if (someCondition) {
        throw new RuntimeException("任务执行失败");
    }
    return "success";
});

try {
    String result = future.get();  // 阻塞等待结果
} catch (ExecutionException e) {
    // 任务内抛出的异常被包装在 ExecutionException 中
    Throwable cause = e.getCause();  // 获取原始异常
    log.error("任务失败: {}", cause.getMessage());
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();
    future.cancel(true);  // 取消任务
}
```

## 4.6 本章总结

- **中断是协作**：`interrupt()` 只设置标志位，线程自行决定响应时机
- **InterruptedException 处理**：捕获后必须恢复标志（`Thread.currentThread().interrupt()`），或向上传播
- **最常见错误**：`catch (InterruptedException e) { }` 吞掉中断，导致调用方无法感知
- **安全停止线程**：`volatile stopped + isInterrupted()` 双重检查，配合 `interrupt()` 唤醒阻塞线程
- **未捕获异常**：设置 `UncaughtExceptionHandler`，确保异常有日志/告警
- **Future 异常**：`future.get()` 抛出 `ExecutionException`，`getCause()` 获取原始异常
