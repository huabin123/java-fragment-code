# 第一章：线程基础与生命周期

## 1.1 为什么需要多线程？

单线程程序的局限：CPU 在等待 I/O（磁盘读写、网络请求）时处于空闲状态。多线程让 CPU 在等待期间去做其他事情。

```
单线程处理 3 个请求（每个 100ms I/O + 10ms CPU）：
请求1: [===CPU10ms===][==I/O 100ms==]
请求2:                                [===CPU10ms===][==I/O 100ms==]
请求3:                                                               [===CPU10ms===][==I/O 100ms==]
总时间：330ms

多线程处理 3 个请求：
请求1: [===CPU10ms===][==I/O 100ms==]
请求2: [===CPU10ms===][==I/O 100ms==]
请求3: [===CPU10ms===][==I/O 100ms==]
总时间：110ms（快 3x）
```

## 1.2 线程的六种状态

`ThreadLifecycleDemo.java` 演示了线程的完整生命周期：

```
NEW → RUNNABLE → BLOCKED/WAITING/TIMED_WAITING → RUNNABLE → TERMINATED
```

| 状态 | 含义 | 触发条件 |
|------|------|---------|
| `NEW` | 已创建，未启动 | `new Thread(...)` 后，`start()` 前 |
| `RUNNABLE` | 运行中或等待 CPU 调度 | 调用 `start()` 后 |
| `BLOCKED` | 等待获取 synchronized 锁 | 尝试进入 synchronized 块/方法 |
| `WAITING` | 无限期等待 | `Object.wait()`、`Thread.join()`、`LockSupport.park()` |
| `TIMED_WAITING` | 有限期等待 | `Thread.sleep(n)`、`wait(n)`、`join(n)` |
| `TERMINATED` | 执行完毕 | `run()` 方法返回或抛出异常 |

```java
// ThreadLifecycleDemo.java → demonstrateLifecycle()
Thread thread = new Thread(() -> {
    try {
        Thread.sleep(1000);  // TIMED_WAITING
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
});

System.out.println(thread.getState());  // NEW

thread.start();
Thread.sleep(100);  // 等待线程进入 sleep
System.out.println(thread.getState());  // TIMED_WAITING

thread.join();
System.out.println(thread.getState());  // TERMINATED
```

## 1.3 创建线程的四种方式

```java
// ThreadBasicDemo.java 演示的四种方式

// 方式1：继承 Thread 类（不推荐，占用继承名额）
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread: " + Thread.currentThread().getName());
    }
}
new MyThread().start();

// 方式2：实现 Runnable 接口（推荐，可组合）
Runnable task = () -> System.out.println("Runnable: " + Thread.currentThread().getName());
new Thread(task).start();

// 方式3：实现 Callable + FutureTask（有返回值）
Callable<Integer> callable = () -> {
    Thread.sleep(100);
    return 42;
};
FutureTask<Integer> future = new FutureTask<>(callable);
new Thread(future).start();
Integer result = future.get();  // 阻塞等待结果

// 方式4：线程池（生产环境首选）
ExecutorService pool = Executors.newFixedThreadPool(4);
pool.submit(() -> System.out.println("Pool: " + Thread.currentThread().getName()));
pool.shutdown();
```

**方式选择**：
- 生产代码 → 方式4（线程池，控制资源）
- 需要返回值 → 方式3（Callable + Future）
- 简单任务 → 方式2（Runnable Lambda）
- 永远不用方式1（浪费继承）

## 1.4 线程的核心属性

```java
// ThreadBasicDemo.java → demonstrateThreadProperties()
Thread thread = new Thread(() -> {}, "worker-thread-1");

thread.getName();         // "worker-thread-1"
thread.getId();           // 唯一 long ID
thread.getPriority();     // 1-10，默认 5（Thread.NORM_PRIORITY）
thread.isDaemon();        // 是否是守护线程
thread.getState();        // 当前状态
thread.isAlive();         // 是否还活着

// 设置守护线程（必须在 start() 前设置）
thread.setDaemon(true);   // JVM 在所有非守护线程结束后退出，不等守护线程
thread.start();
```

**守护线程的典型应用**：JVM 的 GC 线程、后台监控线程、心跳检测线程。如果这些线程是非守护线程，即使主业务结束，JVM 也不会退出。

## 1.5 线程优先级的真相

```java
// ThreadBasicDemo.java → demonstratePriority()
Thread highPriority = new Thread(task);
highPriority.setPriority(Thread.MAX_PRIORITY);  // 10

Thread lowPriority = new Thread(task);
lowPriority.setPriority(Thread.MIN_PRIORITY);   // 1

// 警告：线程优先级只是"建议"，不是保证！
// 不同操作系统对优先级的实现差异很大：
// - Linux：基本忽略 Java 优先级
// - Windows：有一定影响但不可预测
// - 绝不能用优先级来控制线程执行顺序！
```

## 1.6 本章总结

- **为什么多线程**：I/O 等待期间利用 CPU，提升吞吐量
- **六种状态**：NEW → RUNNABLE → BLOCKED/WAITING/TIMED_WAITING → TERMINATED
- **BLOCKED vs WAITING**：BLOCKED 是等 synchronized 锁；WAITING 是主动调用 wait/join/park 等待唤醒
- **创建方式**：生产用线程池；有返回值用 Callable；简单任务用 Runnable Lambda
- **守护线程**：JVM 不等守护线程，用于后台任务；必须在 start() 前设置
- **优先级**：只是操作系统的建议，不能依赖它控制执行顺序
