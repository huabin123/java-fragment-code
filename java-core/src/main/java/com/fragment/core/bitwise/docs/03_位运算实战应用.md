# 第三章：位运算实战应用

## 3.1 权限管理系统

`BitwisePracticalDemo.java → demonstratePermissionManager()` 展示了最经典的位运算实战场景：

```java
static class PermissionManager {
    // 权限标志位定义（每个权限占一位）
    static final int READ    = 1 << 0;  // 0001 = 1
    static final int WRITE   = 1 << 1;  // 0010 = 2
    static final int EXECUTE = 1 << 2;  // 0100 = 4
    static final int DELETE  = 1 << 3;  // 1000 = 8

    private int permissions = 0;  // 初始无任何权限

    // 授予权限（OR：设置对应位为1）
    void grant(int permission) { permissions |= permission; }

    // 撤销权限（AND NOT：清除对应位）
    void revoke(int permission) { permissions &= ~permission; }

    // 检查权限（AND：判断对应位是否为1）
    boolean hasPermission(int perm) { return (permissions & perm) == perm; }

    // 切换权限（XOR：翻转对应位）
    void toggle(int permission) { permissions ^= permission; }
}

// 使用：同时授予读写权限
pm.grant(READ | WRITE);  // permissions = 0011
pm.hasPermission(READ);  // true
pm.revoke(WRITE);        // permissions = 0001
pm.hasPermission(WRITE); // false
```

**同时检查多个权限**：

```java
// 检查是否同时拥有读和执行权限
boolean hasReadAndExec = pm.hasPermission(READ | EXECUTE);
// (permissions & (READ|EXECUTE)) == (READ|EXECUTE)
```

Linux 文件权限（rwx）、Java 的 `AccessFlag`、各种框架的权限系统都是这个模式。

## 3.2 角色权限系统：位掩码组合

`demonstrateRolePermissionManager()` 演示了角色（多权限集合）的实现：

```java
// 角色定义：每个角色是多个权限位的组合
static final int ADMIN   = READ | WRITE | EXECUTE | DELETE;  // 1111
static final int EDITOR  = READ | WRITE;                     // 0011
static final int VIEWER  = READ;                             // 0001

// 用户的权限 = 多个角色的权限合并
int userPerm = EDITOR | EXECUTE;  // 0111（编辑者 + 执行权限）

// 检查用户是否满足某角色的所有权限
boolean isAdmin = (userPerm & ADMIN) == ADMIN;  // false（缺 DELETE）
boolean isEditor = (userPerm & EDITOR) == EDITOR;  // true
```

## 3.3 订单状态机：位运算管理多状态

`demonstrateOrderStatus()` 展示了用位掩码表示订单状态的多种组合：

```java
// 订单状态标志
static final int CREATED   = 1 << 0;  // 已创建
static final int PAID      = 1 << 1;  // 已支付
static final int SHIPPED   = 1 << 2;  // 已发货
static final int DELIVERED = 1 << 3;  // 已送达
static final int CANCELLED = 1 << 4;  // 已取消

// 状态流转
int status = CREATED;
status |= PAID;      // 支付后：CREATED | PAID
status |= SHIPPED;   // 发货后：CREATED | PAID | SHIPPED

// 查询某状态是否已到达
boolean hasPaid = (status & PAID) != 0;

// 查询当前状态是否在"正常流程"中（未取消）
boolean isNormal = (status & CANCELLED) == 0;
```

## 3.4 HashMap 中的位运算

HashMap 源码中大量使用位运算，理解它们需要扎实的位运算基础：

```java
// 1. 容量必须是 2 的幂（tableSizeFor 方法）
static final int tableSizeFor(int cap) {
    int n = cap - 1;
    n |= n >>> 1;   // 让最高位的1扩散到低位
    n |= n >>> 2;
    n |= n >>> 4;
    n |= n >>> 8;
    n |= n >>> 16;
    return n + 1;   // 结果是大于等于 cap 的最小 2 的幂
}
// tableSizeFor(10) → 16，tableSizeFor(16) → 16，tableSizeFor(17) → 32

// 2. 桶定位（等价于取模但更快）
int index = hash & (capacity - 1);
// capacity=16（1_0000），capacity-1=15（0_1111）
// hash & 0b1111 取 hash 的低4位 = hash % 16

// 3. hash 扰动（让高位参与低位计算，减少碰撞）
static final int hash(Object key) {
    int h = key.hashCode();
    return h ^ (h >>> 16);  // 高16位与低16位异或
}

// 4. 扩容时判断节点放高桶还是低桶
// 旧容量 oldCap=16（0b10000），扩容后 newCap=32
// (hash & oldCap) == 0 → 放低桶（原位置）
// (hash & oldCap) != 0 → 放高桶（原位置 + oldCap）
if ((e.hash & oldCap) == 0) loHead = loTail = e;
else hiHead = hiTail = e;
```

## 3.5 BitSet：Java 标准库的位数组

`demonstrateBitSet()` 展示了 `java.util.BitSet` 的应用：

```java
BitSet bitSet = new BitSet(64);  // 64 位

bitSet.set(5);      // 设置第5位
bitSet.set(10, 20); // 设置第10-19位
bitSet.get(5);      // 读取第5位 → true
bitSet.clear(5);    // 清除第5位
bitSet.flip(15);    // 翻转第15位

// 集合操作
BitSet a = new BitSet(); a.set(1); a.set(2); a.set(3);
BitSet b = new BitSet(); b.set(2); b.set(3); b.set(4);
a.and(b);   // 交集：{2, 3}
a.or(b);    // 并集：{1, 2, 3, 4}
a.xor(b);   // 对称差：{1, 4}

// 实用场景：用 1 亿 bit（12.5MB）表示 1 亿个用户是否登录
BitSet onlineUsers = new BitSet(100_000_000);
onlineUsers.set(userId);         // 用户上线
onlineUsers.clear(userId);       // 用户下线
onlineUsers.get(userId);         // 查询是否在线
onlineUsers.cardinality();       // 在线用户数
```

## 3.6 布隆过滤器原理

`demonstrateBloomFilter()` 演示了用 BitSet 实现布隆过滤器的核心思想：

```java
// 简化布隆过滤器（用多个 hash 函数对应多个位）
class SimpleBloomFilter {
    private final BitSet bits;
    private final int[] hashSeeds;

    void add(String key) {
        for (int seed : hashSeeds) {
            bits.set(Math.abs(key.hashCode() * seed) % bits.size());
        }
    }

    boolean mightContain(String key) {
        for (int seed : hashSeeds) {
            if (!bits.get(Math.abs(key.hashCode() * seed) % bits.size())) {
                return false;  // 某位为0，一定不存在
            }
        }
        return true;  // 所有位都为1，可能存在（有误判率）
    }
}
```

**布隆过滤器特性**：
- 说"不存在"一定是真的不存在（100% 准确）
- 说"存在"可能是误判（有一定误判率）
- Redis 的 BloomFilter、Guava 的 `BloomFilter` 都是这个原理

## 3.7 本章总结

- **权限系统**：标志位定义（1<<n）+ grant(|=) + revoke(&=~) + check(&) + toggle(^=)，一个 int 存储多权限
- **HashMap 核心**：容量必须是 2 的幂（tableSizeFor）、桶定位（hash & (cap-1)）、扩容分桶（hash & oldCap）
- **BitSet**：Java 标准库位数组，适合大规模布尔集合（亿级用户在线状态）
- **布隆过滤器**：多个 hash 函数 + BitSet，空间效率极高，"不存在"100% 准确，"存在"有误判率
