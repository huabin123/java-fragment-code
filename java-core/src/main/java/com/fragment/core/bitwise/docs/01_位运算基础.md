# 第一章：位运算基础

## 1.1 为什么要学位运算？

位运算直接操作二进制位，是最接近硬件的计算方式。在 Java 中，位运算的价值体现在：

- **性能**：位运算比乘除法快（编译器会优化，但显式位运算更可控）
- **空间压缩**：一个 int 可以存储 32 个布尔标志（权限系统、状态机）
- **算法基础**：HashMap 的容量计算、ThreadLocal 的 index 计算都依赖位运算
- **面试高频**：判断奇偶、2 的幂次、不用加法做加法等经典题

## 1.2 六种位运算符

`BitwiseBasicDemo.java` 演示了全部六种运算符：

### & 按位与：两位都为 1 才为 1

```java
// demonstrateAndOperator()
int a = 12;  // 1100
int b = 10;  // 1010
int c = a & b; // 1000 = 8

// 核心用法：掩码操作
// 判断奇偶（最低位是否为1）
boolean isOdd = (n & 1) == 1;  // 比 n % 2 != 0 更快

// HashMap 中的桶定位（容量必须是2的幂的原因）
int index = hash & (capacity - 1);  // 等价于 hash % capacity，但更快
```

### | 按位或：任意一位为 1 则为 1

```java
// demonstrateOrOperator()
int a = 12;  // 1100
int b = 10;  // 1010
int c = a | b; // 1110 = 14

// 核心用法：设置标志位
int flags = 0;
flags = flags | 0b0001;  // 设置第1位
flags = flags | 0b0100;  // 设置第3位
// flags = 0b0101
```

### ^ 按位异或：两位不同则为 1

```java
// demonstrateXorOperator()
int a = 12;  // 1100
int b = 10;  // 1010
int c = a ^ b; // 0110 = 6

// 核心特性：a ^ a = 0，a ^ 0 = a，可交换结合
// HashMap 的扰动函数：hash ^ (hash >>> 16)
// 不用临时变量交换两数
a = a ^ b;
b = a ^ b;  // b = (a^b)^b = a
a = a ^ b;  // a = (a^b)^a = b
```

### ~ 按位取反：0 变 1，1 变 0

```java
// demonstrateNotOperator()
int a = 5;   // 00000000 00000000 00000000 00000101
int b = ~a;  // 11111111 11111111 11111111 11111010 = -6

// 规律：~n = -(n+1)
// 用途：清除标志位
flags = flags & ~0b0001;  // 清除第1位（~0b0001 = 全1除了第1位）
```

### << 左移：向左移动，低位补 0

```java
// demonstrateLeftShift()
int a = 1;
int b = a << 3;  // 1 << 3 = 8（等价于 1 * 2^3）

// 规律：左移 n 位 = 乘以 2^n（未溢出时）
// HashMap 初始容量计算：
int capacity = 1 << 4;  // 16
int bigger = 1 << 16;   // 65536
```

### >> 和 >>> 右移

```java
// >> 有符号右移：高位补符号位（正数补0，负数补1）
int a = -8;   // 11111111 ... 11111000
int b = a >> 1;  // 11111111 ... 11111100 = -4（保持负数）

// >>> 无符号右移：高位始终补 0
int c = a >>> 1; // 01111111 ... 11111100 = 大正数（丢失符号）

// HashMap 的扰动函数用 >>>（让高位参与低位计算，不关心符号）
static final int hash(Object key) {
    int h = key.hashCode();
    return h ^ (h >>> 16);  // 高16位与低16位异或
}
```

## 1.3 负数的补码：理解 ~ 的关键

```java
// demonstrateNegativeNumbers()
int n = 5;  // 原码：00000101

// 负数存储为补码 = 反码 + 1
// 5  → 原码：0...0101
// -5 → 反码：1...1010（取反）
//      补码：1...1011（+1）= -5 的实际存储

int neg = -5;
System.out.println(Integer.toBinaryString(neg));
// 输出：11111111111111111111111111111011
```

**为什么用补码？** 加法器只需要一套电路就能处理正负数运算：`5 + (-5) = 0...0101 + 1...1011 = 1 0...0000`，溢出位丢弃，结果正好是 0。

## 1.4 常见陷阱

```java
// demonstrateCommonPitfalls()

// 陷阱1：优先级问题（位运算优先级低于算术运算）
int result1 = 2 + 3 << 1;   // (2+3) << 1 = 10，不是 2 + (3<<1) = 8
// ✅ 加括号明确优先级

// 陷阱2：int 溢出
int big = 1 << 31;  // -2147483648（溢出，变成最小负数）
// ✅ 用 long：1L << 31

// 陷阱3：对 byte/short 运算后自动提升为 int
byte b = 0b11110000;
byte result = (byte)(b >> 4);  // 需要强转，否则编译错误

// 陷阱4：负数右移不能用于除法
int a = -7;
int b = a >> 1;  // -4，不是 -3（向负无穷舍入，不是向零）
int c = a / 2;   // -3（向零舍入）
```

## 1.5 本章总结

- **& 与**：掩码取位，判断奇偶，HashMap 桶定位
- **| 或**：设置标志位
- **^ 异或**：翻转位，不用临时变量交换，扰动函数
- **~ 取反**：清除标志位，规律 ~n = -(n+1)
- **<< 左移**：乘以 2 的幂
- **>> / >>> 右移**：除以 2 的幂，`>>>` 忽略符号位
