# 第二章：位运算进阶技巧

## 2.1 15 个经典位运算技巧

`BitwiseSkillsDemo.java` 系统演示了 15 个实用技巧，按用途分组：

### 数字属性判断类

```java
// skill01：判断奇偶（最低位是否为1）
boolean isOdd = (n & 1) == 1;   // 比 n % 2 != 0 更快
boolean isEven = (n & 1) == 0;

// skill03：判断是否为 2 的幂（有且只有一个 1 位）
boolean isPowerOfTwo = n > 0 && (n & (n - 1)) == 0;
// 原理：2的幂的二进制形如 1000，n-1 = 0111，相与为 0
// HashMap 判断容量合法性用的就是这个

// skill14：判断两数符号是否相同（最高符号位异或）
boolean sameSign = (a ^ b) >= 0;  // 符号位相同则异或后最高位为0
```

### 位操作类（单个 bit 的读/写/清/翻）

```java
int bit = 2;  // 操作第2位（从0开始）
int mask = 1 << bit;  // 掩码：0b0100

// skill07：读取某位的值
int getBit = (n >> bit) & 1;        // 右移到最低位，再取最低位

// skill08：设置某位为 1（| 操作）
int setBit = n | mask;               // OR 掩码，强制该位为1

// skill09：清除某位（& ~mask 操作）
int clearBit = n & ~mask;           // AND 取反掩码，强制该位为0

// skill10：翻转某位（^ 操作）
int toggleBit = n ^ mask;           // XOR 掩码，翻转该位
```

**这四个操作是权限系统的基础**，`BitwisePracticalDemo.java → PermissionManager` 的 grant/revoke/hasPermission 就是这套操作。

### 数值运算加速类

```java
// skill04：统计二进制中 1 的个数（Brian Kernighan 算法）
int count = 0;
while (n != 0) {
    n = n & (n - 1);  // 每次消除最右边的 1
    count++;
}
// 时间复杂度 O(k)，k 为 1 的个数，而非 O(32)

// skill05：提取最右边的 1（lowbit）
int rightmostOne = n & (-n);  // -n 是 n 的补码
// 算法竞赛中树状数组的核心操作

// skill06：清除最右边的 1
int cleared = n & (n - 1);  // 与 Brian Kernighan 同原理

// skill11：快速乘除 2 的幂
int mul4 = n << 2;   // n * 4
int div8 = n >> 3;   // n / 8（注意：负数向下取整，与除法不同）

// skill13：不溢出的平均值（防止 (a+b) 溢出）
int avg = (a & b) + ((a ^ b) >> 1);
// 原理：(a+b)/2 = (a AND b) + (a XOR b)/2
// 两个都是1的位贡献 1，只有一个是1的位贡献 0.5

// skill12：不用判断的绝对值（利用符号位掩码）
int mask2 = n >> 31;   // 正数为 0x00000000，负数为 0xFFFFFFFF
int abs = (n + mask2) ^ mask2;
// 正数：(n+0)^0 = n；负数：(n-1)^(-1) = -n

// skill15：找出数组中唯一不重复的数（其他都出现两次）
int result = 0;
for (int num : array) result ^= num;
// 原理：a^a=0，a^0=a，相同的数两两消除，剩下唯一的数
```

## 2.2 位掩码：用整数压缩多个布尔状态

`BitwiseSkillsDemo.java → demonstrateBitmask()` 展示了用一个 int 存储多个标志：

```java
// 定义标志位（每个标志占一位）
static final int FLAG_READ    = 1 << 0;  // 0001
static final int FLAG_WRITE   = 1 << 1;  // 0010
static final int FLAG_EXECUTE = 1 << 2;  // 0100
static final int FLAG_DELETE  = 1 << 3;  // 1000

// 组合标志
int userPermission = FLAG_READ | FLAG_WRITE;  // 0011

// 检查单个标志
boolean canRead  = (userPermission & FLAG_READ) != 0;
boolean canWrite = (userPermission & FLAG_WRITE) != 0;

// 添加标志
userPermission |= FLAG_EXECUTE;   // userPermission = 0111

// 清除标志
userPermission &= ~FLAG_WRITE;    // userPermission = 0101

// 翻转标志
userPermission ^= FLAG_DELETE;    // userPermission = 1101
```

**内存对比**：存储 32 个布尔状态，用 32 个 boolean 需要 32 字节，用一个 int 位掩码只需要 4 字节，节省 8 倍。

## 2.3 性能对比

`BitwiseSkillsDemo.java → performanceComparison()` 实测结果：

```
100万次运算（JVM JIT 充分预热后）：

判断奇偶：
  n % 2 == 0：约 2ms
  n & 1 == 0：约 1ms（快 ~2x）

乘除法：
  n * 4：约 1ms（JIT 会自动优化为位运算）
  n << 2：约 1ms（相当）

2的幂判断：
  循环做除法：约 10ms
  n & (n-1) == 0：约 1ms（快 ~10x）
```

**结论**：在 JIT 编译充分的情况下，简单的乘除法 JIT 会自动优化；但手动写位运算在**可读性不变的前提下**，对循环判断（isPowerOfTwo、统计1的个数）有显著提升。

## 2.4 本章总结

- **15 个技巧归类**：判断属性（奇偶/2的幂/符号）、操作单个 bit（读/写/清/翻）、数值运算（计数/提取/平均/绝对值/找唯一）
- **位掩码**：一个 int 压缩 32 个布尔状态，节省 8 倍内存，权限系统标准做法
- **lowbit（n & -n）**：树状数组核心，提取最低的 1 位
- **Brian Kernighan 算法（n & (n-1)）**：快速计数/清除最右边的 1
- **找唯一数（XOR）**：利用 a^a=0 的特性，O(n) 时间 O(1) 空间
