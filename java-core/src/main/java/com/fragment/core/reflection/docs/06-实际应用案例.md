# 第六章：反射实际应用案例

## 6.1 JSON 序列化框架（Jackson 原理）

`ReflectionApplicationDemo.java → SimpleJsonSerializer` 演示了用反射实现 JSON 序列化的核心逻辑：

```java
// 序列化：对象 → JSON 字符串
public String toJson(Object obj) throws Exception {
    if (obj == null) return "null";
    if (obj instanceof String) return "\"" + obj + "\"";
    if (obj instanceof Number || obj instanceof Boolean) return obj.toString();

    Class<?> clazz = obj.getClass();
    StringBuilder json = new StringBuilder("{");

    for (Field field : clazz.getDeclaredFields()) {
        if (Modifier.isStatic(field.getModifiers())) continue;
        field.setAccessible(true);

        // 读取字段名（优先用 @JsonProperty 注解）
        String name = field.isAnnotationPresent(JsonProperty.class)
            ? field.getAnnotation(JsonProperty.class).value()
            : field.getName();

        // 读取字段值，递归序列化
        Object value = field.get(obj);
        json.append("\"").append(name).append("\":").append(toJson(value));
        json.append(",");
    }

    if (json.length() > 1) json.deleteCharAt(json.length() - 1);
    return json.append("}").toString();
}

// 反序列化：JSON 字符串 → 对象
public <T> T fromJson(String json, Class<T> type) throws Exception {
    T obj = type.getDeclaredConstructor().newInstance();
    // 解析 JSON key-value 对
    Map<String, String> values = parseJson(json);

    for (Field field : type.getDeclaredFields()) {
        field.setAccessible(true);
        String name = field.getName();
        if (values.containsKey(name)) {
            // 类型转换后赋值
            Object converted = convertToType(values.get(name), field.getType());
            field.set(obj, converted);
        }
    }
    return obj;
}
```

## 6.2 配置自动注入（Spring @Value 原理）

```java
// ReflectionApplicationDemo.java → ConfigInjector
// 把配置文件的值注入到带 @Value 注解的字段

public class ConfigInjector {
    private final Properties config;

    public void inject(Object bean) throws Exception {
        for (Field field : getAllFields(bean.getClass())) {
            Value annotation = field.getAnnotation(Value.class);
            if (annotation == null) continue;

            // 解析 "${key:defaultValue}" 格式
            String expression = annotation.value();  // "${db.url:localhost}"
            String key = expression.substring(2, expression.contains(":")
                ? expression.indexOf(":") : expression.length() - 1);
            String defaultVal = expression.contains(":")
                ? expression.substring(expression.indexOf(":") + 1, expression.length() - 1)
                : null;

            String value = config.getProperty(key, defaultVal);
            if (value != null) {
                field.setAccessible(true);
                // 类型转换：String → 字段的实际类型
                field.set(bean, convertToFieldType(value, field.getType()));
            }
        }
    }
}

// 使用示例
public class DatabaseConfig {
    @Value("${db.host:localhost}")
    private String host;

    @Value("${db.port:3306}")
    private int port;  // 自动转换 String → int
}
```

## 6.3 反射在代码生成中的应用

`ReflectionApplicationDemo.java → CodeGenerator` 展示了用反射自动生成代码：

```java
// 根据实体类反射生成 CRUD 接口
public String generateRepository(Class<?> entityClass) {
    String entityName = entityClass.getSimpleName();

    // 找到 @Id 字段，确定主键类型
    Field idField = Arrays.stream(entityClass.getDeclaredFields())
        .filter(f -> f.isAnnotationPresent(Id.class))
        .findFirst()
        .orElseThrow(() -> new IllegalArgumentException("缺少 @Id 字段"));

    String idType = idField.getType().getSimpleName();

    return String.format("""
        public interface %sRepository {
            %s findById(%s id);
            List<%s> findAll();
            void save(%s entity);
            void deleteById(%s id);
        }
        """, entityName, entityName, idType, entityName, entityName, idType);
}
```

## 6.4 反射的正确使用边界

通过前几章的案例，可以总结出反射的合理使用场景与禁区：

**✅ 适合用反射的场景**：

```
框架代码：不知道用户类型的情况
  - ORM 映射（字段→数据库列）
  - IoC 容器（扫描类、创建 Bean、注入依赖）
  - JSON 序列化（字段→JSON key）
  - 测试框架（发现 @Test 方法）

工具代码：需要统一处理任意类型
  - 深度拷贝工具
  - 对象比较工具
  - 日志工具（自动格式化对象）

插件/扩展系统：运行时加载未知类
  - 动态加载 jar 中的插件类
  - 脚本引擎中执行用户代码
```

**❌ 不应该用反射的场景**：

```java
// ❌ 业务代码中用反射代替正常调用（没有必要且慢）
Method method = userService.getClass().getMethod("findById", Long.class);
User user = (User) method.invoke(userService, 1L);
// 完全可以直接：User user = userService.findById(1L);

// ❌ 用反射绕过封装修改对象状态（破坏设计）
Field balance = account.getClass().getDeclaredField("balance");
balance.setAccessible(true);
balance.set(account, 999999.0);  // 直接修改余额，绕过所有业务规则！

// ❌ 在热点路径上无缓存地使用反射
for (int i = 0; i < 1000000; i++) {
    clazz.getDeclaredField("name").get(obj);  // 每次都查找字段，极慢
}
```

## 6.5 本章总结

- **JSON 序列化**：反射遍历字段 + 递归序列化，`@JsonProperty` 注解定制字段名映射
- **配置注入**：解析 `@Value("${key:default}")` 表达式 + 反射字段赋值 + 类型转换
- **代码生成**：反射读取类结构 → 自动生成 CRUD 接口/SQL 脚本等模板代码
- **边界原则**：反射用于框架/工具代码处理未知类型，不用于已知类型的业务逻辑
