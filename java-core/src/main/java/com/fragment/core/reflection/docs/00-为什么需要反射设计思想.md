# 第零章：为什么需要反射？

## 0.1 反射解决的核心问题：运行时的未知性

在编译时，我们知道所有要操作的类。但框架（Spring、MyBatis、Jackson）在编写时不知道用户会传入什么类型——反射让代码在运行时"发现"并操作未知的类。

```java
// 没有反射：框架需要为每种类型手写代码（不可能完成）
public void inject(UserService service) { ... }
public void inject(OrderService service) { ... }
// 用户每加一个 Service，框架就要改一次代码 → 框架无法存在

// 有反射：框架用统一代码处理任意类型
public void inject(Object bean) throws Exception {
    Class<?> clazz = bean.getClass();  // 运行时获取实际类型
    for (Field field : clazz.getDeclaredFields()) {
        if (field.isAnnotationPresent(Autowired.class)) {
            // 动态注入依赖，不管 bean 是什么类型
        }
    }
}
```

**反射的本质**：Java 在类加载时把类的结构信息（字段、方法、构造函数、注解）存储在 `Class` 对象中，反射 API 让我们在运行时读取和使用这些信息。

---

## 0.2 反射的三类核心能力

**能力一：运行时类型检查**
```java
object.getClass()              // 获取运行时实际类型
clazz.isAssignableFrom(other)  // 类型是否兼容
clazz.isInstance(obj)          // 等价于 instanceof
```

**能力二：运行时结构检查**
```java
clazz.getDeclaredFields()       // 获取所有字段（含 private）
clazz.getDeclaredMethods()      // 获取所有方法
clazz.getDeclaredConstructors() // 获取所有构造函数
clazz.getAnnotations()          // 获取注解
```

**能力三：运行时动态调用**
```java
field.set(obj, value)           // 给字段赋值（即使是 private）
method.invoke(obj, args)        // 调用方法
constructor.newInstance(args)   // 创建实例
```

---

## 0.3 反射驱动了哪些技术？

几乎所有 Java 框架的核心都依赖反射：

| 框架 | 反射的应用 |
|------|-----------|
| Spring IoC | 扫描 `@Component`，反射创建 Bean，`@Autowired` 注入字段 |
| Spring AOP | 反射调用目标方法，读取 `@Transactional` 等注解 |
| JPA/MyBatis | 扫描 `@Entity/@Table`，反射读取字段名映射数据库列 |
| Jackson/Gson | 反射遍历字段，序列化/反序列化 JSON |
| JUnit | 反射查找 `@Test` 方法并调用 |
| Lombok（APT）| 编译时反射分析类结构，生成 getter/setter |

---

## 0.4 反射的代价

反射不是免费的：

1. **性能开销**：反射调用比直接调用慢约 10-50 倍（JIT 优化前），JIT 充分预热后差距缩小到 3-5 倍
2. **安全检查绕过**：`setAccessible(true)` 可以访问 `private` 字段，破坏封装
3. **类型安全丧失**：编译器无法检查反射调用的类型错误，只能在运行时发现
4. **代码可读性**：反射代码比直接调用难读，调试困难

**使用原则**：反射用于框架和工具代码，不用于业务逻辑。业务代码的运行路径是编译时已知的，不需要反射。

---

## 0.5 本章总结

- **反射的必要性**：框架在编写时无法预知用户的类型，反射提供运行时类型信息
- **三类能力**：类型检查、结构检查、动态调用
- **应用场景**：Spring IoC/AOP、ORM 映射、JSON 序列化、单元测试框架都依赖反射
- **代价**：性能开销、安全检查绕过、类型安全丧失——只用于框架代码，不用于业务代码
