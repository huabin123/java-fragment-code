# 第三章：反射与泛型

## 3.1 反射如何获取泛型信息

类型擦除后，运行时的实例本身不保留泛型参数，但 **字段声明、方法签名、类继承关系** 中保留了泛型信息（存储在字节码的 Signature 属性中）。

```java
// GenericReflectionDemo.java → 四种保留泛型信息的入口

// 1. 字段的泛型类型
public class UserService {
    private List<User> users;
    private Map<String, List<Order>> orderMap;
}
Field usersField = UserService.class.getDeclaredField("users");
Type fieldType = usersField.getGenericType();
// ParameterizedType: java.util.List<User>

// 2. 方法参数的泛型类型
Method method = UserService.class.getMethod("findUsers", List.class);
Type[] paramTypes = method.getGenericParameterTypes();
// paramTypes[0] = ParameterizedType: java.util.List<Long>

// 3. 方法返回值的泛型类型
Type returnType = method.getGenericReturnType();
// ParameterizedType: java.util.List<User>

// 4. 类的泛型父类（最常用：获取子类固化的类型参数）
public class UserRepository extends GenericRepository<User, Long> {}
Type superType = UserRepository.class.getGenericSuperclass();
// ParameterizedType: GenericRepository<User, Long>
```

## 3.2 Type 体系：四种类型接口

反射中的泛型类型用 `java.lang.reflect.Type` 接口表示，有四个子接口：

```java
// GenericReflectionDemo.java → analyzeType()
public static void analyzeType(Type type) {
    if (type instanceof Class) {
        // 普通类：String.class, int.class
        Class<?> clazz = (Class<?>) type;
        System.out.println("普通类: " + clazz.getName());

    } else if (type instanceof ParameterizedType) {
        // 参数化类型：List<String>, Map<K, V>
        ParameterizedType pt = (ParameterizedType) type;
        System.out.println("原始类型: " + pt.getRawType());
        System.out.println("类型参数: " + Arrays.toString(pt.getActualTypeArguments()));
        // List<String> → rawType=List, typeArgs=[String]
        // Map<String, List<User>> → rawType=Map, typeArgs=[String, List<User>]

    } else if (type instanceof TypeVariable) {
        // 类型变量：T, E, K, V（未被固化的泛型参数）
        TypeVariable<?> tv = (TypeVariable<?>) type;
        System.out.println("类型变量: " + tv.getName());
        System.out.println("边界: " + Arrays.toString(tv.getBounds()));
        // T extends Comparable<T> → name=T, bounds=[Comparable<T>]

    } else if (type instanceof WildcardType) {
        // 通配符类型：? extends Animal, ? super Dog
        WildcardType wt = (WildcardType) type;
        System.out.println("上界: " + Arrays.toString(wt.getUpperBounds()));
        System.out.println("下界: " + Arrays.toString(wt.getLowerBounds()));

    } else if (type instanceof GenericArrayType) {
        // 泛型数组：T[], List<String>[]
        GenericArrayType gat = (GenericArrayType) type;
        System.out.println("数组元素类型: " + gat.getGenericComponentType());
    }
}
```

## 3.3 实战：从继承关系获取类型参数

`GenericReflectionDemo.java → getGenericTypeFromSuperclass()` 演示了最重要的应用——框架通过继承获取用户指定的类型参数：

```java
// 框架基类
public abstract class BaseConverter<S, T> {
    // 框架在运行时需要知道 S 和 T 的具体类型
    protected Class<S> sourceType;
    protected Class<T> targetType;

    @SuppressWarnings("unchecked")
    protected BaseConverter() {
        // 通过继承关系获取类型参数
        ParameterizedType pt = (ParameterizedType) getClass().getGenericSuperclass();
        // getClass() 是具体子类，getGenericSuperclass() 是带泛型的父类
        this.sourceType = (Class<S>) pt.getActualTypeArguments()[0];
        this.targetType = (Class<T>) pt.getActualTypeArguments()[1];
    }
}

// 用户的子类（固化了类型参数）
public class UserToUserDtoConverter extends BaseConverter<User, UserDto> {
    // sourceType = User.class, targetType = UserDto.class ← 自动推断！
}

UserToUserDtoConverter converter = new UserToUserDtoConverter();
// converter.sourceType = User.class
// converter.targetType = UserDto.class
```

**这是 Spring 的 `ResolvableType`、Jackson 的 `TypeReference`、MyBatis 的 `TypeHandler` 的核心原理**。

## 3.4 Jackson TypeReference 的工作原理

```java
// Jackson 反序列化复杂泛型的标准写法
List<User> users = objectMapper.readValue(json, new TypeReference<List<User>>() {});
// 为什么要用匿名类？为什么不能用 List.class？

// 原因：
// 1. new TypeReference<List<User>>() {} 创建了 TypeReference<List<User>> 的匿名子类
// 2. 匿名子类固化了类型参数 List<User>
// 3. TypeReference 构造函数通过 getClass().getGenericSuperclass() 获取 List<User>

// TypeReference 源码（简化）
public abstract class TypeReference<T> {
    protected final Type _type;

    protected TypeReference() {
        Type superClass = getClass().getGenericSuperclass();
        // superClass = TypeReference<List<User>>（匿名子类的泛型父类）
        _type = ((ParameterizedType) superClass).getActualTypeArguments()[0];
        // _type = List<User>（获取到了！）
    }
}
```

## 3.5 递归解析复杂嵌套泛型

```java
// GenericReflectionDemo.java → resolveType()
// 解析 Map<String, List<User>> 的完整类型树

public static String resolveType(Type type) {
    if (type instanceof Class) {
        return ((Class<?>) type).getSimpleName();

    } else if (type instanceof ParameterizedType) {
        ParameterizedType pt = (ParameterizedType) type;
        String rawName = ((Class<?>) pt.getRawType()).getSimpleName();
        String typeArgs = Arrays.stream(pt.getActualTypeArguments())
            .map(GenericReflectionDemo::resolveType)  // 递归解析
            .collect(Collectors.joining(", "));
        return rawName + "<" + typeArgs + ">";
    }
    return type.toString();
}

// resolveType(Map<String, List<User>>) 的输出：
// "Map<String, List<User>>"
```

## 3.6 本章总结

- **四种保留泛型信息的入口**：字段声明、方法参数、方法返回值、类的泛型父类
- **Type 体系**：`Class`（普通类）、`ParameterizedType`（参数化类型）、`TypeVariable`（类型变量）、`WildcardType`（通配符）、`GenericArrayType`（泛型数组）
- **最重要的模式**：通过子类继承固化类型参数，父类构造函数用 `getGenericSuperclass()` 读取
- **Jackson TypeReference**：匿名子类 + 固化类型参数，绕过类型擦除传递 `List<User>` 类型信息
