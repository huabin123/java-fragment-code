# 第五章：反射实战案例

## 5.1 迷你 ORM 框架

`ReflectionApplicationDemo.java` 中的 `SimpleOrm` 展示了用反射实现 ORM 映射的完整流程：

```java
// 注解定义
@Table(name = "t_user")
public class UserEntity {
    @Id
    @Column(name = "id")
    private Long id;

    @Column(name = "user_name")
    private String username;

    @Column(name = "email")
    private String email;

    @Transient  // 不映射到数据库
    private String tempToken;
}

// ORM 核心逻辑
public class SimpleOrm {

    // 生成 INSERT SQL
    public String generateInsertSql(Object entity) throws Exception {
        Class<?> clazz = entity.getClass();
        Table table = clazz.getAnnotation(Table.class);
        if (table == null) throw new IllegalArgumentException("缺少 @Table 注解");

        List<String> columns = new ArrayList<>();
        List<String> values = new ArrayList<>();

        for (Field field : clazz.getDeclaredFields()) {
            if (!field.isAnnotationPresent(Column.class)) continue;
            if (field.isAnnotationPresent(Transient.class)) continue;

            field.setAccessible(true);
            Column col = field.getAnnotation(Column.class);
            columns.add(col.name());
            values.add("'" + field.get(entity) + "'");
        }

        return String.format("INSERT INTO %s (%s) VALUES (%s)",
            table.name(),
            String.join(", ", columns),
            String.join(", ", values));
    }

    // ResultSet → 对象映射（SELECT 结果反向映射）
    public <T> T mapRow(ResultSet rs, Class<T> type) throws Exception {
        T obj = type.getDeclaredConstructor().newInstance();

        for (Field field : type.getDeclaredFields()) {
            if (!field.isAnnotationPresent(Column.class)) continue;

            Column col = field.getAnnotation(Column.class);
            field.setAccessible(true);

            Object value = rs.getObject(col.name());
            // 类型转换
            if (value != null && !field.getType().isInstance(value)) {
                value = convertType(value, field.getType());
            }
            field.set(obj, value);
        }
        return obj;
    }
}
```

## 5.2 迷你测试框架（JUnit 原理）

```java
// ReflectionApplicationDemo.java → MiniTestRunner
public class MiniTestRunner {

    public void run(Class<?> testClass) throws Exception {
        Object testInstance = testClass.getDeclaredConstructor().newInstance();

        // 查找所有 @Before 方法（每个测试前执行）
        List<Method> beforeMethods = findAnnotatedMethods(testClass, Before.class);

        // 查找所有 @Test 方法
        List<Method> testMethods = findAnnotatedMethods(testClass, Test.class);

        int passed = 0, failed = 0;
        for (Method testMethod : testMethods) {
            try {
                // 执行 @Before 方法
                for (Method before : beforeMethods) {
                    before.setAccessible(true);
                    before.invoke(testInstance);
                }

                // 执行测试方法
                testMethod.setAccessible(true);
                testMethod.invoke(testInstance);

                System.out.println("✅ PASSED: " + testMethod.getName());
                passed++;
            } catch (InvocationTargetException e) {
                Throwable cause = e.getCause();
                // 检查是否是预期异常（@Test(expected = xxx.class)）
                Test ann = testMethod.getAnnotation(Test.class);
                if (ann.expected() != Test.None.class
                        && ann.expected().isInstance(cause)) {
                    System.out.println("✅ PASSED (expected exception): " + testMethod.getName());
                    passed++;
                } else {
                    System.out.println("❌ FAILED: " + testMethod.getName() + " - " + cause.getMessage());
                    failed++;
                }
            }
        }
        System.out.printf("结果：%d 通过，%d 失败%n", passed, failed);
    }

    private List<Method> findAnnotatedMethods(Class<?> clazz, Class<? extends Annotation> annotation) {
        return Arrays.stream(clazz.getDeclaredMethods())
            .filter(m -> m.isAnnotationPresent(annotation))
            .collect(Collectors.toList());
    }
}
```

## 5.3 迷你依赖注入容器

```java
// ReflectionApplicationDemo.java → MiniIoC（Spring IoC 的迷你版）
public class MiniIoC {
    private final Map<Class<?>, Object> beans = new HashMap<>();

    // 注册 Bean
    public void register(Class<?>... classes) throws Exception {
        // 第一步：创建所有实例
        for (Class<?> clazz : classes) {
            if (!clazz.isAnnotationPresent(Component.class)) continue;
            Object instance = clazz.getDeclaredConstructor().newInstance();
            beans.put(clazz, instance);
            // 也按接口类型注册
            for (Class<?> iface : clazz.getInterfaces()) {
                beans.put(iface, instance);
            }
        }

        // 第二步：注入依赖（在所有实例都创建后才注入，避免顺序问题）
        for (Object bean : beans.values()) {
            injectDependencies(bean);
        }
    }

    private void injectDependencies(Object bean) throws Exception {
        for (Field field : bean.getClass().getDeclaredFields()) {
            if (!field.isAnnotationPresent(Autowired.class)) continue;

            field.setAccessible(true);
            Object dependency = beans.get(field.getType());

            if (dependency == null) {
                throw new RuntimeException("未找到类型 " + field.getType() + " 的 Bean");
            }
            field.set(bean, dependency);
        }
    }

    public <T> T getBean(Class<T> type) {
        return type.cast(beans.get(type));
    }
}
```

## 5.4 本章总结

- **迷你 ORM**：`@Table/@Column` 注解 + 反射遍历字段 + 动态 SQL 生成，这是 JPA/MyBatis 的核心思路
- **迷你测试框架**：反射查找 `@Test` 方法并调用，`InvocationTargetException` 解包获取实际失败原因
- **迷你 IoC**：两步法（先创建所有实例，再注入依赖）避免循环依赖初始化问题
