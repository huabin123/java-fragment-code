# 第二章：反射操作字段与方法

## 2.1 字段反射的实战模式

`FieldReflectionDemo.java` 展示了字段反射在框架级代码中的三种核心用法：

### 模式一：ORM 字段-列映射

```java
// FieldReflectionDemo.java → buildInsertSQL()
// 遍历对象所有字段，生成 INSERT SQL
public String buildInsertSQL(Object entity) throws Exception {
    Class<?> clazz = entity.getClass();

    // 获取表名（类上的 @Table 注解）
    String tableName = clazz.getAnnotation(Table.class).name();

    List<String> columns = new ArrayList<>();
    List<Object> values = new ArrayList<>();

    for (Field field : clazz.getDeclaredFields()) {
        if (!field.isAnnotationPresent(Column.class)) continue;
        if (field.isAnnotationPresent(Transient.class)) continue;  // 跳过不映射字段

        Column col = field.getAnnotation(Column.class);
        columns.add(col.name());

        field.setAccessible(true);
        values.add(field.get(entity));
    }

    return "INSERT INTO " + tableName
        + "(" + String.join(", ", columns) + ")"
        + " VALUES (" + buildPlaceholders(values.size()) + ")";
}
```

### 模式二：深度对象拷贝

```java
// FieldReflectionDemo.java → deepCopy()
public static <T> T deepCopy(T source) throws Exception {
    Class<?> clazz = source.getClass();

    // 创建新实例
    T copy = (T) clazz.getDeclaredConstructor().newInstance();

    // 遍历所有字段（含父类字段）
    Class<?> current = clazz;
    while (current != null && current != Object.class) {
        for (Field field : current.getDeclaredFields()) {
            if (Modifier.isStatic(field.getModifiers())) continue;  // 跳过静态字段

            field.setAccessible(true);
            Object value = field.get(source);

            // 递归拷贝嵌套对象
            if (value != null && !isPrimitive(value.getClass())) {
                value = deepCopy(value);
            }
            field.set(copy, value);
        }
        current = current.getSuperclass();  // 递归处理父类字段
    }
    return copy;
}
```

**注意遍历父类字段**：`getDeclaredFields()` 只返回本类字段，父类的 `private` 字段需要通过 `getSuperclass()` 向上遍历获取。

### 模式三：对象属性对比（变更检测）

```java
// FieldReflectionDemo.java → findDifferences()
public List<String> findDifferences(Object obj1, Object obj2) throws Exception {
    List<String> diffs = new ArrayList<>();
    for (Field field : obj1.getClass().getDeclaredFields()) {
        field.setAccessible(true);
        Object v1 = field.get(obj1);
        Object v2 = field.get(obj2);
        if (!Objects.equals(v1, v2)) {
            diffs.add(field.getName() + ": " + v1 + " → " + v2);
        }
    }
    return diffs;
}
// 审计日志：记录更新前后的字段变化
```

## 2.2 方法反射的实战模式

`MethodReflectionDemo.java` 展示了方法反射的三种典型应用：

### 模式一：插件系统动态调用

```java
// MethodReflectionDemo.java → invokePlugin()
// 根据配置文件中的方法名，动态调用对应方法
public Object executeCommand(Object plugin, String commandName, Object... args)
        throws Exception {
    // 查找匹配的方法（名称 + 参数类型自动匹配）
    for (Method method : plugin.getClass().getDeclaredMethods()) {
        if (!method.getName().equals(commandName)) continue;
        if (method.getParameterCount() != args.length) continue;

        method.setAccessible(true);
        return method.invoke(plugin, args);
    }
    throw new IllegalArgumentException("未找到方法: " + commandName);
}
```

### 模式二：事件监听器动态注册

```java
// MethodReflectionDemo.java → registerListeners()
// 扫描类上所有 @EventListener 方法，自动注册
public void registerListeners(Object listener) {
    for (Method method : listener.getClass().getDeclaredMethods()) {
        if (!method.isAnnotationPresent(EventListener.class)) continue;

        EventListener ann = method.getAnnotation(EventListener.class);
        String eventType = ann.value();

        method.setAccessible(true);
        // 注册到事件总线
        eventBus.register(eventType, (event) -> {
            try {
                method.invoke(listener, event);
            } catch (InvocationTargetException e) {
                throw new RuntimeException(e.getCause());
            } catch (IllegalAccessException e) {
                throw new RuntimeException(e);
            }
        });
    }
}
```

### 模式三：链式方法调用（Builder 模式的运行时版）

```java
// MethodReflectionDemo.java → applyProperties()
// 根据 Map 中的属性名，动态调用对应 setter
public void applyProperties(Object bean, Map<String, Object> properties)
        throws Exception {
    for (Map.Entry<String, Object> entry : properties.entrySet()) {
        String propertyName = entry.getKey();
        Object value = entry.getValue();

        // 构造 setter 方法名：name → setName
        String setterName = "set" + Character.toUpperCase(propertyName.charAt(0))
            + propertyName.substring(1);

        try {
            Method setter = bean.getClass().getMethod(setterName, value.getClass());
            setter.invoke(bean, value);
        } catch (NoSuchMethodException e) {
            // setter 不存在，跳过（或记录日志）
        }
    }
}
// 用途：Spring BeanWrapper、MyBatis 结果映射、配置类自动填充
```

## 2.3 `setAccessible(true)` 的安全含义与限制

```java
// 正常情况：private 字段无法从外部访问
Field field = SomeClass.class.getDeclaredField("secret");
field.get(obj);  // 抛出 IllegalAccessException

// setAccessible(true) 绕过访问控制
field.setAccessible(true);  // 告诉 JVM 不做访问控制检查
field.get(obj);  // 成功访问 private 字段

// JDK 9+ 模块化系统的新限制：
// 对不同模块的类调用 setAccessible(true) 会抛出 InaccessibleObjectException
// 除非该模块在 module-info.java 中声明了 opens
module myapp {
    opens com.example.model to myframework;  // 允许 myframework 模块反射访问
}
```

**安全风险**：`setAccessible(true)` 可以修改 `private final` 字段，破坏不可变性：

```java
// 可以修改 private final 字段（理论上，实际受 JIT 内联优化影响）
Field field = String.class.getDeclaredField("value");
field.setAccessible(true);
field.set("hello", new char[]{'w', 'o', 'r', 'l', 'd'});
// 这是非常危险的操作，不应在生产代码中出现！
```

## 2.4 异常处理的最佳实践

```java
// 反射操作的标准异常处理模式
public Object safeInvoke(Method method, Object target, Object... args) {
    try {
        method.setAccessible(true);
        return method.invoke(target, args);
    } catch (IllegalAccessException e) {
        // 通常不会发生（已 setAccessible），如果发生说明有模块限制
        throw new ReflectionException("访问被拒绝: " + method.getName(), e);
    } catch (InvocationTargetException e) {
        // 目标方法本身抛出的异常，必须解包
        Throwable cause = e.getCause();
        if (cause instanceof RuntimeException) throw (RuntimeException) cause;
        if (cause instanceof Error) throw (Error) cause;
        throw new ReflectionException("方法调用失败: " + method.getName(), cause);
    } catch (IllegalArgumentException e) {
        // 参数类型不匹配
        throw new ReflectionException("参数类型错误: " + method.getName(), e);
    }
}
```

## 2.5 本章总结

- **字段反射三模式**：ORM 映射（字段→列）、深度拷贝（含父类字段遍历）、变更检测（属性对比）
- **方法反射三模式**：插件动态调用、注解驱动事件注册、属性名→setter 自动调用
- **父类字段**：`getDeclaredFields()` 不含父类，需用 `getSuperclass()` 向上遍历
- **setAccessible 限制**：JDK 9+ 跨模块反射需要模块声明 `opens`，不能无限制绕过
- **InvocationTargetException**：目标方法异常的包装类，必须 `getCause()` 解包

> **本章对应演示代码**：`FieldReflectionDemo.java`（ORM映射、深度拷贝、变更检测）、`MethodReflectionDemo.java`（插件调用、事件注册、属性设置）
