# 第一章：动态代理基础原理

## 1.1 为什么需要代理？

代理模式解决的核心问题：**在不修改原始类的情况下，为其增加横切关注点**。

```java
// 没有代理：横切关注点（日志、事务、权限）侵入业务逻辑
public class UserServiceImpl implements UserService {
    public User findById(Long id) {
        log.info("开始查询用户 {}", id);      // 业务无关代码
        long start = System.currentTimeMillis();

        User user = database.query(id);        // ↑ 真正的业务逻辑只有这一行

        log.info("查询耗时 {}ms", System.currentTimeMillis() - start);
        return user;
    }
    // 每个方法都要重复这些代码
}

// 有代理：业务代码干净，横切逻辑集中在代理中
public class UserServiceImpl implements UserService {
    public User findById(Long id) {
        return database.query(id);  // 只有业务逻辑
    }
}
// 日志、性能监控、权限校验全部在代理里统一处理
```

**代理的本质**：代理对象与目标对象实现相同的接口（或是其子类），调用方感知不到差异，但代理在调用前后可以插入任意逻辑。

## 1.2 静态代理 vs 动态代理

### 静态代理：手工编写代理类

```java
// 必须手工为每个接口写一个代理类
public class UserServiceProxy implements UserService {
    private UserService target;

    public UserServiceProxy(UserService target) {
        this.target = target;
    }

    @Override
    public User findById(Long id) {
        System.out.println("代理 - 方法调用前: findById");
        User result = target.findById(id);
        System.out.println("代理 - 方法调用后: findById");
        return result;
    }

    @Override
    public void save(User user) {
        System.out.println("代理 - 方法调用前: save");
        target.save(user);
        System.out.println("代理 - 方法调用后: save");
    }
    // ...还需要为每个方法重复这套模板代码
}
```

**静态代理的问题**：
- 每个接口都需要手写一个代理类，代码量大
- 接口新增方法后，代理类也必须跟着修改
- 横切逻辑重复分散在每个方法中

### 动态代理：运行时生成代理类

动态代理在运行时通过字节码技术生成代理类，无需手工编写，所有方法的增强逻辑集中在一个 `InvocationHandler` 中。

---

## 1.3 JDK 动态代理的三要素

`JdkProxyDemo.java` 展示了完整用法：

```java
// 三要素：ClassLoader + 接口数组 + InvocationHandler
UserService proxy = (UserService) Proxy.newProxyInstance(
    target.getClass().getClassLoader(),    // 1. 类加载器：用哪个 ClassLoader 加载生成的代理类
    target.getClass().getInterfaces(),     // 2. 接口数组：代理类要实现哪些接口
    new LoggingInvocationHandler(target)   // 3. 处理器：所有方法调用都转发到这里
);
```

**InvocationHandler 的实现**：

```java
// JdkProxyDemo.java → LoggingInvocationHandler
class LoggingInvocationHandler implements InvocationHandler {
    private final Object target;

    public LoggingInvocationHandler(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        // 方法调用前增强
        System.out.println("[LOG] 调用方法: " + method.getName());
        long start = System.currentTimeMillis();

        // 调用目标方法
        Object result = method.invoke(target, args);

        // 方法调用后增强
        long cost = System.currentTimeMillis() - start;
        System.out.println("[LOG] 方法完成: " + method.getName() + ", 耗时: " + cost + "ms");

        return result;
    }
}
```

三个参数的含义：
- `proxy`：生成的代理对象本身（很少直接使用，避免递归调用）
- `method`：当前被调用的方法对象（含方法名、参数类型、返回类型）
- `args`：调用时传入的实际参数

## 1.4 代理类的运行时结构

JDK 动态代理生成的代理类（反编译后大致结构）：

```java
// 运行时生成的类，类似于：
public final class $Proxy0 extends Proxy implements UserService {

    // 缓存所有方法的 Method 对象（避免每次反射查找）
    private static Method m1;  // findById
    private static Method m2;  // save
    // ...

    static {
        m1 = Class.forName("UserService").getMethod("findById", Long.class);
        m2 = Class.forName("UserService").getMethod("save", User.class);
    }

    // 构造函数：接收 InvocationHandler
    public $Proxy0(InvocationHandler h) { super(h); }

    // 每个接口方法都转发给 InvocationHandler
    public User findById(Long id) {
        return (User) h.invoke(this, m1, new Object[]{id});
    }

    public void save(User user) {
        h.invoke(this, m2, new Object[]{user});
    }
}
```

---

## 1.5 JDK 动态代理的核心限制

**必须有接口**：JDK 动态代理通过 `extends Proxy implements 接口` 实现，若目标类没有实现接口，无法使用 JDK 代理。

```java
// ❌ 没有接口的类无法用 JDK 动态代理
Calculator calculator = new Calculator();  // 没有实现任何接口
Proxy.newProxyInstance(
    calculator.getClass().getClassLoader(),
    calculator.getClass().getInterfaces(),  // 返回空数组！
    handler
);  // 生成的代理没有任何方法可以调用
```

这就是 CGLIB 存在的原因——通过创建子类绕过"必须有接口"的限制。

---

## 1.6 本章总结

- **代理的价值**：在不修改业务代码的情况下，统一添加日志、事务、权限等横切关注点
- **静态代理缺点**：手工编写，每个接口一个代理类，方法重复代码多
- **JDK 代理三要素**：ClassLoader + 接口数组 + InvocationHandler
- **invoke 三参数**：proxy（代理对象）、method（方法反射对象）、args（实际参数）
- **核心限制**：必须基于接口，无接口时用 CGLIB
