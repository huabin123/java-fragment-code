# 第二章：AIO 核心组件详解

## 2.1 AsynchronousChannelGroup：线程池管理

```java
// CompletionHandlerDemo.java → demonstrateChannelGroup()

// AIO 的所有回调都在 ChannelGroup 绑定的线程池中执行
// 不指定时使用默认全局线程池

// 自定义线程池（生产推荐）
ExecutorService executor = new ThreadPoolExecutor(
    4, 16,
    60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(256),
    r -> new Thread(r, "aio-worker-" + count.incrementAndGet())
);
AsynchronousChannelGroup group = AsynchronousChannelGroup.withThreadPool(executor);

// 使用自定义 group 创建 Channel
AsynchronousServerSocketChannel server =
    AsynchronousServerSocketChannel.open(group);
AsynchronousSocketChannel client =
    AsynchronousSocketChannel.open(group);

// group 关闭时等待所有已提交任务完成
group.shutdown();
group.awaitTermination(30, TimeUnit.SECONDS);

// 注意：
// 1. 回调中不要执行耗时操作（阻塞 AIO 线程池，影响其他 I/O 完成通知）
// 2. 业务逻辑应提交到独立的业务线程池处理
```

## 2.2 AsynchronousServerSocketChannel：异步服务端

```java
// AsynchronousSocketChannelDemo.java → AioServer

AsynchronousServerSocketChannel server =
    AsynchronousServerSocketChannel.open()
        .bind(new InetSocketAddress(8080), 128);  // 第二个参数：backlog

// accept() 是异步的，注册后立即返回
server.accept(null, new CompletionHandler<AsynchronousSocketChannel, Void>() {
    @Override
    public void completed(AsynchronousSocketChannel client, Void attachment) {
        // 有新连接到来
        System.out.println("新连接: " + client.getRemoteAddress());

        // ❗ 立即再次调用 accept()，继续接受下一个连接
        // （AIO 的 accept 是一次性的，处理完一个连接后必须重新注册）
        server.accept(null, this);

        // 开始读取该连接的数据
        handleConnection(client);
    }

    @Override
    public void failed(Throwable exc, Void attachment) {
        if (!(exc instanceof AsynchronousCloseException)) {
            exc.printStackTrace();
            server.accept(null, this);  // 出错后继续接受新连接
        }
    }
});

// 主线程等待（AIO 回调在独立线程池中执行，主线程不能退出）
Thread.currentThread().join();
```

## 2.3 AsynchronousSocketChannel：异步读写

```java
// AsynchronousSocketChannelDemo.java → handleConnection()

private void handleConnection(AsynchronousSocketChannel channel) {
    ByteBuffer buffer = ByteBuffer.allocate(4096);

    // 异步读（带超时）
    channel.read(buffer, 30L, TimeUnit.SECONDS, buffer,
        new CompletionHandler<Integer, ByteBuffer>() {
            @Override
            public void completed(Integer bytesRead, ByteBuffer buf) {
                if (bytesRead == -1) {
                    closeChannel(channel);
                    return;
                }

                buf.flip();
                byte[] data = new byte[buf.remaining()];
                buf.get(data);

                // 处理数据，然后写回响应
                ByteBuffer response = ByteBuffer.wrap(process(data));
                writeResponse(channel, response, buf);
            }

            @Override
            public void failed(Throwable exc, ByteBuffer buf) {
                if (exc instanceof InterruptedByTimeoutException) {
                    System.out.println("读超时，关闭连接");
                }
                closeChannel(channel);
            }
        }
    );
}

private void writeResponse(AsynchronousSocketChannel channel,
                           ByteBuffer response, ByteBuffer readBuf) {
    channel.write(response, response, new CompletionHandler<Integer, ByteBuffer>() {
        @Override
        public void completed(Integer bytesWritten, ByteBuffer resp) {
            if (resp.hasRemaining()) {
                // 未写完，继续写（write 可能不会一次写完）
                channel.write(resp, resp, this);
            } else {
                // 写完，继续读下一条消息
                readBuf.clear();
                handleConnection(channel);  // 递归继续读
            }
        }

        @Override
        public void failed(Throwable exc, ByteBuffer resp) {
            closeChannel(channel);
        }
    });
}
```

## 2.4 AsynchronousFileChannel：异步文件 I/O

```java
// AsynchronousFileChannelDemo.java → demonstrateAsyncFileRead()
// AIO 最有价值的应用场景：大文件异步读写

AsynchronousFileChannel fileChannel = AsynchronousFileChannel.open(
    Paths.get("large-file.dat"),
    StandardOpenOption.READ
);

ByteBuffer buffer = ByteBuffer.allocate(8192);
long position = 0;

// 异步读文件（从 position 位置开始读）
fileChannel.read(buffer, position, null, new CompletionHandler<Integer, Void>() {
    @Override
    public void completed(Integer bytesRead, Void attachment) {
        if (bytesRead == -1) {
            System.out.println("文件读取完毕");
            fileChannel.close();
            return;
        }
        buffer.flip();
        processChunk(buffer);
        buffer.clear();
        // 读下一块（更新 position）
        fileChannel.read(buffer, position + bytesRead, null, this);
    }

    @Override
    public void failed(Throwable exc, Void attachment) {
        exc.printStackTrace();
    }
});

// 也可以用 Future 方式（适合简单场景）
Future<Integer> future = fileChannel.read(buffer, 0);
int bytesRead = future.get(5, TimeUnit.SECONDS);  // 最多等5秒
```

## 2.5 用 CompletableFuture 组合异步操作

```java
// CompletionHandlerDemo.java → demonstrateCompletableFuture()
// 解决回调地狱：将 AIO 回调包装为 CompletableFuture

// 工具方法：将 AIO 读操作包装为 CompletableFuture
public static CompletableFuture<Integer> asyncRead(
        AsynchronousSocketChannel channel, ByteBuffer buffer) {
    CompletableFuture<Integer> future = new CompletableFuture<>();
    channel.read(buffer, null, new CompletionHandler<Integer, Void>() {
        @Override public void completed(Integer n, Void v) { future.complete(n); }
        @Override public void failed(Throwable e, Void v) { future.completeExceptionally(e); }
    });
    return future;
}

// 链式组合，避免回调嵌套
asyncRead(channel, buffer)
    .thenApply(n -> { buffer.flip(); return decode(buffer); })
    .thenApply(message -> handleBusiness(message))
    .thenCompose(response -> asyncWrite(channel, encode(response)))
    .exceptionally(e -> { closeChannel(channel); return null; });
```

## 2.6 本章总结

- **ChannelGroup**：管理 AIO 线程池，回调不做耗时操作，业务逻辑提交到独立线程池
- **accept() 一次性**：每次 accept 完成后必须立即重新调用，否则不再接受新连接
- **read/write 带超时**：推荐使用带超时的重载版本，防止连接永久挂起
- **AsynchronousFileChannel**：文件异步 I/O 是 AIO 最有价值的使用场景
- **CompletableFuture 包装**：解决多层嵌套回调的可读性问题
