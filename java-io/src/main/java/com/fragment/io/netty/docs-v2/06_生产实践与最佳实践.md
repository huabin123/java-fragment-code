# 第六章：生产实践与最佳实践

## 6.1 心跳检测：感知死连接

TCP 连接在以下情况下会"静默断开"而服务端无法感知：
- 客户端进程崩溃（OS 未发 FIN，如断电）
- 中间防火墙/NAT 超时断开连接
- 网络分区

`HeartbeatDemo.java` 演示了完整的心跳实现方案：

```java
// 服务端 Pipeline：空闲检测 + 心跳处理
pipeline.addLast(new IdleStateHandler(
    60,   // readerIdleTime：60秒未收到任何数据 → 触发 READER_IDLE 事件
    0,    // writerIdleTime：0=不检测写空闲
    0     // allIdleTime：0=不检测全空闲
));
pipeline.addLast(new ServerHeartbeatHandler());

// 服务端心跳处理 Handler
public class ServerHeartbeatHandler extends ChannelInboundHandlerAdapter {

    @Override
    public void userEventTriggered(ChannelHandlerContext ctx, Object evt) {
        if (evt instanceof IdleStateEvent) {
            IdleStateEvent event = (IdleStateEvent) evt;
            if (event.state() == IdleState.READER_IDLE) {
                // 60秒没收到客户端任何数据（包括心跳）→ 认定连接死亡
                System.out.println("连接空闲超时，主动关闭: " + ctx.channel().remoteAddress());
                ctx.close();
            }
        } else {
            ctx.fireUserEventTriggered(evt);
        }
    }

    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) {
        // 收到心跳包，回复 PONG
        if (msg instanceof String && "PING".equals(msg)) {
            ctx.writeAndFlush("PONG\n");
        } else {
            ctx.fireChannelRead(msg);  // 业务消息继续传播
        }
    }
}

// 客户端 Pipeline：每30秒无写操作时主动发送心跳
pipeline.addLast(new IdleStateHandler(0, 30, 0));
pipeline.addLast(new ClientHeartbeatHandler());

public class ClientHeartbeatHandler extends ChannelInboundHandlerAdapter {
    @Override
    public void userEventTriggered(ChannelHandlerContext ctx, Object evt) {
        if (evt instanceof IdleStateEvent) {
            // 30秒没有向服务端发送数据，主动发 PING
            ctx.writeAndFlush("PING\n");
        }
    }
}
```

## 6.2 连接管理：ChannelGroup

```java
// 用 ChannelGroup 管理所有连接（线程安全）
// project/push/ 目录中的推送服务就是基于此实现的

private static final ChannelGroup ALL_CHANNELS =
    new DefaultChannelGroup(GlobalEventExecutor.INSTANCE);

// 连接建立时加入
@Override
public void channelActive(ChannelHandlerContext ctx) {
    ALL_CHANNELS.add(ctx.channel());
    System.out.println("在线人数: " + ALL_CHANNELS.size());
}

// 连接断开时自动移除（ChannelGroup 会自动清理已关闭的 Channel）

// 广播消息给所有在线用户
ALL_CHANNELS.writeAndFlush(Unpooled.copiedBuffer("系统通知: 服务将于5分钟后重启\n",
    CharsetUtil.UTF_8));

// 优雅关闭所有连接
ALL_CHANNELS.close().awaitUninterruptibly();
```

## 6.3 优雅关闭

```java
// 不优雅关闭会导致：正在处理的请求被中断、客户端收到 Connection Reset
// 正确的关闭流程：

EventLoopGroup boss   = new NioEventLoopGroup(1);
EventLoopGroup worker = new NioEventLoopGroup();

// 注册 JVM 关闭钩子
Runtime.getRuntime().addShutdownHook(new Thread(() -> {
    System.out.println("开始优雅关闭...");

    // 1. 停止接受新连接
    boss.shutdownGracefully(0, 5, TimeUnit.SECONDS);

    // 2. 等待正在进行的 I/O 操作完成（最多等5秒）
    worker.shutdownGracefully(0, 5, TimeUnit.SECONDS);

    try {
        boss.terminationFuture().sync();
        worker.terminationFuture().sync();
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
    System.out.println("Netty 已关闭");
}));

// shutdownGracefully(quietPeriod, timeout, unit)
// quietPeriod：安静期，期间无任务提交才确认关闭（默认2秒）
// timeout：最长等待时间（默认15秒）
```

## 6.4 生产参数配置

```java
// 服务端完整生产配置
new ServerBootstrap()
    .group(boss, worker)
    .channel(NioServerSocketChannel.class)

    // 服务端 Socket 参数
    .option(ChannelOption.SO_BACKLOG, 1024)          // 连接队列大小（影响并发连接数）
    .option(ChannelOption.SO_REUSEADDR, true)         // 地址复用（快速重启服务）

    // 客户端 Socket 参数
    .childOption(ChannelOption.SO_KEEPALIVE, true)    // TCP keepalive（OS级别）
    .childOption(ChannelOption.TCP_NODELAY, true)     // 禁用 Nagle 算法（降低延迟）
    .childOption(ChannelOption.SO_RCVBUF, 128 * 1024) // 接收缓冲区 128KB
    .childOption(ChannelOption.SO_SNDBUF, 128 * 1024) // 发送缓冲区 128KB
    .childOption(ChannelOption.WRITE_BUFFER_WATER_MARK,
        new WriteBufferWaterMark(32 * 1024, 64 * 1024)) // 写缓冲水位线，防止 OOM

    .childHandler(new ChannelInitializer<SocketChannel>() {
        @Override
        protected void initChannel(SocketChannel ch) {
            ch.pipeline()
                .addLast(new IdleStateHandler(60, 0, 0))      // 心跳检测
                .addLast(new LengthFieldBasedFrameDecoder(     // 拆包
                    65535, 0, 4, 0, 4))
                .addLast(new LengthFieldPrepender(4))          // 加包头
                .addLast(BUSINESS_POOL, new BusinessHandler());// 业务（独立线程池）
        }
    });
```

## 6.5 常见问题排查

### 内存泄漏（Direct buffer memory OOM）

```bash
# 症状：应用运行一段时间后 OOM: Direct buffer memory
# 排查：开启 Netty 内存泄漏检测
java -Dio.netty.leakDetection.level=advanced -jar app.jar

# 日志中出现类似：
# LEAK: ByteBuf.release() was not called before it's garbage-collected.
#       Recent access records: ...
#       → 找到对应的 Handler，加上 release() 或改用 SimpleChannelInboundHandler
```

### CPU 100%（空轮询 Bug）

```bash
# 症状：CPU 持续 100%，但实际无流量
# Netty 已内置修复（512 次空轮询后重建 Selector）
# 检查：是否在 EventLoop 线程中执行了耗时操作（阻塞了 select）
# jstack <PID> | grep "nioEventLoop" 查看线程在做什么
```

### 消息乱序（未保证写顺序）

```java
// ❌ 错误：多线程同时 writeAndFlush 可能导致消息顺序错误
threadA.execute(() -> ctx.writeAndFlush(msgA));
threadB.execute(() -> ctx.writeAndFlush(msgB));

// ✅ 需要保证顺序时，通过 EventLoop 排队
ctx.channel().eventLoop().execute(() -> {
    ctx.write(msgA);
    ctx.write(msgB);
    ctx.flush();  // 批量 flush，性能更好
});
```

### 写缓冲区积压（Write Buffer Full）

```java
// 场景：发送速度 > 消费速度，写缓冲区无限增长 → OOM
// 解决：检查 Channel.isWritable()，不可写时停止生产

if (ctx.channel().isWritable()) {
    ctx.writeAndFlush(message);
} else {
    // 写缓冲区已满（超过高水位线 64KB）
    // 丢弃或等待，等 channelWritabilityChanged 回调再恢复
}

// 监听水位变化
@Override
public void channelWritabilityChanged(ChannelHandlerContext ctx) {
    if (ctx.channel().isWritable()) {
        resumeProducing();   // 恢复生产
    } else {
        pauseProducing();    // 暂停生产
    }
}
```

## 6.7 本章总结

**五条生产核心实践**：
1. **心跳检测必须做**：`IdleStateHandler`（服务端检测读空闲，客户端检测写空闲），死连接超时主动关闭
2. **ChannelGroup 管理连接**：线程安全，支持广播，自动清理已关闭的 Channel
3. **优雅关闭**：`shutdownGracefully()` + JVM ShutdownHook，避免正在处理的请求被强制中断
4. **写缓冲水位线**：设置 `WriteBufferWaterMark`，配合 `isWritable()` 实现背压控制
5. **TCP 参数**：`TCP_NODELAY=true`（低延迟）、`SO_KEEPALIVE=true`（OS 级保活）、`SO_BACKLOG=1024`（高并发连接）

> **本章对应演示代码**：`HeartbeatDemo.java`（心跳完整实现）；`project/push/`（推送服务，含 ChannelGroup 和断线重连）；`project/rpc/`（完整 RPC 框架）
